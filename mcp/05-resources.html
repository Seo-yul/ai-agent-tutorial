Resources 심화

정적/동적 리소스, URI 패턴, 구독, 그리고 멀티미디어 데이터를 다룬다

Resources 개요

핵심 개념: Resource란 무엇인가

MCP에서 Resource는 서버가 외부에 노출하는 읽기 전용 데이터이다. 고유한 URI로 식별되며, 클라이언트는 이 URI를 통해 데이터에 접근한다. 설정 파일, DB 레코드, API 응답, 이미지 등 다양한 형태의 데이터를 리소스로 제공할 수 있다.

Tools와 Resources의 차이

Tools
  • 부수 효과를 일으킬 수 있다
  • LLM이 직접 호출한다
  • 파라미터를 JSON Schema로 정의한다
  • 데이터 쓰기, API 호출, 파일 변경 등
Resources
  • 읽기 전용, 부수 효과 없음
  • 클라이언트(앱)가 URI로 요청한다
  • 데이터를 URI 패턴으로 식별한다
  • 설정 조회, 파일 읽기, 상태 확인 등

프로토콜 수준 흐름

sequenceDiagram
    participant H as Host (LLM 앱)
    participant C as MCP Client
    participant S as MCP Server

    H->>C: 리소스 목록 조회
    C->>S: resources/list
    S-->>C: Resource[] (uri, name, mimeType)
    C-->>H: 리소스 목록 전달

    H->>C: 리소스 템플릿 조회
    C->>S: resources/templates/list
    S-->>C: ResourceTemplate[] (uriTemplate, name)

    H->>C: 특정 리소스 읽기
    C->>S: resources/read {uri}
    S-->>C: ResourceContents[] (text 또는 blob)
    C-->>H: 데이터 전달
→ resources/read 요청
{
  "method": "resources/read",
  "params": {
    "uri": "config://database"
  }
}
← resources/read 응답
{
  "contents": [
    {
      "uri": "config://database",
      "mimeType": "application/json",
      "text": "{\"host\": \"localhost\", \"port\": 5432}"
    }
  ]
}

정적 리소스

정적 리소스는 고정된 URI로 접근한다. URI에 변수 부분이 없으며, @mcp.resource() 데코레이터에 고정 URI를 전달하여 정의한다. 

import json
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("config-server")

@mcp.resource("config://app")
def get_app_config() -> str:
    """애플리케이션 설정을 반환한다"""
    return json.dumps({
        "version": "2.1.0",
        "debug": False,
        "max_connections": 100,
    }, indent=2)


@mcp.resource("config://features")
def get_feature_flags() -> str:
    """기능 플래그 상태를 반환한다"""
    return json.dumps({
        "new_dashboard": True,
        "beta_api": False,
        "dark_mode": True,
    }, indent=2)
TIP

"정적 리소스"라 해서 반환값이 항상 같아야 하는 것은 아니다. URI가 고정이라는 의미이며, 함수 실행 시점의 최신 데이터를 반환할 수 있다. 예를 들어 system://status는 호출할 때마다 현재 CPU 사용률을 반환할 수 있다.

Resource Template

Resource Template은 URI에 {parameter} 형태의 변수를 포함한다. 클라이언트가 요청할 때 변수에 실제 값이 들어가며, 함수의 파라미터로 자동 매핑된다. 프로토콜에서 resources/templates/list를 통해 사용 가능한 템플릿 목록을 제공한다. 

@mcp.resource("users://{user_id}/profile")
def get_user_profile(user_id: str) -> str:
    """사용자 프로필을 반환한다"""
    profiles = {
        "alice": {"name": "Alice Kim", "role": "developer", "team": "backend"},
        "bob": {"name": "Bob Lee", "role": "designer", "team": "ux"},
    }
    profile = profiles.get(user_id, {})
    return json.dumps(profile, ensure_ascii=False, indent=2)

users://alice/profile로 요청하면 user_id="alice"가 함수에 전달된다.

다중 경로 파라미터

@mcp.resource("repos://{owner}/{repo}/info")
def get_repo_info(owner: str, repo: str) -> str:
    """리포지토리 정보를 반환한다"""
    return json.dumps({
        "owner": owner,
        "repo": repo,
        "full_name": f"{owner}/{repo}",
        "url": f"https://github.com/{owner}/{repo}",
    }, indent=2)

# 요청 예시: repos://anthropic/mcp-python-sdk/info
# -> owner="anthropic", repo="mcp-python-sdk"

반환 타입

리소스 함수는 다양한 타입을 반환할 수 있다. FastMCP는 반환값의 타입에 따라 적절한 MIME 타입을 자동 추론한다.

문자열 반환 (text)

@mcp.resource("docs://readme")
def get_readme() -> str:
    """README 문서를 반환한다"""
    return """# 프로젝트 소개
이 프로젝트는 MCP 서버 예제이다.

## 설치 방법
pip install -r requirements.txt
"""
# mimeType: "text/plain" (자동 추론)

JSON 반환

@mcp.resource("data://products")
def get_products() -> str:
    """상품 목록을 JSON으로 반환한다"""
    products = [
        {"id": 1, "name": "노트북", "price": 1200000},
        {"id": 2, "name": "마우스", "price": 35000},
    ]
    return json.dumps(products, ensure_ascii=False, indent=2)
# mimeType: "text/plain" (기본값, 문자열이므로)

바이너리 반환 (이미지 등)

@mcp.resource("images://{name}")
def get_image(name: str) -> bytes:
    """이미지 파일을 바이너리로 반환한다"""
    image_path = Path(f"./images/{name}")
    if not image_path.exists():
        return b""
    return image_path.read_bytes()
# bytes 반환 -> base64 인코딩 -> BlobResourceContents
# mimeType: 확장자에서 자동 추론 (예: .png -> "image/png")

MIME 타입 명시적 지정

Deep Dive: MIME 타입을 명시적으로 지정해야 하는 경우

FastMCP는 반환 타입과 URI 확장자로 MIME 타입을 자동 추론한다. 하지만 URI에 확장자가 없거나, 추론 결과가 정확하지 않은 경우에는 mime_type 파라미터로 명시적으로 지정해야 한다. 특히 JSON 데이터를 반환하면서 클라이언트가 이를 JSON으로 파싱하길 원한다면 "application/json"을 명시하는 것이 좋다. 

# MIME 타입을 명시적으로 지정하는 예제

@mcp.resource("config://database", mime_type="application/json")
def get_db_config() -> str:
    """데이터베이스 설정을 JSON으로 반환한다"""
    return json.dumps({
        "host": "localhost",
        "port": 5432,
        "name": "myapp",
    })

@mcp.resource("reports://{id}/chart", mime_type="image/svg+xml")
def get_chart_svg(id: str) -> str:
    """차트를 SVG 형식으로 반환한다"""
    return f"""<svg xmlns="http://www.w3.org/2000/svg" width="200" height="100">
  <rect width="100" height="{id}0" fill="steelblue"/>
</svg>"""

@mcp.resource("data://{name}.csv", mime_type="text/csv")
def get_csv_data(name: str) -> str:
    """CSV 데이터를 반환한다"""
    return "id,name,value\n1,alpha,100\n2,beta,200\n"

리소스 구독

Deep Dive: 리소스 구독과 실시간 업데이트

MCP는 리소스 변경을 실시간으로 감지하는 구독(subscription) 메커니즘을 지원한다. 클라이언트가 특정 리소스 URI를 구독하면, 서버는 해당 리소스가 변경될 때 notifications/resources/updated 알림을 보낸다. 클라이언트는 이 알림을 받으면 resources/read로 최신 데이터를 다시 요청한다. 

sequenceDiagram
    participant C as MCP Client
    participant S as MCP Server

    C->>S: resources/subscribe {uri: "config://app"}
    S-->>C: OK

    Note over S: 설정 데이터가 변경됨

    S--)C: notifications/resources/updated {uri: "config://app"}
    C->>S: resources/read {uri: "config://app"}
    S-->>C: 최신 데이터 반환

    C->>S: resources/unsubscribe {uri: "config://app"}
    S-->>C: OK
→ notifications/resources/updated
{
  "method": "notifications/resources/updated",
  "params": {
    "uri": "config://app"
  }
}

구독 가능한 리소스 구현

FastMCP에서 리소스 변경 알림을 보내려면 서버 인스턴스의 알림 메서드를 사용한다. 리소스 데이터가 변경되는 시점에 명시적으로 알림을 발행해야 한다. 

import json
import asyncio
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("realtime-config")

# 인메모리 설정 저장소
_config = {
    "theme": "light",
    "language": "ko",
    "notifications_enabled": True,
}


@mcp.resource("config://app")
def get_config() -> str:
    """현재 애플리케이션 설정을 반환한다"""
    return json.dumps(_config, ensure_ascii=False, indent=2)


@mcp.tool()
async def update_config(key: str, value: str) -> str:
    """설정 값을 업데이트한다

    Args:
        key: 설정 키 (theme, language, notifications_enabled)
        value: 새 값
    """
    if key not in _config:
        raise ValueError(f"알 수 없는 설정 키이다: {key}")

    # 값 업데이트
    old_value = _config[key]
    # boolean 변환 처리
    if isinstance(_config[key], bool):
        _config[key] = value.lower() in ("true", "1", "yes")
    else:
        _config[key] = value

    # 구독자에게 변경 알림을 보낸다
    await mcp.notify_resource_updated("config://app")

    return f"설정 변경 완료: {key} = {old_value} -> {_config[key]}"
INFO

notify_resource_updated(uri)는 해당 URI를 구독 중인 클라이언트에게만 알림을 보낸다. 구독하지 않은 클라이언트는 알림을 받지 않는다. 알림은 "데이터가 변경되었다"는 신호만 전송하며, 변경된 데이터 자체를 포함하지 않는다. 클라이언트가 resources/read로 새 데이터를 직접 요청해야 한다.

리소스 목록 변경 알림

Deep Dive: 리소스 목록 자체가 변경될 때

기존 리소스의 내용이 변경되는 것과, 리소스 목록 자체가 변경되는 것은 다르다. 새 리소스가 추가되거나, 기존 리소스가 삭제되면 notifications/resources/list_changed를 보내야 한다. 이를 받은 클라이언트는 resources/list를 다시 호출하여 최신 목록을 갱신한다.

resources/updated (내용 변경)

기존 리소스의 데이터가 변경됨

클라이언트: resources/read를 재호출

await mcp.notify_resource_updated(
    "config://app"
)
resources/list_changed (목록 변경)

리소스가 추가/삭제됨

클라이언트: resources/list를 재호출

await mcp.notify_resource_list_changed()
import json
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("dynamic-registry")

# 동적 리소스 레지스트리
_registry: dict[str, dict] = {}


@mcp.tool()
async def register_service(name: str, url: str, description: str = "") -> str:
    """새 서비스를 리소스로 등록한다

    Args:
        name: 서비스 이름 (URI 식별자)
        url: 서비스 URL
        description: 서비스 설명
    """
    _registry[name] = {"url": url, "description": description}

    # 동적으로 리소스를 등록한다
    @mcp.resource(f"services://{name}")
    def _get_service() -> str:
        return json.dumps(_registry[name], indent=2)

    # 리소스 목록이 변경되었음을 알린다
    await mcp.notify_resource_list_changed()

    return f"서비스 등록 완료: {name}"


@mcp.tool()
async def unregister_service(name: str) -> str:
    """서비스 리소스를 제거한다

    Args:
        name: 제거할 서비스 이름
    """
    if name not in _registry:
        raise ValueError(f"등록되지 않은 서비스이다: {name}")

    del _registry[name]
    # 리소스 목록 변경 알림
    await mcp.notify_resource_list_changed()

    return f"서비스 제거 완료: {name}"

에러 처리

리소스를 찾을 수 없거나 접근 권한이 없는 경우에 대한 에러 처리 전략을 살펴본다.

리소스 없음 처리

import json

@mcp.resource("users://{user_id}/profile")
def get_user_profile(user_id: str) -> str:
    """사용자 프로필을 반환한다"""
    profiles = {
        "alice": {"name": "Alice", "role": "admin"},
        "bob": {"name": "Bob", "role": "user"},
    }

    profile = profiles.get(user_id)
    if profile is None:
        # 방법 1: 에러 정보를 JSON으로 반환
        return json.dumps({
            "error": "not_found",
            "message": f"사용자를 찾을 수 없다: {user_id}",
            "available_users": list(profiles.keys()),
        }, ensure_ascii=False)

    return json.dumps(profile, ensure_ascii=False, indent=2)


@mcp.resource("files://{path}")
def get_file_content(path: str) -> str:
    """파일 내용을 반환한다"""
    from pathlib import Path as P

    file_path = P(f"./data/{path}").resolve()

    if not file_path.exists():
        # 방법 2: 예외를 발생시킨다
        raise FileNotFoundError(f"파일을 찾을 수 없다: {path}")

    return file_path.read_text(encoding="utf-8")
TIP

리소스에서 예외를 발생시키면 MCP 프로토콜의 에러 응답으로 변환된다. 에러 JSON을 반환하는 방식은 클라이언트가 정상 응답으로 처리할 수 있어 더 유연하지만, 명확한 에러 상황에서는 예외를 발생시키는 것이 프로토콜 의미론에 맞다.

보안

Path Traversal 방지

파일 시스템에 접근하는 리소스는 ../를 이용한 경로 탐색 공격에 취약하다. resolve()로 경로를 정규화한 뒤, 허용된 디렉토리 내에 있는지 반드시 확인해야 한다. 

from pathlib import Path

BASE_DIR = Path("./workspace").resolve()

def _validate_resource_path(user_path: str) -> Path:
    """리소스 경로를 검증한다"""
    target = (BASE_DIR / user_path).resolve()
    if not str(target).startswith(str(BASE_DIR)):
        raise PermissionError("접근이 거부되었다: 허용 범위를 벗어난 경로이다")
    return target


@mcp.resource("workspace://{filepath}")
def get_workspace_file(filepath: str) -> str:
    """워크스페이스 파일을 반환한다"""
    safe_path = _validate_resource_path(filepath)

    if not safe_path.exists():
        raise FileNotFoundError(f"파일을 찾을 수 없다: {filepath}")
    if not safe_path.is_file():
        raise ValueError(f"디렉토리는 직접 읽을 수 없다: {filepath}")

    return safe_path.read_text(encoding="utf-8")

접근 제어 패턴

# 허용 확장자 목록
ALLOWED_EXTENSIONS = {".py", ".js", ".json", ".md", ".txt", ".yaml"}
# 금지 파일명 패턴
FORBIDDEN_PATTERNS = {".env", ".secret", "credentials", "password"}

@mcp.resource("code://{filepath}")
def get_code_file(filepath: str) -> str:
    """소스 코드 파일을 반환한다 (민감 파일 차단)"""
    safe_path = _validate_resource_path(filepath)

    # 확장자 확인
    if safe_path.suffix.lower() not in ALLOWED_EXTENSIONS:
        raise PermissionError(
            f"허용되지 않은 파일 형식이다: {safe_path.suffix}"
        )

    # 민감 파일 차단
    name_lower = safe_path.name.lower()
    for pattern in FORBIDDEN_PATTERNS:
        if pattern in name_lower:
            raise PermissionError(
                f"보안 정책에 의해 접근이 차단되었다: {safe_path.name}"
            )

    return safe_path.read_text(encoding="utf-8")
WARNING

리소스는 읽기 전용이지만, 민감한 데이터(환경 변수, 인증 정보, 개인 정보)를 노출하면 보안 사고로 이어질 수 있다. 파일 시스템 리소스에서는 반드시 경로 검증과 확장자/파일명 필터링을 적용하라.

실전 예제: 설정 관리 서버 (구독 포함)

환경별 설정을 리소스로 제공하고, 설정 변경 시 구독자에게 실시간으로 알리는 종합 서버를 구축한다. 

"""설정 관리 MCP 서버

환경별 설정을 리소스로 제공하고, 변경 시 구독자에게 실시간 알림을 보낸다.
"""
import json
import copy
from datetime import datetime
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Config Manager Pro")


# ── 설정 데이터 ─────────────────────────────────────
CONFIGS = {
    "development": {
        "env": "development",
        "debug": True,
        "database": {
            "host": "localhost",
            "port": 5432,
            "name": "myapp_dev",
            "pool_size": 5,

# ── 정적 리소스 ─────────────────────────────────────
@mcp.resource("config://environments", mime_type="application/json")
def list_environments() -> str:
    """사용 가능한 환경 목록을 반환한다"""
    envs = [
        {"name": name, "description": f"{name} 환경 설정"}
        for name in CONFIGS
    ]
    return json.dumps(envs, ensure_ascii=False, indent=2)


@mcp.resource("config://compare/all", mime_type="application/json")
def compare_all() -> str:
    """모든 환경의 주요 설정을 비교한다"""
    comparison = {}
    for env_name, config in CONFIGS.items():
        comparison[env_name] = {
            "debug": config["debug"],
            "db_host": config["database"]["host"],
            "db_pool": config["database"]["pool_size"],
            "cache_backend": config["cache"]["backend"],
            "features_enabled": sum(
                1 for v in config["features"].values() if v
            ),
        }
    return json.dumps(comparison, ensure_ascii=False, indent=2)


@mcp.resource("config://changelog", mime_type="application/json")
def get_changelog() -> str:
    """설정 변경 이력을 반환한다"""
    return json.dumps(_change_log[-50:], ensure_ascii=False, indent=2)


# ── 동적 리소스 (Resource Template) ─────────────────
@mcp.resource("config://{env}", mime_type="application/json")
def get_env_config(env: str) -> str:
    """특정 환경의 전체 설정을 반환한다"""
    config = CONFIGS.get(env)
    if config is None:
        return json.dumps(
            {"error": f"알 수 없는 환경: {env}",
             "available": list(CONFIGS.keys())},
            ensure_ascii=False,
        )
    return json.dumps(config, ensure_ascii=False, indent=2)


@mcp.resource("config://{env}/{section}", mime_type="application/json")
def get_config_section(env: str, section: str) -> str:
    """특정 환경의 특정 섹션 설정을 반환한다"""
    config = CONFIGS.get(env, {})
    value = config.get(section)
    if value is None:
        return json.dumps(
            {"error": f"'{env}' 환경에 '{section}' 섹션이 없다"},
            ensure_ascii=False,
        )
    return json.dumps(value, ensure_ascii=False, indent=2)


# ── 도구: 설정 업데이트 (구독 알림 포함) ────────────
@mcp.tool()
async def update_feature_flag(
    env: str, feature: str, enabled: bool
) -> str:
    """환경의 기능 플래그를 업데이트한다

    Args:
        env: 환경 이름 (development, staging, production)
        feature: 기능 이름
        enabled: 활성화 여부
    """
    config = CONFIGS.get(env)
    if config is None:
        raise ValueError(f"알 수 없는 환경: {env}")

    if feature not in config["features"]:
        raise ValueError(
            f"알 수 없는 기능: {feature}. "
            f"사용 가능: {list(config['features'].keys())}"
        )

    old_value = config["features"][feature]
    config["features"][feature] = enabled

    # 변경 이력 기록
    _change_log.append({
        "timestamp": datetime.now().isoformat(),
        "env": env,
        "field": f"features.{feature}",
        "old": old_value,
        "new": enabled,
    })

    # 구독자에게 변경 알림
    await mcp.notify_resource_updated(f"config://{env}")
    await mcp.notify_resource_updated(f"config://{env}/features")
    await mcp.notify_resource_updated("config://compare/all")
    await mcp.notify_resource_updated("config://changelog")

    return f"[{env}] {feature}: {old_value} -> {enabled}"


if __name__ == "__main__":
    mcp.run()

이 서버의 핵심 설계 포인트를 정리한다.

  1. 1
    정적 + 동적 리소스 조합 전체 환경 목록(정적)과 개별 환경 설정(동적 템플릿)을 함께 제공한다.
  2. 2
    MIME 타입 명시 JSON 데이터에 application/json을 명시하여 클라이언트가 정확히 파싱할 수 있도록 한다.
  3. 3
    구독 알림 발행 설정이 변경되면 관련된 모든 리소스 URI에 대해 변경 알림을 보낸다. 비교 뷰와 변경 이력 리소스에도 알림을 발행한다.
  4. 4
    Tool과 Resource 연계 Tool(update_feature_flag)로 데이터를 변경하고, Resource로 변경된 데이터를 조회하는 패턴이다.
  5. 5
    변경 이력 추적 모든 변경 사항을 _change_log에 기록하고, 이를 리소스로 제공한다. 감사(audit) 용도로 활용할 수 있다.

다음 단계

Resources의 정적/동적 패턴, URI 템플릿, MIME 타입, 구독 메커니즘, 보안까지 심층적으로 살펴보았다. 다음 챕터에서는 Prompts를 심도 있게 다루며, 다중 메시지 프롬프트, EmbeddedResource, 멀티모달 콘텐츠, 그리고 도구 연계 패턴을 배운다.
