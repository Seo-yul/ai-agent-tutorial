서버 컴포지션

여러 MCP 서버를 하나의 애플리케이션에 조합한다

서버 컴포지션이란?

MCP 서버가 커지면 하나의 서버에 모든 도구, 리소스, 프롬프트를 넣는 것은 유지보수가 어려워진다. 서버 컴포지션은 관심사별로 여러 MCP 서버를 분리하여 만들고, 하나의 ASGI 애플리케이션에서 조합하여 서빙하는 패턴이다.

이 패턴은 마이크로서비스 아키텍처와 유사하다. 각 서버는 독립적으로 개발하고 테스트할 수 있으며, 필요에 따라 자유롭게 조합할 수 있다.

  • 관심사 분리 — 각 서버가 하나의 도메인에 집중한다
  • 독립적 개발 — 서버별로 별도 팀이 개발할 수 있다
  • 유연한 배포 — 필요한 서버만 골라서 조합할 수 있다
  • 재사용성 — 동일한 서버를 여러 애플리케이션에서 재사용할 수 있다

기본 마운트

streamable_http_app() 메서드를 사용하면 각 FastMCP 서버를 Starlette의 ASGI 앱으로 변환할 수 있다. 이렇게 변환된 앱을 Mount로 서로 다른 경로에 마운트하면 하나의 애플리케이션에서 여러 MCP 서버를 동시에 서빙할 수 있다. 

import contextlib
from starlette.applications import Starlette
from starlette.routing import Mount
from mcp.server.fastmcp import FastMCP

echo_mcp = FastMCP(name="EchoServer", stateless_http=True, json_response=True)

@echo_mcp.tool()
def echo(message: str) -> str:
    """메시지를 그대로 반환한다"""
    return f"Echo: {message}"

math_mcp = FastMCP(name="MathServer", stateless_http=True, json_response=True)

@math_mcp.tool()
def add(a: int, b: int) -> int:
    """두 수를 더한다"""
    return a + b

@contextlib.asynccontextmanager
async def lifespan(app):
    async with contextlib.AsyncExitStack() as stack:
        await stack.enter_async_context(echo_mcp.session_manager.run())
        await stack.enter_async_context(math_mcp.session_manager.run())
        yield

app = Starlette(
    routes=[
        Mount("/echo", echo_mcp.streamable_http_app()),
        Mount("/math", math_mcp.streamable_http_app()),
    ],
    lifespan=lifespan,
)

위 코드는 두 개의 독립적인 MCP 서버(EchoServer, MathServer)를 하나의 Starlette 앱에 마운트한다. 클라이언트는 /echo/mcp와 /math/mcp로 각 서버에 접근할 수 있다.

INFO

stateless_http=True는 각 요청을 독립적으로 처리하는 Stateless 모드를 활성화한다. json_response=True는 스트리밍 대신 일반 JSON 응답을 반환하도록 설정한다. 서버 컴포지션에서는 이 두 옵션을 함께 사용하는 것이 일반적이다.

경로 설정

기본적으로 streamable_http_app()은 /mcp 경로에서 요청을 수신한다. 따라서 Mount("/echo", ...)로 마운트하면 실제 엔드포인트는 /echo/mcp가 된다.

streamable_http_path 설정을 변경하면 이 경로를 커스터마이징할 수 있다: 

# 기본 경로: /echo/mcp, /math/mcp
echo_mcp = FastMCP(name="EchoServer", stateless_http=True, json_response=True)
math_mcp = FastMCP(name="MathServer", stateless_http=True, json_response=True)

# streamable_http_path를 "/"로 설정하면 마운트 경로가 곧 엔드포인트가 된다
# 결과: /echo, /math
echo_mcp.settings.streamable_http_path = "/"
math_mcp.settings.streamable_http_path = "/"

app = Starlette(
    routes=[
        Mount("/echo", echo_mcp.streamable_http_app()),
        Mount("/math", math_mcp.streamable_http_app()),
    ],
    lifespan=lifespan,
)
설정 Echo 엔드포인트 Math 엔드포인트
기본값 (/mcp) /echo/mcp /math/mcp
streamable_http_path = "/" /echo /math
streamable_http_path = "/api" /echo/api /math/api
TIP

경로를 "/"로 설정하면 마운트 경로 자체가 엔드포인트가 되어 URL 구조가 깔끔해진다. 실제 배포 시에는 이 방식이 더 직관적이다.

Lifespan 관리

여러 MCP 서버를 하나의 앱에서 실행할 때는 각 서버의 라이프사이클을 올바르게 관리해야 한다. AsyncExitStack을 사용하면 모든 서버의 세션 매니저를 안전하게 시작하고 종료할 수 있다. 

import contextlib

@contextlib.asynccontextmanager
async def lifespan(app):
    """앱의 라이프사이클을 관리한다"""
    async with contextlib.AsyncExitStack() as stack:
        # 각 서버의 세션 매니저를 시작한다
        # AsyncExitStack은 등록된 역순으로 종료를 보장한다
        await stack.enter_async_context(echo_mcp.session_manager.run())
        await stack.enter_async_context(math_mcp.session_manager.run())
        await stack.enter_async_context(admin_mcp.session_manager.run())

        # yield 이전: 서버 시작 시 실행되는 코드
        print("모든 MCP 서버가 시작되었다")
        yield
        # yield 이후: 서버 종료 시 실행되는 코드
        print("모든 MCP 서버가 종료된다")
WARNING

lifespan 함수를 설정하지 않으면 세션 매니저가 초기화되지 않아 요청 처리 시 오류가 발생한다. 서버 컴포지션을 사용할 때는 반드시 모든 서버의 session_manager.run()을 lifespan에 등록해야 한다.

AsyncExitStack은 등록된 컨텍스트 매니저를 역순으로 종료한다. 따라서 의존성이 있는 서버들은 의존되는 서버를 먼저 등록하고, 의존하는 서버를 나중에 등록해야 한다.

실전 예제: 마이크로서비스 MCP

API 서버, Chat 서버, Admin 서버를 분리하여 하나의 애플리케이션에 마운트하는 완전한 예제이다. 각 서버는 독립적인 도메인을 담당한다. 

import contextlib
from datetime import datetime
from starlette.applications import Starlette
from starlette.routing import Mount
from mcp.server.fastmcp import FastMCP


# ============================================================
# API 서버: 데이터 조회 관련 도구
# ============================================================

api_mcp = FastMCP(name="APIServer", stateless_http=True, json_response=True)
api_mcp.settings.streamable_http_path = "/"


@api_mcp.tool()
def get_users() -> list[dict]:
    """등록된 사용자 목록을 조회한다"""
    return [
        {"id": 1, "name": "홍길동", "role": "admin"},
        {"id": 2, "name": "김철수", "role": "user"},
        {"id": 3, "name": "이영희", "role": "user"},
    ]


@api_mcp.tool()
def get_products(category: str = "all") -> list[dict]:
    """상품 목록을 조회한다"""
    products = [
        {"id": 101, "name": "노트북", "category": "electronics", "price": 1500000},
        {"id": 102, "name": "마우스", "category": "electronics", "price": 35000},
        {"id": 103, "name": "프로그래밍 책", "category": "books", "price": 32000},
    ]
    if category != "all":
        products = [p for p in products if p["category"] == category]
    return products


# ============================================================
# Chat 서버: 대화 관련 도구
# ============================================================

chat_mcp = FastMCP(name="ChatServer", stateless_http=True, json_response=True)
chat_mcp.settings.streamable_http_path = "/"


@chat_mcp.tool()
def send_message(channel: str, message: str) -> dict:
    """채널에 메시지를 전송한다"""
    return {
        "status": "sent",
        "channel": channel,
        "message": message,
        "timestamp": datetime.now().isoformat(),
    }


@chat_mcp.tool()
def get_channels() -> list[str]:
    """사용 가능한 채널 목록을 반환한다"""
    return ["general", "random", "dev", "design", "marketing"]


@chat_mcp.tool()
def get_recent_messages(channel: str, limit: int = 5) -> list[dict]:
    """채널의 최근 메시지를 조회한다"""
    return [
        {"user": "홍길동", "message": f"{channel} 채널 테스트 메시지 {i}", "time": "10:00"}
        for i in range(1, limit + 1)
    ]


# ============================================================
# Admin 서버: 관리 도구
# ============================================================

admin_mcp = FastMCP(name="AdminServer", stateless_http=True, json_response=True)
admin_mcp.settings.streamable_http_path = "/"


@admin_mcp.tool()
def get_server_status() -> dict:
    """서버 상태를 조회한다"""
    return {
        "status": "healthy",
        "uptime": "72h 15m",
        "cpu_usage": "23%",
        "memory_usage": "45%",
        "active_connections": 127,
    }


@admin_mcp.tool()
def get_logs(level: str = "info", limit: int = 10) -> list[dict]:
    """서버 로그를 조회한다"""
    return [
        {
            "level": level,
            "message": f"샘플 {level} 로그 메시지 #{i}",
            "timestamp": datetime.now().isoformat(),
        }
        for i in range(1, limit + 1)
    ]


# ============================================================
# 앱 조합
# ============================================================

@contextlib.asynccontextmanager
async def lifespan(app):
    """모든 MCP 서버의 라이프사이클을 관리한다"""
    async with contextlib.AsyncExitStack() as stack:
        await stack.enter_async_context(api_mcp.session_manager.run())
        await stack.enter_async_context(chat_mcp.session_manager.run())
        await stack.enter_async_context(admin_mcp.session_manager.run())
        yield

app = Starlette(
    routes=[
        Mount("/api", api_mcp.streamable_http_app()),
        Mount("/chat", chat_mcp.streamable_http_app()),
        Mount("/admin", admin_mcp.streamable_http_app()),
    ],
    lifespan=lifespan,
)

서버 실행

# uvicorn으로 실행
uvicorn app:app --host 0.0.0.0 --port 8000

# 또는 개발 모드 (자동 리로드)
uvicorn app:app --host 0.0.0.0 --port 8000 --reload

실행 후 각 서버에 접근할 수 있는 엔드포인트는 다음과 같다:

# API 서버 (사용자, 상품 조회)
http://localhost:8000/api

# Chat 서버 (메시지 전송, 채널 관리)
http://localhost:8000/chat

# Admin 서버 (서버 상태, 로그 조회)
http://localhost:8000/admin
TIP

각 MCP 서버는 독립적이므로, 개별 서버를 별도의 파이썬 모듈로 분리하여 관리할 수 있다. 예를 들어 servers/api.py, servers/chat.py, servers/admin.py로 분리한 후 메인 앱에서 import하여 마운트하면 코드 구조가 훨씬 깔끔해진다.

클라이언트에서 접속

각 서버에 별도로 연결하여 도구를 사용할 수 있다:

from mcp.client.streamable_http import streamable_http_client
from mcp.client.session import ClientSession

async def main():
    # API 서버에 연결
    async with streamable_http_client("http://localhost:8000/api") as (r, w, _):
        async with ClientSession(r, w) as session:
            await session.initialize()

            # 사용자 목록 조회
            result = await session.call_tool("get_users", {})
            print(result)

    # Chat 서버에 연결
    async with streamable_http_client("http://localhost:8000/chat") as (r, w, _):
        async with ClientSession(r, w) as session:
            await session.initialize()

            # 채널 목록 조회
            result = await session.call_tool("get_channels", {})
            print(result)
