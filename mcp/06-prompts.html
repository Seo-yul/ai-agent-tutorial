Prompts 심화

프롬프트 템플릿, 다중 메시지, 임베디드 리소스, 그리고 멀티모달 콘텐츠

Prompts 개요

핵심 개념: Prompt란 무엇인가

MCP에서 Prompt는 서버가 미리 정의해 둔 프롬프트 템플릿이다. 클라이언트(앱)는 사용 가능한 프롬프트 목록을 조회하고, 파라미터를 전달하여 완성된 메시지를 받는다. 이 메시지를 LLM에게 전달하면, 일관된 형식으로 작업을 요청할 수 있다. 코드 리뷰, 버그 분석, 문서 생성 등 반복적인 작업 패턴을 표준화하는 데 유용하다.

프로토콜 수준 흐름

sequenceDiagram
    participant U as 사용자
    participant C as MCP Client
    participant S as MCP Server
    participant L as LLM

    C->>S: prompts/list
    S-->>C: Prompt[] (name, description, arguments)
    Note over U,C: 사용자가 프롬프트 선택 + 인자 입력
    C->>S: prompts/get {name, arguments}
    S-->>C: GetPromptResult {messages[]}
    C->>L: messages를 LLM에 전달
    L-->>C: LLM 응답
← prompts/list 응답 예시
{
  "prompts": [
    {
      "name": "code_review",
      "description": "코드 리뷰 프롬프트",
      "arguments": [
        {
          "name": "code",
          "description": "리뷰할 코드",
          "required": true
        },
        {
          "name": "language",
          "description": "프로그래밍 언어",
          "required": false
        }
      ]
    }
  ]
}
← prompts/get 응답 예시
{
  "messages": [
    {
      "role": "user",
      "content": {
        "type": "text",
        "text": "다음 Python 코드를 리뷰해주세요..."
      }
    }
  ]
}

기본 프롬프트

가장 간단한 프롬프트는 문자열을 반환하는 함수이다. @mcp.prompt() 데코레이터를 사용하며, 반환된 문자열은 자동으로 UserMessage로 감싸진다. 

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("prompt-server")

@mcp.prompt()
def summarize(text: str, language: str = "ko") -> str:
    """텍스트 요약 프롬프트"""
    return f"다음 텍스트를 {language} 언어로 3줄 이내로 요약하라:\n\n{text}"


@mcp.prompt()
def translate(text: str, source: str = "ko", target: str = "en") -> str:
    """텍스트 번역 프롬프트"""
    return (
        f"다음 {source} 텍스트를 {target}로 번역하라.\n"
        f"원문의 뉘앙스를 최대한 살려 자연스럽게 번역하라.\n\n"
        f"원문:\n{text}"
    )

함수의 파라미터가 프롬프트의 인자(argument)가 된다. 기본값이 있는 파라미터는 선택적 인자이고, 없는 파라미터는 필수 인자이다. 함수의 docstring은 프롬프트 설명으로 사용된다.

함수 정의
def summarize(text, language)
→
인자 추출
text: 필수
language: 선택
→
문자열 반환
→
UserMessage로 변환

다중 메시지 프롬프트

복잡한 프롬프트는 여러 메시지로 구성된 대화 형태로 정의할 수 있다. UserMessage와 AssistantMessage를 조합하여 few-shot 패턴을 구현하거나, 시스템 지시와 사용자 입력을 논리적으로 분리할 수 있다. 

from mcp.server.fastmcp.prompts import base

@mcp.prompt()
def code_review(code: str, language: str = "python") -> list[base.Message]:
    """코드 리뷰 프롬프트"""
    return [
        base.UserMessage(
            f"당신은 시니어 {language} 개발자이다. "
            f"다음 코드를 보안, 성능, 가독성 관점에서 리뷰하라.\n"
            f"결과는 심각도별로 분류하라."
        ),
        base.UserMessage(f"```{language}\n{code}\n```"),
    ]

few-shot 패턴

UserMessage와 AssistantMessage를 교차 배치하면, LLM에게 원하는 응답 형식을 예시로 보여줄 수 있다. 

@mcp.prompt()
def extract_entities(text: str) -> list[base.Message]:
    """텍스트에서 엔티티를 추출하는 프롬프트 (few-shot)"""
    return [
        # 지시 메시지
        base.UserMessage(
            "텍스트에서 인물, 장소, 날짜 엔티티를 추출하여 JSON으로 반환하라."
        ),
        # 예시 1: 사용자 입력
        base.UserMessage("김철수는 2024년 3월 서울에서 열린 컨퍼런스에 참석했다."),
        # 예시 1: 기대하는 응답
        base.AssistantMessage(
            '{"persons": ["김철수"], "locations": ["서울"], '
            '"dates": ["2024년 3월"]}'
        ),
        # 예시 2: 사용자 입력
        base.UserMessage(
            "이영희와 박지민은 부산 해운대에서 5월 1일에 만났다."
        ),
        # 예시 2: 기대하는 응답
        base.AssistantMessage(
            '{"persons": ["이영희", "박지민"], '
            '"locations": ["부산", "해운대"], '
            '"dates": ["5월 1일"]}'
        ),
        # 실제 입력
        base.UserMessage(text),
    ]
TIP

few-shot 패턴은 LLM에게 응답 형식과 품질의 기준을 명시적으로 보여준다. JSON 형식, 분류 체계, 코드 스타일 등 일관된 출력이 필요한 경우에 효과적이다.

파라미터화된 프롬프트

선택적 파라미터를 활용하면 하나의 프롬프트로 다양한 변형을 만들 수 있다. 조건부 메시지를 포함시켜 파라미터 유무에 따라 프롬프트 구조를 동적으로 변경한다. 

@mcp.prompt()
def debug_analysis(
    error_message: str,
    code: str = "",
    stack_trace: str = "",
    environment: str = "",
) -> list[base.Message]:
    """버그 분석 프롬프트 (조건부 메시지 포함)"""
    messages = [
        base.UserMessage(
            "당신은 디버깅 전문가이다. 다음 오류를 체계적으로 분석하라."
        ),
        base.UserMessage(f"## 에러 메시지\n```\n{error_message}\n```"),
    ]

    # 선택적 파라미터: 코드가 제공된 경우에만 포함
    if code:
        messages.append(
            base.UserMessage(f"## 관련 코드\n```python\n{code}\n```")
        )

    # 선택적 파라미터: 스택 트레이스가 있으면 포함
    if stack_trace:
        messages.append(
            base.UserMessage(f"## 스택 트레이스\n```\n{stack_trace}\n```")
        )

    # 선택적 파라미터: 실행 환경 정보
    if environment:
        messages.append(
            base.UserMessage(f"## 실행 환경\n{environment}")
        )

    # 분석 요청 (항상 포함)
    messages.append(
        base.UserMessage(
            "다음 항목을 포함하여 분석하라:\n"
            "1. 근본 원인\n"
            "2. 영향 범위\n"
            "3. 수정 코드\n"
            "4. 테스트 케이스\n"
            "5. 재발 방지 조치"
        )
    )

    return messages

EmbeddedResource

Deep Dive: 프롬프트에 리소스를 임베드하기

EmbeddedResource는 프롬프트 메시지 안에 MCP 리소스의 URI를 포함시키는 기능이다. 프롬프트가 실행될 때 해당 URI의 리소스 데이터가 자동으로 로드되어 메시지에 포함된다. 이를 통해 프롬프트와 리소스를 느슨하게 결합하면서, LLM에게 풍부한 문맥을 제공할 수 있다. 

from mcp.server.fastmcp import FastMCP
from mcp.server.fastmcp.prompts import base
from mcp.types import EmbeddedResource, TextResourceContents

mcp = FastMCP("embedded-resource-demo")

# 리소스 정의
@mcp.resource("docs://api-guide")
def get_api_guide() -> str:
    """API 가이드 문서"""
    return """# API 가이드
## 인증
모든 요청에 Bearer 토큰을 포함해야 한다.

## 엔드포인트
- GET /users - 사용자 목록
- POST /users - 사용자 생성
- GET /users/{id} - 사용자 조회
"""

@mcp.resource("docs://style-guide")
def get_style_guide() -> str:
    """코드 스타일 가이드"""
    return """# 코드 스타일 가이드
- 함수명: snake_case
- 클래스명: PascalCase
- 상수: UPPER_SNAKE_CASE
- 줄 길이: 최대 100자
"""


# EmbeddedResource를 사용하는 프롬프트
@mcp.prompt()
def review_with_guidelines(code: str) -> list[base.Message]:
    """코드 스타일 가이드를 참조하여 코드를 리뷰하는 프롬프트"""
    return [
        base.UserMessage(
            "다음 코드 스타일 가이드를 참고하여 코드를 리뷰하라."
        ),
        # EmbeddedResource: 리소스 URI를 메시지에 임베드한다
        base.UserMessage(
            EmbeddedResource(
                type="resource",
                resource=TextResourceContents(
                    uri="docs://style-guide",
                    text=get_style_guide(),  # 리소스 데이터를 직접 포함
                    mimeType="text/plain",
                ),
            )
        ),
        base.UserMessage(f"## 리뷰 대상 코드\n```python\n{code}\n```"),
        base.UserMessage(
            "스타일 가이드 위반 사항과 개선점을 항목별로 정리하라."
        ),
    ]
INFO

EmbeddedResource를 사용하면 리소스의 데이터가 프롬프트 메시지의 일부로 전달된다. 클라이언트가 별도로 resources/read를 호출할 필요 없이, 프롬프트 응답 안에 데이터가 포함되므로 LLM이 즉시 참조할 수 있다.

이미지를 포함한 프롬프트

Deep Dive: 멀티모달 프롬프트

MCP 프롬프트는 텍스트뿐만 아니라 이미지도 포함할 수 있다. ImageContent를 사용하여 base64 인코딩된 이미지를 메시지에 포함시키면, 멀티모달 LLM(Claude, GPT-4V 등)이 이미지를 분석하는 프롬프트를 만들 수 있다. 

import base64
from pathlib import Path
from mcp.server.fastmcp.prompts import base
from mcp.types import ImageContent, TextContent

@mcp.prompt()
def analyze_screenshot(
    image_path: str,
    analysis_type: str = "ui",
) -> list[base.Message]:
    """스크린샷을 분석하는 멀티모달 프롬프트

    Args:
        image_path: 분석할 이미지 파일 경로
        analysis_type: 분석 유형 (ui, accessibility, performance)
    """
    # 이미지 파일 읽기 및 base64 인코딩
    img_path = Path(image_path)
    if not img_path.exists():
        return [base.UserMessage(f"이미지를 찾을 수 없다: {image_path}")]

    img_data = base64.b64encode(img_path.read_bytes()).decode("utf-8")

    # 확장자에서 MIME 타입 추론
    mime_map = {".png": "image/png", ".jpg": "image/jpeg", ".gif": "image/gif"}
    mime_type = mime_map.get(img_path.suffix.lower(), "image/png")

    # 분석 유형별 지시
    instructions = {
        "ui": "UI/UX 관점에서 레이아웃, 색상, 타이포그래피, 사용성을 분석하라.",
        "accessibility": "접근성(a11y) 관점에서 대비, 글꼴 크기, 네비게이션을 분석하라.",
        "performance": "성능 관점에서 이미지 최적화, 렌더링 블로킹 요소를 분석하라.",
    }

    instruction = instructions.get(analysis_type, instructions["ui"])

    return [
        base.UserMessage(f"다음 스크린샷을 분석하라. 분석 관점: {instruction}"),
        # ImageContent를 직접 UserMessage에 포함한다
        base.UserMessage(
            ImageContent(
                type="image",
                data=img_data,
                mimeType=mime_type,
            )
        ),
        base.UserMessage(
            "분석 결과를 다음 형식으로 정리하라:\n"
            "1. 긍정적 요소\n"
            "2. 개선 필요 사항\n"
            "3. 구체적인 개선 제안"
        ),
    ]
INFO

이미지 콘텐츠는 base64 인코딩된 문자열로 전송된다. 큰 이미지는 전송 크기가 커지므로, 필요한 경우 리사이즈하여 사용하는 것이 좋다. 멀티모달을 지원하지 않는 LLM에서는 이미지 콘텐츠를 무시하거나 에러를 반환할 수 있다.

프롬프트 인자 스키마

Deep Dive: PromptArgument 커스터마이징

FastMCP는 함수의 파라미터를 분석하여 PromptArgument를 자동 생성한다. 하지만 @mcp.prompt() 데코레이터에서 인자 스키마를 수동으로 지정하면 required와 description을 더 세밀하게 제어할 수 있다.

기본적으로 FastMCP는 다음 규칙으로 인자를 생성한다.

함수 파라미터결과
code: str (기본값 없음)required: true
language: str = "python"required: false
docstring의 Args 항목description에 반영
 
@mcp.prompt()
def generate_tests(
    code: str,
    framework: str = "pytest",
    coverage: str = "comprehensive",
) -> list[base.Message]:
    """테스트 코드 생성 프롬프트

    Args:
        code: 테스트를 작성할 대상 코드
        framework: 테스트 프레임워크 (pytest 또는 unittest)
        coverage: 테스트 범위 (basic, comprehensive, edge_cases)
    """
    coverage_map = {
        "basic": "핵심 기능의 기본 동작만 테스트하라.",
        "comprehensive": "정상, 에러, 경계값을 모두 테스트하라.",
        "edge_cases": "엣지 케이스와 예외 상황에 집중하라.",
    }

    return [
        base.UserMessage(
            f"다음 코드에 대한 테스트를 {framework}로 작성하라.\n\n"
            f"테스트 범위: {coverage_map.get(coverage, coverage_map['comprehensive'])}\n\n"
            f"```python\n{code}\n```"
        ),
    ]

위 프롬프트는 다음과 같은 인자 스키마를 생성한다.

← prompts/list에서 generate_tests의 arguments
{
  "arguments": [
    {
      "name": "code",
      "description": "테스트를 작성할 대상 코드",
      "required": true
    },
    {
      "name": "framework",
      "description": "테스트 프레임워크 (pytest 또는 unittest)",
      "required": false
    },
    {
      "name": "coverage",
      "description": "테스트 범위 (basic, comprehensive, edge_cases)",
      "required": false
    }
  ]
}
TIP

docstring의 Args 섹션에 각 파라미터의 설명을 작성하면, 프롬프트 인자의 description에 자동 반영된다. 사용자(또는 클라이언트 UI)가 각 인자의 용도를 쉽게 파악할 수 있도록 구체적으로 작성하라.

프롬프트 + 도구 연계

Deep Dive: 프롬프트에서 도구 사용을 유도하는 패턴

프롬프트 메시지 안에 "이 서버의 도구를 사용하라"는 지시를 포함시키면, LLM이 프롬프트를 받은 뒤 자연스럽게 서버의 도구를 호출하는 워크플로를 만들 수 있다. 이를 통해 "데이터 조회 -> 분석 -> 액션" 패턴을 하나의 프롬프트로 시작할 수 있다. 

from mcp.server.fastmcp import FastMCP
from mcp.server.fastmcp.prompts import base

mcp = FastMCP("workflow-demo")


# 도구 정의
@mcp.tool()
def search_codebase(query: str, file_pattern: str = "*.py") -> list[dict]:
    """코드베이스에서 패턴을 검색한다

    Args:
        query: 검색 키워드
        file_pattern: 파일 패턴
    """
    # 검색 구현
    return [{"file": "app.py", "line": 42, "content": f"match: {query}"}]


@mcp.tool()
def run_tests(test_path: str = "tests/") -> str:
    """테스트를 실행한다

    Args:
        test_path: 테스트 디렉토리
    """
    return "All tests passed (15/15)"


@mcp.tool()
def create_issue(title: str, body: str, labels: list[str] = []) -> dict:
    """이슈를 생성한다

    Args:
        title: 이슈 제목
        body: 이슈 본문
        labels: 레이블 목록
    """
    return {"id": 42, "url": "https://github.com/org/repo/issues/42"}


# 프롬프트: 도구 사용을 유도한다
@mcp.prompt()
def investigate_bug(description: str) -> list[base.Message]:
    """버그를 조사하고 이슈를 생성하는 워크플로 프롬프트

Deep Dive: 동적 프롬프트 등록과 알림

서버가 런타임에 프롬프트를 추가하거나 제거하면, notifications/prompts/list_changed 알림을 보내야 한다. 클라이언트는 이 알림을 받으면 prompts/list를 다시 호출하여 최신 프롬프트 목록을 갱신한다.

from mcp.server.fastmcp import FastMCP
from mcp.server.fastmcp.prompts import base

mcp = FastMCP("dynamic-prompts")

# 프롬프트 레지스트리
_prompt_templates: dict[str, dict] = {}


@mcp.tool()
async def register_prompt_template(
    name: str,
    template: str,
    description: str = "",
) -> str:
    """새 프롬프트 템플릿을 동적으로 등록한다

    Args:
        name: 프롬프트 이름
        template: 프롬프트 템플릿 문자열 ({input} 자리표시자 사용)
        description: 프롬프트 설명
    """
    _prompt_templates[name] = {
        "template": template,
        "description": description,
    }

    # 동적으로 프롬프트 등록
    @mcp.prompt(name=name, description=description)
    def _dynamic_prompt(input: str) -> str:
        return template.replace("{input}", input)

    # 프롬프트 목록 변경 알림
    await mcp.notify_prompts_list_changed()

    return f"프롬프트 등록 완료: {name}"


@mcp.tool()
async def remove_prompt_template(name: str) -> str:
    """프롬프트 템플릿을 제거한다

    Args:
        name: 제거할 프롬프트 이름
    """
    if name not in _prompt_templates:
        raise ValueError(f"등록되지 않은 프롬프트이다: {name}")

    del _prompt_templates[name]

    # 프롬프트 목록 변경 알림
    await mcp.notify_prompts_list_changed()

    return f"프롬프트 제거 완료: {name}"

실전 예제: 개발 도우미 프롬프트 서버

코드 리뷰, 버그 분석, 문서 생성, 테스트 생성 프롬프트를 종합적으로 제공하는 개발 도우미 서버를 구축한다. EmbeddedResource, 도구 연계, 다중 메시지 패턴을 모두 활용한다. 

"""개발 도우미 프롬프트 MCP 서버

코드 리뷰, 버그 분석, 문서 생성, 테스트 생성 프롬프트를 제공한다.
도구 연계와 EmbeddedResource 패턴을 활용한다.
"""
import json
from mcp.server.fastmcp import FastMCP
from mcp.server.fastmcp.prompts import base
from mcp.types import EmbeddedResource, TextResourceContents

mcp = FastMCP("Dev Assistant")


# ── 리소스: 코딩 규칙 ──────────────────────────────
@mcp.resource("guide://coding-standards", mime_type="text/plain")
def get_coding_standards() -> str:
    """프로젝트 코딩 표준"""
    return """# 코딩 표준
- 함수 최대 줄 수: 30줄
- 파라미터 최대 개수: 5개
- 반드시 타입 힌트를 사용한다
- 모든 공개 함수에 docstring을 작성한다
- 매직 넘버 사용 금지 (상수로 정의)
"""


@mcp.resource("guide://commit-convention", mime_type="text/plain")
def get_commit_convention() -> str:
    """커밋 메시지 규칙"""
    return """# 커밋 메시지 컨벤션
형식: (): 
타입: feat, fix, docs, style, refactor, test, chore
scope: 선택 사항 (모듈 또는 파일명)
description: 현재 시제, 50자 이내
"""


# ── 도구: 코드 분석 ────────────────────────────────
@mcp.tool()
def analyze_complexity(code: str) -> dict:
    """코드의 복잡도를 분석한다

    Args:
        code: 분석할 Python 코드
    """
    lines = code.strip().split("\n")
    functions = sum(1 for l in lines if l.strip().startswith("def "))
    classes = sum(1 for l in lines if l.strip().startswith("class "))
    return {
        "total_lines": len(lines),
        "functions": functions,
        "classes": classes,
        "avg_lines_per_function": len(lines) // max(functions, 1),
    }


# ── 프롬프트 1: 코드 리뷰 ─────────────────────────
@mcp.prompt()
def review_code(
    code: str,
    language: str = "python",
    focus: str = "all",
) -> list[base.Message]:
    """코드 리뷰 프롬프트 (코딩 표준 참조 + 도구 연계)

    Args:
        code: 리뷰할 코드
        language: 프로그래밍 언어
        focus: 리뷰 초점 (all, security, performance, style)
    """
    focus_map = {
        "all": "보안, 성능, 코드 스타일, 가독성을 종합적으로",
        "security": "보안 취약점과 인젝션 공격 가능성을 중심으로",
        "performance": "성능 병목, 메모리 누수, 알고리즘 효율성을 중심으로",
        "style": "코드 컨벤션, 네이밍, 구조 설계를 중심으로",
    }

    return [
        base.UserMessage(
            f"당신은 시니어 {language} 개발자이다. "
            f"다음 코드를 {focus_map.get(focus, focus_map['all'])} 리뷰하라."
        ),
        # 코딩 표준을 EmbeddedResource로 참조
        base.UserMessage(
            EmbeddedResource(
                type="resource",
                resource=TextResourceContents(
                    uri="guide://coding-standards",
                    text=get_coding_standards(),
                    mimeType="text/plain",
                ),
            )
        ),
        base.UserMessage(f"```{language}\n{code}\n```"),
        base.UserMessage(
            "리뷰 결과를 다음 형식으로 정리하라:\n"
            "- [심각] 즉시 수정 필요\n"
            "- [주의] 개선 권장\n"
            "- [제안] 더 나은 방법\n\n"
            "코드 복잡도를 확인하려면 `analyze_complexity` 도구를 사용하라."
        ),
    ]


# ── 프롬프트 2: 버그 분석 ─────────────────────────
@mcp.prompt()
def analyze_bug(
    description: str,
    code: str = "",
    error_log: str = "",
) -> list[base.Message]:
    """버그 분석 프롬프트 (도구 연계)

    Args:
        description: 버그 설명
        code: 관련 코드 (선택)
        error_log: 에러 로그 (선택)
    """
    messages = [
        base.UserMessage(
            "당신은 디버깅 전문가이다. 다음 버그를 체계적으로 분석하라."
        ),
        base.UserMessage(f"## 버그 설명\n{description}"),
    ]

    if code:
        messages.append(
            base.UserMessage(f"## 관련 코드\n```python\n{code}\n```")
        )

    if error_log:
        messages.append(
            base.UserMessage(f"## 에러 로그\n```\n{error_log}\n```")
        )

    messages.append(
        base.UserMessage(
            "다음 항목을 포함하여 분석하라:\n"
            "1. 근본 원인\n"
            "2. 영향 범위\n"
            "3. 수정 코드\n"
            "4. 테스트 케이스\n"
            "5. 재발 방지 조치"
        )
    )

    return messages


# ── 프롬프트 3: 커밋 메시지 생성 ──────────────────
@mcp.prompt()
def commit_message(
    diff: str,
    convention: str = "conventional",
) -> list[base.Message]:
    """커밋 메시지 생성 프롬프트 (컨벤션 가이드 참조)

    Args:
        diff: Git diff 내용
        convention: 컨벤션 유형 (conventional, simple)
    """
    return [
        base.UserMessage("다음 커밋 컨벤션을 참고하여 커밋 메시지를 작성하라."),
        base.UserMessage(
            EmbeddedResource(
                type="resource",
                resource=TextResourceContents(
                    uri="guide://commit-convention",
                    text=get_commit_convention(),
                    mimeType="text/plain",
                ),
            )
        ),
        base.UserMessage(f"```diff\n{diff}\n```"),
        base.UserMessage("커밋 메시지만 출력하라. 설명은 불필요하다."),
    ]


# ── 프롬프트 4: API 문서 생성 ─────────────────────
@mcp.prompt()
def generate_api_docs(
    code: str,
    format: str = "markdown",
) -> list[base.Message]:
    """API 문서 생성 프롬프트

    Args:
        code: 문서화할 코드
        format: 출력 형식 (markdown, docstring)
    """
    format_instructions = {
        "markdown": "Markdown 형식으로 API 문서를 작성하라.",
        "docstring": "Google 스타일 docstring을 각 함수/클래스에 추가하라.",
    }

    return [
        base.UserMessage(
            f"다음 코드에 대한 API 문서를 작성하라.\n\n"
            f"출력 형식: {format_instructions.get(format, format_instructions['markdown'])}\n\n"
            f"각 함수/클래스에 대해 다음을 포함하라:\n"
            f"- 설명, 파라미터, 반환값, 예외, 사용 예제"
        ),
        base.UserMessage(f"```python\n{code}\n```"),
    ]


if __name__ == "__main__":
    mcp.run()

이 서버의 핵심 설계 포인트를 정리한다.

  1. 1
    EmbeddedResource 활용 코딩 표준과 커밋 컨벤션을 리소스로 정의하고, 프롬프트에서 임베드하여 LLM이 규칙을 참조하도록 한다.
  2. 2
    도구 연계 패턴 코드 리뷰 프롬프트에서 analyze_complexity 도구 사용을 유도하여, 프롬프트 -> 도구 호출의 자연스러운 워크플로를 구성한다.
  3. 3
    조건부 메시지 버그 분석 프롬프트에서 선택적 파라미터(code, error_log)가 제공된 경우에만 해당 메시지를 포함시킨다.
  4. 4
    다중 메시지 구조 지시, 참조 자료, 입력, 출력 형식 요청을 별도 메시지로 분리하여 논리적 구조를 명확히 한다.
  5. 5
    파라미터 설명 docstring의 Args 섹션을 상세히 작성하여, 클라이언트 UI에서 각 인자의 용도와 선택지를 쉽게 파악할 수 있도록 한다.
