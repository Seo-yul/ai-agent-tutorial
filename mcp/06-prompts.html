Chapter 6. Prompts 심화

재사용 가능한 프롬프트 템플릿을 정의하고 활용한다

6.1 Prompts 개요

MCP의 Prompts는 미리 정의된 프롬프트 템플릿이다. 사용자나 LLM이 일관된 방식으로 특정 작업을 요청할 수 있도록, 서버 측에서 프롬프트의 구조와 내용을 미리 작성해 둔다.

Prompts를 사용하면 다음과 같은 이점이 있다.

  • 일관성 — 매번 같은 형식의 프롬프트를 사용하여 결과의 품질을 유지한다
  • 재사용성 — 한 번 정의하면 여러 클라이언트에서 동일하게 사용할 수 있다
  • 파라미터화 — 입력값을 받아 동적으로 프롬프트를 생성한다
  • 발견 가능성 — 클라이언트가 사용 가능한 프롬프트 목록을 자동으로 조회할 수 있다
sequenceDiagram
    participant C as 클라이언트
    participant S as MCP 서버
    participant L as LLM

    C->>S: 1. 프롬프트 목록 요청 (list_prompts)
    S-->>C: 프롬프트 목록 반환
    Note over C: 2. 사용자가 프롬프트 선택 + 파라미터 입력
    C->>S: get_prompt("code_review", {"code": "...", "lang": "py"})
    S-->>C: 3. 완성된 프롬프트 메시지 반환 [UserMessage, ...]
    C->>L: 4. 해당 메시지를 LLM에 전달
    L-->>C: LLM 처리 결과

6.2 기본 프롬프트

가장 간단한 프롬프트는 문자열을 반환하는 함수로 정의한다. @mcp.prompt() 데코레이터를 사용하며, 함수의 파라미터가 프롬프트의 인자가 된다.

텍스트 요약 프롬프트

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("prompt-server")

@mcp.prompt()
def summarize(text: str, language: str = "ko") -> str:
    """텍스트 요약 프롬프트"""
    return f"다음 텍스트를 {language} 언어로 요약해주세요:\n\n{text}"

summarize라는 이름의 프롬프트가 등록된다. 필수 파라미터 text와 선택 파라미터 language(기본값: "ko")를 받는다.

번역 프롬프트

@mcp.prompt()
def translate(text: str, source_lang: str = "ko", target_lang: str = "en") -> str:
    """텍스트 번역 프롬프트"""
    return (
        f"다음 {source_lang} 텍스트를 {target_lang}로 번역해주세요.\n"
        f"원문의 뉘앙스와 문맥을 최대한 살려서 자연스럽게 번역해주세요.\n\n"
        f"원문:\n{text}"
    )
TIP

프롬프트 함수의 docstring은 클라이언트에서 프롬프트 설명으로 표시된다. 프롬프트의 용도를 명확히 작성하면 사용자가 올바른 프롬프트를 선택하는 데 도움이 된다.

6.3 다중 메시지 프롬프트

더 복잡한 프롬프트는 여러 메시지로 구성된 대화 형태로 정의할 수 있다. mcp.server.fastmcp.prompts.base 모듈의 Message 클래스를 사용한다.

코드 리뷰 프롬프트

from mcp.server.fastmcp.prompts import base

@mcp.prompt()
def code_review(code: str, language: str = "python") -> list[base.Message]:
    """코드 리뷰 프롬프트"""
    return [
        base.UserMessage(
            f"다음 {language} 코드를 리뷰해주세요. "
            f"보안 취약점, 성능 문제, 코드 스타일을 확인해주세요."
        ),
        base.UserMessage(code),
    ]

다중 메시지 프롬프트는 base.Message 객체의 리스트를 반환한다. 사용 가능한 메시지 타입은 다음과 같다.

  • base.UserMessage(content) — 사용자 역할의 메시지
  • base.AssistantMessage(content) — 어시스턴트 역할의 메시지

디버깅 분석 프롬프트

@mcp.prompt()
def debug_analysis(
    error_message: str,
    code: str,
    stack_trace: str = "",
) -> list[base.Message]:
    """버그 분석 프롬프트"""
    messages = [
        base.UserMessage(
            "당신은 숙련된 Python 디버깅 전문가이다. "
            "다음 오류를 분석하고 해결책을 제시해주세요."
        ),
        base.UserMessage(f"## 에러 메시지\n```\n{error_message}\n```"),
        base.UserMessage(f"## 관련 코드\n```python\n{code}\n```"),
    ]

    if stack_trace:
        messages.append(
            base.UserMessage(f"## 스택 트레이스\n```\n{stack_trace}\n```")
        )

    messages.append(
        base.UserMessage(
            "위 정보를 바탕으로 다음을 분석해주세요:\n"
            "1. 오류의 근본 원인\n"
            "2. 구체적인 수정 방법\n"
            "3. 재발 방지를 위한 권장 사항"
        )
    )

    return messages
INFO

다중 메시지 프롬프트는 LLM에게 대화 컨텍스트를 제공하는 데 유용하다. 시스템 지시, 사용자 입력, 분석 요청 등을 논리적으로 분리할 수 있다.

6.4 스타일별 프롬프트

같은 작업에 대해 다양한 스타일 옵션을 제공하면, 사용자가 상황에 맞는 출력을 선택할 수 있다.

문서 생성 프롬프트

@mcp.prompt()
def generate_docs(
    code: str,
    style: str = "detailed",
    language: str = "python",
) -> str:
    """코드 문서화 프롬프트 (스타일 선택 가능)"""

    style_instructions = {
        "detailed": (
            "각 함수/클래스에 대해 상세한 문서를 작성해주세요. "
            "파라미터 설명, 반환값, 사용 예제를 모두 포함한다."
        ),
        "concise": (
            "각 함수/클래스에 대해 한 줄 요약만 작성해주세요. "
            "핵심 기능만 간결하게 설명한다."
        ),
        "tutorial": (
            "초보자를 위한 튜토리얼 형식으로 문서를 작성해주세요. "
            "단계별로 설명하고 실행 가능한 예제를 포함한다."
        ),
    }

    instruction = style_instructions.get(style, style_instructions["detailed"])

    return (
        f"다음 {language} 코드에 대한 문서를 작성해주세요.\n\n"
        f"스타일: {style}\n"
        f"지침: {instruction}\n\n"
        f"```{language}\n{code}\n```"
    )

커밋 메시지 생성 프롬프트

@mcp.prompt()
def commit_message(
    diff: str,
    convention: str = "conventional",
) -> list[base.Message]:
    """Git 커밋 메시지 생성 프롬프트"""

    conventions = {
        "conventional": (
            "Conventional Commits 형식을 사용한다.\n"
            "형식: (): \n"
            "타입: feat, fix, docs, style, refactor, test, chore"
        ),
        "gitmoji": (
            "Gitmoji 형식을 사용한다.\n"
            "형식:  \n"
            "예: ✨ 새로운 기능 추가"
        ),
        "simple": (
            "간결하고 명확한 한 줄 메시지를 작성한다.\n"
            "현재 시제를 사용하며 50자 이내로 작성한다."
        ),
    }

    convention_guide = conventions.get(convention, conventions["conventional"])

    return [
        base.UserMessage(
            f"다음 git diff를 분석하여 커밋 메시지를 작성해주세요.\n\n"
            f"컨벤션:\n{convention_guide}"
        ),
        base.UserMessage(f"```diff\n{diff}\n```"),
    ]

6.5 실전 예제: 개발 도우미 프롬프트 서버

코드 리뷰, 버그 분석, 문서 생성 프롬프트를 모두 제공하는 개발 도우미 MCP 서버를 완성해 보자. 

import json
from mcp.server.fastmcp import FastMCP
from mcp.server.fastmcp.prompts import base

mcp = FastMCP("dev-assistant-prompts")


# ── 코드 리뷰 프롬프트 ──────────────────────────────────

@mcp.prompt()
def review_code(
    code: str,
    language: str = "python",
    focus: str = "all",
) -> list[base.Message]:
    """코드 리뷰 프롬프트

    focus 옵션: all, security, performance, style
    """
    focus_map = {
        "all": "보안, 성능, 코드 스타일, 가독성을 종합적으로",
        "security": "보안 취약점과 인젝션 공격 가능성을 중심으로",
        "performance": "성능 병목, 메모리 누수, 알고리즘 효율성을 중심으로",
        "style": "코드 컨벤션, 네이밍, 구조 설계를 중심으로",
    }
    focus_desc = focus_map.get(focus, focus_map["all"])

    return [
        base.UserMessage(
            f"당신은 시니어 {language} 개발자이다. "
            f"다음 코드를 {focus_desc} 리뷰해주세요.\n\n"
            f"리뷰 결과를 다음 형식으로 정리해주세요:\n"
            f"- 🔴 심각: 즉시 수정 필요\n"
            f"- 🟡 주의: 개선 권장\n"
            f"- 🟢 제안: 더 나은 방법 제안"
        ),
        base.UserMessage(f"```{language}\n{code}\n```"),
    ]


# ── 버그 분석 프롬프트 ──────────────────────────────────

@mcp.prompt()
def analyze_bug(
    description: str,
    code: str = "",
    error_log: str = "",
    environment: str = "",
) -> list[base.Message]:
    """버그 분석 프롬프트"""
    messages = [
        base.UserMessage(
            "당신은 디버깅 전문가이다. 다음 버그를 체계적으로 분석해주세요."
        ),
        base.UserMessage(f"## 버그 설명\n{description}"),
    ]

    if code:
        messages.append(
            base.UserMessage(f"## 관련 코드\n```python\n{code}\n```")
        )

    if error_log:
        messages.append(
            base.UserMessage(f"## 에러 로그\n```\n{error_log}\n```")
        )

    if environment:
        messages.append(
            base.UserMessage(f"## 실행 환경\n{environment}")
        )

    messages.append(
        base.UserMessage(
            "다음 항목을 포함하여 분석해주세요:\n"
            "1. **원인 분석**: 버그의 근본 원인\n"
            "2. **영향 범위**: 이 버그가 영향을 미치는 범위\n"
            "3. **해결 방법**: 구체적인 수정 코드\n"
            "4. **테스트**: 수정 후 확인해야 할 테스트 케이스\n"
            "5. **예방 조치**: 유사한 버그를 방지하기 위한 조치"
        )
    )

    return messages


# ── 문서 생성 프롬프트 ──────────────────────────────────

@mcp.prompt()
def generate_api_docs(
    code: str,
    format: str = "markdown",
) -> list[base.Message]:
    """API 문서 생성 프롬프트

    format 옵션: markdown, restructuredtext, docstring
    """
    format_instructions = {
        "markdown": "Markdown 형식으로 API 문서를 작성한다.",
        "restructuredtext": "reStructuredText 형식으로 API 문서를 작성한다.",
        "docstring": "Google 스타일 docstring을 각 함수/클래스에 추가한다.",
    }
    instruction = format_instructions.get(format, format_instructions["markdown"])

    return [
        base.UserMessage(
            f"다음 코드에 대한 API 문서를 작성해주세요.\n\n"
            f"출력 형식: {format}\n"
            f"지침: {instruction}\n\n"
            f"각 함수/클래스에 대해 다음을 포함해주세요:\n"
            f"- 설명\n"
            f"- 파라미터 (타입, 기본값, 설명)\n"
            f"- 반환값\n"
            f"- 예외\n"
            f"- 사용 예제"
        ),
        base.UserMessage(f"```python\n{code}\n```"),
    ]


# ── 테스트 생성 프롬프트 ──────────────────────────────────

@mcp.prompt()
def generate_tests(
    code: str,
    framework: str = "pytest",
    coverage: str = "comprehensive",
) -> list[base.Message]:
    """테스트 코드 생성 프롬프트

    framework 옵션: pytest, unittest
    coverage 옵션: basic, comprehensive, edge_cases
    """
    coverage_map = {
        "basic": "핵심 기능에 대한 기본 테스트만 작성한다.",
        "comprehensive": "정상 케이스, 에러 케이스, 경계값을 포괄하는 테스트를 작성한다.",
        "edge_cases": "엣지 케이스와 예외 상황에 집중한 테스트를 작성한다.",
    }
    coverage_desc = coverage_map.get(coverage, coverage_map["comprehensive"])

    return [
        base.UserMessage(
            f"다음 코드에 대한 테스트를 {framework}로 작성해주세요.\n\n"
            f"테스트 범위: {coverage}\n"
            f"지침: {coverage_desc}\n\n"
            f"각 테스트에는 한글로 된 설명을 docstring에 포함한다."
        ),
        base.UserMessage(f"```python\n{code}\n```"),
    ]


if __name__ == "__main__":
    mcp.run()
