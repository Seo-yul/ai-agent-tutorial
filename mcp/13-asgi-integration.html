13.1 왜 ASGI 통합인가?

핵심 개념: ASGI 통합의 의미

ASGI(Asynchronous Server Gateway Interface)는 Python 비동기 웹 애플리케이션의 표준 인터페이스다. MCP 서버를 별도 프로세스로 배포하는 대신, 기존 ASGI 앱(Starlette, FastAPI 등)에 MCP 엔드포인트를 마운트하면 하나의 서버에서 REST API와 MCP를 동시에 제공할 수 있다. 이를 통해 배포 단위를 통합하고, 데이터베이스 연결 풀이나 캐시 같은 자원을 공유하며, 인증/CORS 등의 미들웨어를 한 번만 설정하면 된다.

별도 서버 배포
  • REST API와 MCP 서버를 각각 독립 프로세스로 실행
  • 2개의 포트, 2개의 배포 파이프라인 필요
  • 데이터베이스 연결 풀 등 자원을 공유하려면 외부 의존성(Redis 등) 필요
  • 인증/CORS 미들웨어를 각 서버에 개별 설정
  • 모니터링 대상이 2개로 분산
ASGI 통합 배포
  • 하나의 ASGI 앱에서 REST + MCP를 동시 서빙
  • 1개의 포트, 1개의 배포 파이프라인
  • 인메모리로 DB 풀, 캐시 등을 직접 공유
  • 미들웨어 한 번 설정으로 모든 엔드포인트에 적용
  • 모니터링/로깅 통합

13.2 기본 마운트

Starlette 앱에 MCP 서버를 마운트하는 기본 패턴을 살펴보자. FastMCP의 streamable_http_app() 메서드가 반환하는 ASGI 앱을 Starlette의 Mount에 연결하면 된다. 

import contextlib
from starlette.applications import Starlette
from starlette.routing import Mount, Route
from starlette.responses import JSONResponse
from mcp.server.fastmcp import FastMCP

# MCP 서버 생성
mcp = FastMCP("API Server", json_response=True)

@mcp.tool()
def hello(name: str) -> str:
    """인사한다"""
    return f"안녕, {name}!"

# REST API 핸들러
async def health_check(request):
    return JSONResponse({"status": "ok"})

async def api_users(request):
    return JSONResponse({"users": ["김철수", "이영희"]})

# lifespan: MCP 세션 매니저 수명 관리
@contextlib.asynccontextmanager
async def lifespan(app):
    async with mcp.session_manager.run():
        yield

app = Starlette(
    routes=[
        Route("/health", health_check),
        Route("/api/users", api_users),
        Mount("/mcp", app=mcp.streamable_http_app()),
    ],
    lifespan=lifespan,
)

Lifespan 핸들러의 역할

lifespan은 ASGI 앱의 시작과 종료 시점에 호출되는 비동기 컨텍스트 매니저다. MCP의 session_manager.run()은 세션 생성/소멸, 타임아웃 관리 등을 담당하므로 반드시 lifespan 내에서 실행해야 한다. 이를 빠뜨리면 MCP 엔드포인트가 정상 동작하지 않는다.

엔드포인트 구조

streamable_http_app()이 반환하는 ASGI 앱은 3가지 HTTP 메서드를 처리한다.

메서드 경로 역할
POST /mcp/mcp JSON-RPC 요청 전송. 세션이 없으면 자동 생성한다.
GET /mcp/mcp SSE 스트림 연결. 서버가 보내는 notification과 응답을 실시간 수신한다.
DELETE /mcp/mcp 세션 종료. 클라이언트가 연결을 정리할 때 호출한다.
INFO

경로가 /mcp/mcp인 이유는 Mount("/mcp", ...)와 FastMCP의 기본 streamable_http_path="/mcp"가 결합되기 때문이다. 이를 깔끔하게 정리하는 방법은 13.4절에서 다룬다.

13.3 uvicorn 실행

위 코드를 server.py로 저장한 후 uvicorn으로 실행한다. 

# 개발 모드 — 코드 변경 시 자동 리로드
uvicorn server:app --host 0.0.0.0 --port 8000 --reload

# 프로덕션 모드 — 단일 워커
uvicorn server:app --host 0.0.0.0 --port 8000

# 프로덕션 모드 — SSL 적용
uvicorn server:app --host 0.0.0.0 --port 443 \
  --ssl-keyfile ./key.pem --ssl-certfile ./cert.pem

실행 후 REST API 동작을 확인한다.

# 헬스 체크
curl http://localhost:8000/health
# {"status": "ok"}

# 사용자 목록
curl http://localhost:8000/api/users
# {"users": ["김철수", "이영희"]}
WARNING

MCP Streamable HTTP는 SSE(Server-Sent Events) 스트림을 사용하므로, --workers 옵션으로 여러 워커를 띄울 때는 세션 어피니티(session affinity)가 보장되어야 한다. 이에 대해서는 13.10절 '세션 관리와 스케일링'에서 자세히 다룬다.

13.4 경로 커스터마이징

streamable_http_path 설정을 변경하면 Mount 경로와 조합하여 최종 MCP 엔드포인트 경로를 원하는 대로 구성할 수 있다. 

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Custom Path Server")

# streamable_http_path를 "/"로 설정하면
# Mount 경로가 곧 최종 MCP 엔드포인트가 된다
mcp.settings.streamable_http_path = "/"
Mount 경로 streamable_http_path 최종 MCP 엔드포인트 권장 여부
/mcp /mcp (기본값) /mcp/mcp 비추천 (경로 중복)
/mcp / /mcp 추천
/api/v1/mcp / /api/v1/mcp 추천 (API 버전 포함)
/ /mcp /mcp 가능 (Mount 없이 직접 노출)
TIP

streamable_http_path = "/"로 설정하고 Mount 경로에서 전체 경로를 관리하는 것이 가장 직관적이다. 이후 모든 예제에서 이 패턴을 사용한다.

13.5 미들웨어 활용

Starlette의 미들웨어 시스템을 활용하면 REST API와 MCP 엔드포인트 모두에 공통 기능을 적용할 수 있다.

CORS 미들웨어

브라우저 기반 MCP 클라이언트를 지원하려면 CORS 설정이 필수다. 

from starlette.middleware import Middleware
from starlette.middleware.cors import CORSMiddleware

app = Starlette(
    routes=[...],
    middleware=[
        Middleware(
            CORSMiddleware,
            allow_origins=["https://myapp.com", "http://localhost:3000"],
            allow_methods=["GET", "POST", "DELETE"],
            allow_headers=["Authorization", "Content-Type", "Mcp-Session-Id"],
            expose_headers=["Mcp-Session-Id"],
        ),
    ],
    lifespan=lifespan,
)
WARNING

MCP Streamable HTTP는 Mcp-Session-Id 헤더를 사용하므로, allow_headers와 expose_headers에 반드시 포함시켜야 한다. 프로덕션에서는 allow_origins=["*"] 대신 명시적 도메인을 지정한다.

인증 미들웨어

API Key 또는 Bearer Token 기반의 인증 미들웨어를 작성하여 MCP 엔드포인트를 보호할 수 있다. 

from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import JSONResponse

class BearerAuthMiddleware(BaseHTTPMiddleware):
    """Bearer 토큰 인증 미들웨어 (MCP 경로에만 적용)"""

    def __init__(self, app, secret_key: str = "my-secret"):
        super().__init__(app)
        self.secret_key = secret_key

    async def dispatch(self, request: Request, call_next):
        # /mcp 경로에만 인증 적용
        if request.url.path.startswith("/mcp"):
            # CORS preflight는 통과시킨다
            if request.method == "OPTIONS":
                return await call_next(request)

            auth = request.headers.get("authorization", "")
            if not auth.startswith("Bearer "):
                return JSONResponse(
                    {"error": "Authorization header required"},
                    status_code=401,
                )

            token = auth[7:]
            if not self._validate_token(token):
                return JSONResponse(
                    {"error": "Invalid token"},
                    status_code=403,
                )

        return await call_next(request)

    def _validate_token(self, token: str) -> bool:
        # 실제로는 JWT 검증, DB 조회 등을 수행한다
        return token == self.secret_key

요청 로깅 미들웨어

import time
import logging

logger = logging.getLogger("mcp-server")

class RequestLoggingMiddleware(BaseHTTPMiddleware):
    """모든 요청의 처리 시간을 로깅한다"""

    async def dispatch(self, request: Request, call_next):
        start = time.time()
        response = await call_next(request)
        elapsed = (time.time() - start) * 1000

        logger.info(
            f"{request.method} {request.url.path} "
            f"-> {response.status_code} ({elapsed:.1f}ms)"
        )
        response.headers["X-Process-Time"] = f"{elapsed:.1f}ms"
        return response

미들웨어 적용 순서

미들웨어는 선언 순서대로 바깥에서 안쪽으로 감싸진다. 순서가 중요한 이유를 다이어그램으로 살펴보자. 

graph LR
    REQ["HTTP 요청"] --> CORS["1. CORS"]
    CORS --> LOG["2. 로깅"]
    LOG --> AUTH["3. 인증"]
    AUTH --> APP["앱 핸들러"]
    APP --> AUTH2["3. 인증"]
    AUTH2 --> LOG2["2. 로깅"]
    LOG2 --> CORS2["1. CORS"]
    CORS2 --> RES["HTTP 응답"]

app = Starlette(
    routes=[...],
    middleware=[
        # 1. CORS — 가장 바깥. OPTIONS preflight를 먼저 처리
        Middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"],
                   allow_headers=["*"], expose_headers=["Mcp-Session-Id"]),
        # 2. 로깅 — 모든 요청을 기록
        Middleware(RequestLoggingMiddleware),
        # 3. 인증 — 인증 실패 시 여기서 차단
        Middleware(BearerAuthMiddleware, secret_key="prod-secret"),
    ],
    lifespan=lifespan,
)
WARNING

인증 미들웨어를 CORS보다 뒤에(안쪽에) 선언해야 한다. 그래야 브라우저의 preflight(OPTIONS) 요청이 인증 없이 CORS 미들웨어에서 처리된다. 순서를 반대로 하면 preflight 요청이 인증에 걸려 CORS 오류가 발생한다.

13.6 FastAPI 통합

FastAPI는 Starlette 위에 구축된 프레임워크이므로, 동일한 방식으로 MCP를 마운트할 수 있다. FastAPI의 자동 문서 생성, 의존성 주입, Pydantic 검증 등을 REST API에 활용하면서 동시에 MCP 엔드포인트를 제공하는 것이 가능하다. 

# Starlette 방식
import contextlib
from starlette.applications import Starlette
from starlette.routing import Mount, Route
from starlette.responses import JSONResponse
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("My Server")
mcp.settings.streamable_http_path = "/"

@mcp.tool()
def hello(name: str) -> str:
    return f"안녕, {name}!"

async def health(request):
    return JSONResponse({"status": "ok"})

@contextlib.asynccontextmanager
async def lifespan(app):
    async with mcp.session_manager.run():
        yield

app = Starlette(
    routes=[
        Route("/health", health),
        Mount("/mcp", app=mcp.streamable_http_app()),
    ],
    lifespan=lifespan,
)
# FastAPI 방식
import contextlib
from fastapi import FastAPI
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("My Server")
mcp.settings.streamable_http_path = "/"

@mcp.tool()
def hello(name: str) -> str:
    return f"안녕, {name}!"

@contextlib.asynccontextmanager
async def lifespan(app):
    async with mcp.session_manager.run():
        yield

app = FastAPI(lifespan=lifespan)

# FastAPI에도 mount() 메서드가 있다
app.mount("/mcp", app=mcp.streamable_http_app())

@app.get("/health")
async def health():
    return {"status": "ok"}

REST 엔드포인트와 MCP 도구의 공존

FastAPI의 강력한 기능(Pydantic 모델, 의존성 주입, 자동 문서)을 REST API에 활용하면서, 같은 비즈니스 로직을 MCP 도구로도 노출하는 패턴이다. 

import contextlib
from fastapi import FastAPI, HTTPException, Depends
from pydantic import BaseModel
from mcp.server.fastmcp import FastMCP, Context

# ── 공유 비즈니스 로직 ───────────────────────────────
class UserDB:
    """REST API와 MCP 도구가 공유하는 데이터 레이어"""
    _users = {1: {"name": "김철수", "email": "cs@test.com"}}
    _next_id = 2

    @classmethod
    def get(cls, user_id: int) -> dict | None:
        return cls._users.get(user_id)

    @classmethod
    def create(cls, name: str, email: str) -> dict:
        user = {"name": name, "email": email}
        cls._users[cls._next_id] = user
        cls._next_id += 1
        return {**user, "id": cls._next_id - 1}

    @classmethod
    def search(cls, query: str) -> list[dict]:
        q = query.lower()
        return [
            {**u, "id": uid}
            for uid, u in cls._users.items()
            if q in u["name"].lower() or q in u["email"].lower()
        ]


# ── FastAPI REST API ─────────────────────────────────
class UserCreate(BaseModel):
    name: str
    email: str

mcp = FastMCP("User Service")
mcp.settings.streamable_http_path = "/"

@contextlib.asynccontextmanager
async def lifespan(app):
    async with mcp.session_manager.run():
        yield

app = FastAPI(title="User Service", lifespan=lifespan)
app.mount("/mcp", app=mcp.streamable_http_app())

@app.get("/api/users/{user_id}")
async def get_user(user_id: int):
    user = UserDB.get(user_id)
    if not user:
        raise HTTPException(404, "사용자를 찾을 수 없음")
    return {**user, "id": user_id}

@app.post("/api/users", status_code=201)
async def create_user(body: UserCreate):
    return UserDB.create(body.name, body.email)


# ── MCP 도구 (같은 UserDB 사용) ────────────────────
@mcp.tool()
async def search_users(ctx: Context, query: str) -> str:
    """사용자를 검색한다"""
    await ctx.info(f"검색: {query}")
    results = UserDB.search(query)
    if not results:
        return "검색 결과 없음"
    return "\n".join(f"- {u['name']} ({u['email']})" for u in results)

@mcp.tool()
async def get_user_info(ctx: Context, user_id: int) -> str:
    """사용자 정보를 조회한다"""
    user = UserDB.get(user_id)
    if not user:
        await ctx.warning(f"사용자 {user_id} 없음")
        return f"ID {user_id}에 해당하는 사용자 없음"
    return f"이름: {user['name']}, 이메일: {user['email']}"
TIP

FastAPI를 사용하면 /docs 경로에서 REST API의 Swagger UI를 자동으로 볼 수 있다. MCP 도구는 별도의 문서가 필요 없는데, 클라이언트가 tools/list로 도구 목록과 스키마를 동적으로 조회하기 때문이다.

13.7 다중 서버 마운트

하나의 ASGI 앱에 여러 MCP 서버를 도메인별로 분리하여 마운트할 수 있다. 각 MCP 서버는 독립적인 도구/리소스 세트를 가지며, 서로 다른 경로에서 서비스된다. 

import contextlib
from fastapi import FastAPI
from mcp.server.fastmcp import FastMCP

# 도메인별 MCP 서버 생성
api_mcp = FastMCP("API Tools", stateless_http=True, json_response=True)
api_mcp.settings.streamable_http_path = "/"

chat_mcp = FastMCP("Chat Tools", stateless_http=True, json_response=True)
chat_mcp.settings.streamable_http_path = "/"

@api_mcp.tool()
def get_user(user_id: int) -> str:
    """사용자 정보를 조회한다"""
    return f"사용자 {user_id}: 김철수"

@api_mcp.tool()
def list_products() -> str:
    """상품 목록을 조회한다"""
    return "상품: [노트북, 키보드, 마우스]"

@chat_mcp.tool()
def send_message(channel: str, message: str) -> str:
    """메시지를 전송한다"""
    return f"[{channel}] 전송 완료: {message}"

@chat_mcp.tool()
def get_history(channel: str, limit: int = 10) -> str:
    """채팅 기록을 조회한다"""
    return f"[{channel}] 최근 {limit}개 메시지"

# 여러 세션 매니저를 하나의 lifespan에서 관리
@contextlib.asynccontextmanager
async def lifespan(app):
    async with contextlib.AsyncExitStack() as stack:
        await stack.enter_async_context(api_mcp.session_manager.run())
        await stack.enter_async_context(chat_mcp.session_manager.run())
        yield

app = FastAPI(lifespan=lifespan)
app.mount("/api-mcp", app=api_mcp.streamable_http_app())
app.mount("/chat-mcp", app=chat_mcp.streamable_http_app())

MCP 클라이언트 설정에서 각 서버에 별도로 연결한다. 

{
  "mcpServers": {
    "api": { "url": "http://localhost:8000/api-mcp" },
    "chat": { "url": "http://localhost:8000/chat-mcp" }
  }
}
TIP

stateless_http=True를 설정하면 서버가 세션 상태를 유지하지 않으므로 수평 확장(스케일 아웃)이 용이하다. 세션이 필요 없는 간단한 도구 서버에 적합하다.

13.8 Docker 배포

ASGI 통합 서버를 Docker로 컨테이너화하면 일관된 배포 환경을 보장하고, 클라우드 환경이나 Kubernetes에서 쉽게 운영할 수 있다.

Dockerfile (멀티스테이지 빌드)

gunicorn + uvicorn workers

프로덕션에서는 gunicorn을 프로세스 매니저로 사용하고, uvicorn을 워커 클래스로 지정하는 것이 일반적이다. 단, MCP의 세션 관리 특성상 워커 수 설정에 주의가 필요하다. 

# gunicorn으로 프로덕션 실행

Nginx 리버스 프록시 + SSL Termination

Nginx를 앞단에 두어 SSL 종단, 정적 파일 서빙, 로드 밸런싱을 처리한다. MCP의 SSE 스트리밍을 위해 프록시 버퍼링을 반드시 비활성화해야 한다. 

# /etc/nginx/conf.d/mcp-server.conf

upstream mcp_backend {
    # 세션 어피니티: ip_hash로 같은 클라이언트를 같은 워커에 연결
    ip_hash;
    server 127.0.0.1:8000;
    server 127.0.0.1:8001;
}

server {
    listen 443 ssl http2;
    server_name mcp.example.com;

    ssl_certificate     /etc/letsencrypt/live/mcp.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/mcp.example.com/privkey.pem;

    # MCP 엔드포인트 — SSE 스트리밍을 위한 설정
    location /mcp {
        proxy_pass http://mcp_backend;
        proxy_http_version 1.1;

        # SSE에 필수: 프록시 버퍼링 비활성화
        proxy_buffering off;
        proxy_cache off;

        # 타임아웃 설정 (SSE 연결은 오래 유지됨)
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;

        # 헤더 전달
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Connection '';
    }

    # REST API 엔드포인트
    location /api {
        proxy_pass http://mcp_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /health {
        proxy_pass http://mcp_backend;
    }
}
WARNING

SSE 엔드포인트에서 proxy_buffering off와 proxy_cache off는 반드시 설정해야 한다. 버퍼링이 활성화되면 서버가 보낸 이벤트가 Nginx 버퍼에 갇혀 클라이언트에 즉시 전달되지 않는다. proxy_read_timeout도 충분히 길게 설정한다.

13.10 세션 관리와 스케일링

핵심 개념: Stateful vs Stateless

MCP Streamable HTTP 세션은 기본적으로 stateful이다. 초기화(initialize) 이후 클라이언트와 서버 사이에 세션 상태가 유지되며, Mcp-Session-Id 헤더를 통해 세션이 식별된다. 이 상태가 서버 메모리에 저장되므로, 같은 클라이언트의 요청이 항상 같은 서버 인스턴스로 라우팅되어야 한다(세션 어피니티). 반면 stateless_http=True를 사용하면 매 요청이 독립적이므로 자유롭게 스케일 아웃할 수 있다.

Stateful (기본값)
  • 세션 상태 유지 — 구독, 알림 등 지원
  • SSE 스트림으로 서버 → 클라이언트 푸시 가능
  • 스케일 아웃 시 세션 어피니티 필요
  • 서버 재시작 시 세션 유실
  • 리소스 구독, 진행률 보고 등 완전한 MCP 기능
Stateless (stateless_http=True)
  • 매 요청이 독립적 — 세션 없음
  • SSE 스트림 없음 — 요청/응답만
  • 자유롭게 수평 확장 가능
  • 서버 재시작 영향 없음
  • 구독/알림 불가, 간단한 도구 호출에 적합

로드 밸런서 세션 어피니티

Stateful 모드에서 여러 서버 인스턴스를 운영할 때, 로드 밸런서가 같은 세션의 요청을 같은 인스턴스로 보내야 한다.

로드 밸런서 세션 어피니티 설정 비고
Nginx ip_hash 또는 sticky cookie ip_hash가 가장 간단
AWS ALB Target Group Sticky Sessions 쿠키 기반, duration 설정 필요
Kubernetes Service sessionAffinity: ClientIP timeout 설정 권장
Traefik sticky.cookie IngressRoute에서 설정
Deep Dive: 언제 Stateless를 선택하는가?

stateless_http=True는 다음 조건을 모두 만족할 때 적합하다. (1) 도구가 단순한 요청-응답 패턴이고 서버 → 클라이언트 푸시가 필요 없다. (2) 리소스 구독(resources/subscribe)을 사용하지 않는다. (3) 진행률 보고(notifications/progress)가 불필요하다. (4) 높은 동시 요청 처리와 수평 확장이 최우선이다. 위 조건 중 하나라도 해당하지 않으면 기본 stateful 모드를 사용하고 세션 어피니티를 설정한다.

@dataclass
class Item:
    id: int
    name: str
    description: str
    price: float
    category: str

    def to_dict(self) -> dict:
        return asdict(self)


@dataclass
class ItemStore:
    items: dict[int, Item] = field(default_factory=dict)
    next_id: int = 1

    def create(self, name: str, description: str,
               price: float, category: str) -> Item:
        item = Item(self.next_id, name, description, price, category)
        self.items[self.next_id] = item
        self.next_id += 1
        return item

    def get(self, item_id: int) -> Item | None:
        return self.items.get(item_id)

    def list_all(self, category: str | None = None) -> list[Item]:
        items = list(self.items.values())
        if category:
            items = [i for i in items if i.category == category]
        return items

    def search(self, query: str) -> list[Item]:
        q = query.lower()
        return [
            i for i in self.items.values()
            if q in i.name.lower() or q in i.description.lower()
        ]

    def update(self, item_id: int, **kwargs) -> Item | None:
        item = self.items.get(item_id)
        if not item:
            return None
        for key, val in kwargs.items():
            if hasattr(item, key) and val is not None:
                setattr(item, key, val)
        return item

    def delete(self, item_id: int) -> bool:
        return self.items.pop(item_id, None) is not None

class RequestTracingMiddleware(BaseHTTPMiddleware):
    """모든 요청에 추적 ID를 부여하고 처리 시간을 로깅한다"""

    async def dispatch(self, request: Request, call_next):
        request_id = str(uuid.uuid4())[:8]
        start = time.time()

        # 요청 헤더에 추적 ID 설정 (하위 핸들러에서 접근 가능)
        request.state.request_id = request_id

        response = await call_next(request)
        elapsed = (time.time() - start) * 1000

        logger.info(
            "http_request",
            request_id=request_id,
            method=request.method,
            path=request.url.path,
            status=response.status_code,
            elapsed_ms=round(elapsed, 2),
        )

        response.headers["X-Request-Id"] = request_id
        response.headers["X-Process-Time"] = f"{elapsed:.1f}ms"
        return response


class APIKeyMiddleware(BaseHTTPMiddleware):
    """MCP 경로에 대한 API 키 인증"""

    VALID_KEYS = {"prod-key-001", "prod-key-002", "dev-key"}

    async def dispatch(self, request: Request, call_next):
        if request.url.path.startswith("/mcp"):
            if request.method == "OPTIONS":
                return await call_next(request)

            api_key = request.headers.get("x-api-key", "")
            if api_key not in self.VALID_KEYS:
                logger.warning(
                    "auth_failed",
                    path=request.url.path,
                    reason="invalid_api_key",
                )
                return JSONResponse(
                    {"error": "유효하지 않은 API 키"},
                    status_code=401,
                )

        return await call_next(request)

mcp = FastMCP("Product Catalog MCP", json_response=True)

@mcp.tool()
async def search_items(ctx: Context, query: str) -> str:
    """상품을 이름 또는 설명으로 검색한다"""
    await ctx.info(f"검색어: {query}")
    results = store.search(query)

    if not results:
        await ctx.info("검색 결과 없음")
        return "검색 결과가 없다."

    await ctx.info(f"{len(results)}개 상품 발견")
    lines = []
    for item in results:
        lines.append(
            f"[{item.id}] {item.name} - "
            f"{item.price:,.0f}원 ({item.category})"
        )
    return "\n".join(lines)

@mcp.tool()
async def get_item_detail(ctx: Context, item_id: int) -> str:
    """상품 상세 정보를 조회한다"""
    item = store.get(item_id)
    if not item:
        await ctx.warning(f"상품 {item_id} 없음")
        return f"ID {item_id}에 해당하는 상품이 없다."

    await ctx.info(f"상품 조회: {item.name}")
    return json.dumps(item.to_dict(), ensure_ascii=False, indent=2)

@mcp.tool()
async def list_by_category(ctx: Context, category: str) -> str:
    """카테고리별 상품 목록을 조회한다"""
    items = store.list_all(category=category)
    if not items:
        return f"'{category}' 카테고리에 상품이 없다."

    await ctx.info(f"카테고리 '{category}': {len(items)}개")
    lines = [f"=== {category} ==="]
    for item in items:
        lines.append(f"  [{item.id}] {item.name} - {item.price:,.0f}원")
    return "\n".join(lines)

@mcp.tool()
async def get_catalog_stats(ctx: Context) -> str:
    """상품 카탈로그 통계를 반환한다"""
    all_items = store.list_all()
    categories: dict[str, int] = {}
    total_value = 0.0

    for item in all_items:
        categories[item.category] = categories.get(item.category, 0) + 1
        total_value += item.price

    stats = {
        "total_items": len(all_items),
        "categories": categories,
        "total_value": f"{total_value:,.0f}원",
        "avg_price": f"{total_value / len(all_items):,.0f}원" if all_items else "0원",
    }

    await ctx.info({"event": "stats", **stats})
    return json.dumps(stats, ensure_ascii=False, indent=2)

# MCP 마운트
app.mount("/mcp", app=mcp.streamable_http_app())

@app.get("/health")
async def health():
    """헬스 체크"""
    return {
        "status": "ok",
        "service": "product-catalog",
        "items_count": len(store.list_all()),
    }

@app.get("/api/items")
async def list_items(category: str | None = None):
    """상품 목록 조회"""
    items = store.list_all(category=category)
    return {"items": [i.to_dict() for i in items]}

@app.get("/api/items/{item_id}")
async def get_item(item_id: int):
    """상품 단건 조회"""
    item = store.get(item_id)
    if not item:
        raise HTTPException(404, "상품을 찾을 수 없음")
    return {"item": item.to_dict()}

@app.post("/api/items", status_code=201)
async def create_item(body: ItemCreate):
    """상품 생성"""
    item = store.create(body.name, body.description, body.price, body.category)
    logger.info("item_created", item_id=item.id, name=item.name)
    return {"item": item.to_dict()}

@app.put("/api/items/{item_id}")
async def update_item(item_id: int, body: ItemUpdate):
    """상품 수정"""
    item = store.update(item_id, **body.model_dump(exclude_none=True))
    if not item:
        raise HTTPException(404, "상품을 찾을 수 없음")
    return {"item": item.to_dict()}

@app.delete("/api/items/{item_id}")
async def delete_item(item_id: int):
    """상품 삭제"""
    if not store.delete(item_id):
        raise HTTPException(404, "상품을 찾을 수 없음")
    logger.info("item_deleted", item_id=item_id)
    return {"message": f"상품 {item_id} 삭제됨"}

TIP

이 예제의 ItemStore를 SQLAlchemy나 MongoDB로 교체하면 실제 프로덕션에서 사용할 수 있는 수준이 된다. Docker 배포 시에는 13.8절의 Dockerfile과 docker-compose.yml을 참고하여 데이터베이스와 함께 컨테이너화하면 된다.

INFO

FastAPI의 /docs 경로에 접속하면 REST API의 Swagger UI를 확인할 수 있다. MCP 도구는 tools/list 요청으로 클라이언트가 동적으로 조회하므로 별도의 API 문서가 필요 없다.
