MCP Python SDK 튜토리얼 v1.12.x
기본 과정

환경 설정

MCP 개발을 위한 Python 환경을 구성한다

사전 요구사항

MCP Python SDK를 사용하려면 Python 3.10 이상이 필요하다. 터미널에서 현재 설치된 Python 버전을 확인하자.

python3 --version
# 출력 예시: Python 3.12.4

Python 3.10 미만이 설치되어 있다면, python.org에서 최신 버전을 다운로드하거나, 아래 uv를 설치한 후 uv를 통해 Python을 관리할 수도 있다.

INFO

macOS 사용자는 brew install python, Ubuntu 사용자는 sudo apt install python3으로도 설치할 수 있다.

uv 설치

uv는 Rust로 작성된 고속 Python 패키지 매니저이자 프로젝트 관리 도구이다. pip보다 10~100배 빠르며, 프로젝트별 가상 환경을 자동으로 관리해준다. MCP 공식 문서에서도 uv를 권장하고 있다.

설치 명령어

macOS / Linux:

# uv 설치 (공식 설치 스크립트)
curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell):

# Windows용 uv 설치
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

설치가 완료되면 터미널을 재시작하고 버전을 확인한다.

uv --version
# 출력 예시: uv 0.5.x
TIP

uv를 사용하면 pip install, python -m venv, pip freeze 등의 작업을 모두 하나의 도구로 처리할 수 있다. 가상 환경 생성과 패키지 설치를 자동으로 해주기 때문에, 직접 venv를 관리할 필요가 없다.

프로젝트 초기화

uv를 사용하여 새 프로젝트를 만들고 MCP SDK를 설치한다.

# 새 프로젝트 생성
uv init mcp-demo

# 프로젝트 디렉토리로 이동
cd mcp-demo

# MCP Python SDK 설치
uv add mcp

uv init은 프로젝트 디렉토리를 생성하고 기본 설정 파일(pyproject.toml)을 만든다. uv add mcp는 MCP SDK를 의존성에 추가하고 자동으로 설치한다.

TIP

HTTP 기반 도구를 만들 계획이라면 uv add mcp httpx로 HTTP 클라이언트 라이브러리도 함께 설치하는 것을 권장한다.

프로젝트 구조

위 명령어를 실행하면 다음과 같은 디렉토리 구조가 생성된다.

mcp-demo/
├── .python-version     # Python 버전 지정
├── .venv/              # 가상 환경 (uv가 자동 생성)
├── pyproject.toml      # 프로젝트 설정 및 의존성
├── README.md           # 프로젝트 설명
└── main.py             # 기본 Python 파일

각 파일의 역할을 살펴보자:

파일/디렉토리 설명
.python-version 프로젝트에서 사용할 Python 버전을 명시한다.
.venv/ uv가 자동으로 생성하는 가상 환경이다. 직접 관리할 필요 없다.
pyproject.toml 프로젝트 메타데이터와 의존성을 정의한다. uv add로 패키지를 추가하면 이 파일이 업데이트된다.
main.py 기본 엔트리 포인트이다. 이 파일에 MCP 서버 코드를 작성하거나, 별도의 server.py를 생성할 수 있다.

pyproject.toml 파일을 열어보면 다음과 같은 내용이 포함되어 있다:

[project]
name = "mcp-demo"
version = "0.1.0"
description = ""
requires-python = ">=3.10"
dependencies = [
    "mcp",
]

설치 확인

MCP SDK가 정상적으로 설치되었는지 확인하자. main.py 파일에 다음 코드를 작성한다.

from mcp.server.fastmcp import FastMCP

# FastMCP 임포트에 성공하면 SDK가 올바르게 설치된 것이다
print("MCP SDK 설치 완료!")

# SDK 버전 확인
import mcp
print(f"MCP 버전: {mcp.__version__}")

uv로 실행해본다:

uv run main.py
# 출력:
# MCP SDK 설치 완료!
# MCP 버전: 1.12.x

위와 같이 에러 없이 출력되면 환경 설정이 완료된 것이다.

WARNING

ModuleNotFoundError가 발생한다면, uv add mcp를 다시 실행하고 uv run으로 실행하고 있는지 확인하자. python main.py가 아닌 uv run main.py를 사용해야 uv의 가상 환경에서 실행된다.
