클라이언트 구현, LLM 에이전트 통합, Sampling, 그리고 다중 서버 연결

8.1 클라이언트 개요

핵심 개념: MCP 클라이언트의 4가지 역할

MCP 클라이언트는 단순히 서버에 연결하는 것이 아니라, 다음 4가지 핵심 역할을 수행한다.

  1. 연결 관리 — Transport를 통해 서버에 연결하고, 세션을 초기화하며, 연결 끊김에 대응한다.
  2. 기능 발견 — 서버가 제공하는 도구, 리소스, 프롬프트 목록을 동적으로 조회한다.
  3. 기능 호출 — 발견한 도구를 호출하고, 리소스를 읽고, 프롬프트를 가져온다.
  4. 알림 처리 — 서버가 보내는 목록 변경 알림, 로그 메시지 등을 수신하고 처리한다.

Claude Desktop, Cursor 같은 LLM 호스트가 대표적인 MCP 클라이언트이지만, MCP Python SDK를 사용하면 직접 클라이언트를 구현할 수 있다. 이를 통해 자동화 스크립트, 테스트 도구, 커스텀 AI 에이전트 등 다양한 활용이 가능하다.

클라이언트 라이프사이클

stateDiagram-v2
    [*] --> Connecting: Transport 연결
    Connecting --> Initializing: 파이프/HTTP 연결 완료
    Initializing --> Ready: initialize() 성공
capabilities 교환
    Ready --> Working: 도구/리소스/프롬프트 사용
    Working --> Ready: 작업 완료
    Ready --> Disconnected: 연결 끊김
    Working --> Disconnected: 에러 발생
    Disconnected --> Connecting: 재연결 시도
    Disconnected --> [*]: 종료

핵심 포인트는 initialize()가 반드시 첫 번째로 호출되어야 한다는 것이다. 이 과정에서 클라이언트와 서버가 프로토콜 버전, 지원 기능(capabilities), 서버/클라이언트 정보를 교환한다. 초기화 없이 다른 메서드를 호출하면 서버가 요청을 거부한다.

8.2 stdio 클라이언트

stdio 클라이언트는 로컬에서 서버 프로세스를 직접 실행하고, stdin/stdout 파이프로 연결한다. 별도의 서버 프로세스를 미리 띄울 필요가 없다 — 클라이언트가 서버를 자동으로 시작한다.

기본 연결 및 도구 호출

import asyncio
from mcp.client.session import ClientSession
from mcp.client.stdio import StdioServerParameters, stdio_client

async def main():
    # 서버 실행 파라미터 설정
    server_params = StdioServerParameters(
        command="uv",
        args=["run", "server.py"],
    )

    # 서버에 연결 (자동으로 프로세스 시작)
    async with stdio_client(server_params) as (read, write):
        async with ClientSession(read, write) as session:
            # 초기화 (반드시 첫 번째 호출)
            await session.initialize()

            # 도구 목록 조회
            tools = await session.list_tools()
            print(f"도구 목록: {[t.name for t in tools.tools]}")

            # 도구 호출
            result = await session.call_tool("add", {"a": 5, "b": 3})
            print(f"결과: {result.content[0].text}")

            # 리소스 읽기
            from pydantic import AnyUrl
            resource = await session.read_resource(AnyUrl("config://app"))
            print(f"리소스: {resource.contents[0].text}")

            # 프롬프트 가져오기
            prompt = await session.get_prompt("summarize", {"text": "MCP란..."})
            print(f"프롬프트: {prompt.messages[0].content.text}")

asyncio.run(main())

8.3 Streamable HTTP 클라이언트

Streamable HTTP 클라이언트는 이미 실행 중인 원격 MCP 서버에 HTTP를 통해 연결한다. 서버가 사전에 실행 중이어야 하며, 서버의 URL을 지정하여 연결한다.

기본 연결

import asyncio
from mcp.client.session import ClientSession
from mcp.client.streamable_http import streamable_http_client

async def main():
    url = "http://localhost:8000/mcp"

    async with streamable_http_client(url) as (read, write, _):
        async with ClientSession(read, write) as session:
            await session.initialize()

            # 도구 목록 조회
            tools = await session.list_tools()
            for tool in tools.tools:
                print(f"  - {tool.name}: {tool.description}")

            # 도구 호출
            result = await session.call_tool("hello", {"name": "세계"})
            print(f"결과: {result.content[0].text}")

asyncio.run(main())
INFO

streamable_http_client()는 3개의 값 (read, write, session_info)를 반환한다. 세 번째 값은 세션 관련 메타데이터인데, 대부분의 경우 사용하지 않으므로 _로 무시한다.

인증 헤더 추가

8.4 LLM 에이전트 루프

핵심 개념: MCP의 진짜 가치 — 에이전트 루프

MCP 클라이언트를 직접 호출하는 것도 유용하지만, MCP의 진짜 가치는 LLM과 결합하여 에이전트 루프를 구성할 때 드러난다. 에이전트 루프에서 LLM은 사용자의 자연어 요청을 분석하고, 필요한 도구를 스스로 선택하여 호출하며, 결과를 기반으로 다음 행동을 결정한다. 이 과정이 목표 달성까지 반복된다.

에이전트 루프 패턴

graph TD
    A["사용자 입력"] --> B["LLM 호출\n(도구 목록과 함께)"]
    B --> C{"LLM 응답에\ntool_use가 있는가?"}
    C -->|"예"| D["MCP tool 호출"]
    D --> E["결과를 대화에 추가"]
    E --> B
    C -->|"아니오"| F["최종 텍스트 응답\n사용자에게 반환"]
  1. 1
    도구 목록 변환 MCP 서버에서 list_tools()로 가져온 도구 목록을 LLM API 형식에 맞게 변환한다.
  2. 2
    LLM 호출 사용자 메시지와 도구 목록을 함께 LLM에 전달한다.
  3. 3
    tool_use 감지 LLM 응답에 tool_use(Anthropic) 또는 tool_calls(OpenAI) 블록이 있는지 확인한다.
  4. 4
    MCP tool 호출 LLM이 선택한 도구를 MCP 클라이언트로 호출하고, 결과를 tool_result 메시지로 대화에 추가한다.
  5. 5
    반복 또는 종료 LLM이 더 이상 도구를 호출하지 않고 텍스트 응답만 반환하면 루프가 종료된다.

MCP 도구를 LLM API 형식으로 변환

MCP 도구의 스키마와 LLM API의 도구 스키마는 약간 다르다. 다음 코드는 MCP 도구를 Anthropic API 형식으로 변환하는 유틸리티 함수이다. 

def mcp_tools_to_anthropic_tools(mcp_tools):
    """MCP 도구 목록을 Anthropic API 형식으로 변환한다"""
    anthropic_tools = []
    for tool in mcp_tools:
        anthropic_tools.append({
            "name": tool.name,
            "description": tool.description or "",
            "input_schema": tool.inputSchema,
        })
    return anthropic_tools


def mcp_tools_to_openai_tools(mcp_tools):
    """MCP 도구 목록을 OpenAI API 형식으로 변환한다"""
    openai_tools = []
    for tool in mcp_tools:
        openai_tools.append({
            "type": "function",
            "function": {
                "name": tool.name,
                "description": tool.description or "",
                "parameters": tool.inputSchema,
            },
        })
    return openai_tools

Anthropic API 에이전트 루프

다음은 Anthropic Claude API와 MCP 클라이언트를 결합한 에이전트 루프의 핵심 구현이다. LLM이 tool_use 블록을 반환하면 MCP로 도구를 호출하고, 결과를 다시 LLM에 피드백하는 과정을 반복한다. 

import anthropic

async def agent_loop(
    session,          # MCP ClientSession
    user_message: str,
    anthropic_tools: list,
):
    """Anthropic API 기반 에이전트 루프"""
    client = anthropic.Anthropic()
    messages = [{"role": "user", "content": user_message}]

    while True:
        # 1. LLM 호출 (도구 목록 포함)
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=4096,
            tools=anthropic_tools,
            messages=messages,
        )

        # 2. 응답을 대화에 추가
        messages.append({"role": "assistant", "content": response.content})

        # 3. stop_reason 확인 — tool_use가 아니면 루프 종료
        if response.stop_reason != "tool_use":
            # 최종 텍스트 응답 추출
            final_text = ""
            for block in response.content:
                if hasattr(block, "text"):
                    final_text += block.text
            return final_text

        # 4. tool_use 블록 처리
        tool_results = []
        for block in response.content:
            if block.type != "tool_use":
                continue

            tool_name = block.name
            tool_args = block.input
            print(f"  [도구 호출] {tool_name}({tool_args})")

            # 5. MCP를 통해 도구 호출
            result = await session.call_tool(tool_name, tool_args)
            result_text = result.content[0].text if result.content else ""
            print(f"  [도구 결과] {result_text}")

            tool_results.append({
                "type": "tool_result",
                "tool_use_id": block.id,
                "content": result_text,
            })

        # 6. 도구 결과를 대화에 추가하고 다시 루프
        messages.append({"role": "user", "content": tool_results})
Deep Dive: LLM의 도구 선택 과정

LLM은 도구 목록(이름, 설명, 파라미터 스키마)을 시스템 프롬프트의 일부로 받는다. 사용자 메시지를 분석한 후, 목표를 달성하기 위해 어떤 도구를 어떤 인자로 호출할지 스스로 결정한다. 이 과정에서 도구의 description이 매우 중요한 역할을 한다 — 설명이 부정확하면 LLM이 잘못된 도구를 선택하거나, 올바른 도구가 있는데도 사용하지 않을 수 있다.

LLM이 한 번의 호출로 여러 도구를 동시에 사용할 수도 있다 (parallel tool use). 예를 들어 "서울과 부산의 날씨를 알려줘"라는 요청에 대해 get_weather("서울")과 get_weather("부산")을 동시에 호출할 수 있다.

OpenAI API 에이전트 루프

OpenAI API를 사용하는 경우, tool_calls 처리 방식이 약간 다르다.

import json
from openai import OpenAI

async def agent_loop_openai(
    session,          # MCP ClientSession
    user_message: str,
    openai_tools: list,
):
    """OpenAI API 기반 에이전트 루프"""
    client = OpenAI()
    messages = [{"role": "user", "content": user_message}]

    while True:
        # 1. LLM 호출
        response = client.chat.completions.create(
            model="gpt-4o",
            tools=openai_tools,
            messages=messages,
        )

        choice = response.choices[0]
        messages.append(choice.message)

        # 2. tool_calls가 없으면 루프 종료
        if not choice.message.tool_calls:
            return choice.message.content

        # 3. 각 tool_call 처리
        for tool_call in choice.message.tool_calls:
            tool_name = tool_call.function.name
            tool_args = json.loads(tool_call.function.arguments)
            print(f"  [도구 호출] {tool_name}({tool_args})")

            # 4. MCP를 통해 도구 호출
            result = await session.call_tool(tool_name, tool_args)
            result_text = result.content[0].text if result.content else ""

            # 5. 도구 결과를 대화에 추가
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call.id,
                "content": result_text,
            })

8.5 Sampling

핵심 개념: Sampling — 서버가 LLM을 호출한다

일반적으로 클라이언트가 LLM을 호출하지만, Sampling은 이 방향을 뒤집는다. MCP 서버가 도구 실행 중에 클라이언트의 LLM을 호출하여 텍스트 생성을 요청하는 기능이다. 이를 통해 서버가 중간 판단이 필요한 복잡한 작업을 수행하거나, 자율 에이전트 체인을 구성할 수 있다.

Sampling이 필요한 상황

  • 중간 판단 — 서버가 여러 단계 작업을 수행하는 도중, 다음 단계를 LLM에게 결정하게 할 때
  • 콘텐츠 생성 — 서버가 데이터를 수집한 후, LLM에게 요약이나 분석을 요청할 때
  • 분류/추출 — 서버가 원본 데이터를 가지고 있고, LLM에게 카테고리 분류나 엔티티 추출을 요청할 때
  • 자율 에이전트 — 서버가 반복적으로 LLM을 호출하여 복잡한 작업을 자율적으로 수행할 때

Sampling 흐름

sequenceDiagram
    participant U as 사용자
    participant C as 클라이언트 (LLM Host)
    participant S as MCP 서버

    U->>C: "이 코드를 리뷰해줘"
    C->>S: tools/call "code_review"
    Note over S: 코드 분석 시작...
    S->>C: sampling/createMessage
"이 함수의 복잡도를 평가해줘"
    Note over C: LLM 호출 (Human-in-the-loop 가능)
    C-->>S: 텍스트 생성 결과
    Note over S: LLM 평가 결과를 활용하여
최종 리뷰 보고서 작성
    S-->>C: tools/call 결과 (리뷰 보고서)
    C-->>U: 리뷰 결과 표시

서버 측: Sampling 요청하기

서버 코드에서 ctx.session.create_message()를 호출하여 클라이언트의 LLM에 텍스트 생성을 요청한다.

from mcp.server.fastmcp import FastMCP, Context
from mcp.types import SamplingMessage, TextContent

mcp = FastMCP("sampling-server")

@mcp.tool()
async def smart_summarize(text: str, ctx: Context) -> str:
    """텍스트를 분석한 후, LLM에게 요약을 요청한다"""

    # 1. 서버 측에서 기본 분석 수행
    word_count = len(text.split())
    char_count = len(text)

    # 2. 클라이언트의 LLM에 Sampling 요청
    result = await ctx.session.create_message(
        messages=[
            SamplingMessage(
                role="user",
                content=TextContent(
                    type="text",
                    text=f"다음 텍스트를 3줄로 요약해라.\n\n{text}",
                ),
            ),
        ],
        max_tokens=500,
    )

    # 3. LLM 응답과 분석 결과를 결합하여 반환
    summary = result.content.text if hasattr(result.content, 'text') else str(result.content)
    return (
        f"=== 텍스트 분석 ===\n"
        f"단어 수: {word_count}\n"
        f"글자 수: {char_count}\n\n"
        f"=== LLM 요약 ===\n{summary}"
    )

if __name__ == "__main__":
    mcp.run()

클라이언트 측: SamplingHandler 구현

클라이언트는 서버의 Sampling 요청을 처리하기 위해 sampling_callback을 제공해야 한다. 이 콜백이 없으면 서버의 Sampling 요청이 실패한다. 

import anthropic
from mcp.client.session import ClientSession
from mcp.client.stdio import StdioServerParameters, stdio_client
from mcp.types import CreateMessageRequestParams, CreateMessageResult, TextContent

# Sampling 핸들러: 서버의 LLM 호출 요청을 처리
async def sampling_handler(
    params: CreateMessageRequestParams,
) -> CreateMessageResult:
    """서버가 Sampling을 요청하면 이 함수가 호출된다"""
    client = anthropic.Anthropic()

    # 서버의 메시지를 Anthropic API 형식으로 변환
    messages = []
    for msg in params.messages:
        content = msg.content.text if hasattr(msg.content, 'text') else str(msg.content)
        messages.append({
            "role": msg.role,
            "content": content,
        })

    # LLM 호출
    response = client.messages.create(
        model=params.modelPreferences.hints[0].name
              if params.modelPreferences and params.modelPreferences.hints
              else "claude-sonnet-4-20250514",
        max_tokens=params.maxTokens,
        messages=messages,
    )

    # 결과를 MCP 형식으로 반환
    return CreateMessageResult(
        role="assistant",
        content=TextContent(
            type="text",
            text=response.content[0].text,
        ),
        model=response.model,
    )


async def main():
    params = StdioServerParameters(
        command="uv",
        args=["run", "server.py"],
    )

    async with stdio_client(params) as (read, write):
        # sampling_callback을 전달하여 Sampling 지원 활성화

            read, write,
            sampling_callback=sampling_handler,

            print(result.content[0].text)

TIP: Human-in-the-Loop

Sampling 핸들러에서 LLM을 바로 호출하지 않고, 사용자에게 확인을 요청할 수 있다. 예를 들어 "서버가 다음 내용으로 LLM 호출을 요청했다. 허용할까?" 라는 프롬프트를 띄워 사용자가 직접 승인/거부를 결정하게 하면, 보안이 중요한 환경에서 서버의 LLM 남용을 방지할 수 있다.

8.6 다중 서버 연결

import asyncio
from contextlib import AsyncExitStack
from mcp.client.session import ClientSession
from mcp.client.stdio import StdioServerParameters, stdio_client

# 서버 설정 정의
SERVERS = {
    "filesystem": StdioServerParameters(
        command="npx",
        args=["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
    ),
    "database": StdioServerParameters(
        command="uv",
        args=["run", "db_server.py"],
    ),
    "web-search": StdioServerParameters(
        command="uv",
        args=["run", "search_server.py"],
    ),
}


async def connect_all_servers():
    """여러 MCP 서버에 동시 연결하고 도구를 통합한다"""
    sessions: dict[str, ClientSession] = {}
    all_tools = []              # 모든 서버의 도구 통합 목록
    tool_to_session = {}        # 도구 이름 → 세션 매핑

    async with AsyncExitStack() as stack:
        # 각 서버에 연결
        for name, params in SERVERS.items():
            # stdio 연결 설정
            read, write = await stack.enter_async_context(
                stdio_client(params)
            )
            session = await stack.enter_async_context(
                ClientSession(read, write)
            )
            await session.initialize()
            sessions[name] = session

            # 도구 수집
            tools = await session.list_tools()
            for tool in tools.tools:
                all_tools.append(tool)
                tool_to_session[tool.name] = session
                print(f"  [{name}] {tool.name}: {tool.description}")

        print(f"\n총 {len(all_tools)}개 도구 발견")

        # 이제 LLM이 선택한 도구를 올바른 세션으로 라우팅
        async def call_tool(tool_name: str, args: dict):
            session = tool_to_session.get(tool_name)
            if session is None:
                raise ValueError(f"알 수 없는 도구: {tool_name}")
            return await session.call_tool(tool_name, args)

        # 에이전트 루프에서 call_tool 사용
        # result = await call_tool("read_file", {"path": "/tmp/test.txt"})

asyncio.run(connect_all_servers())
WARNING: 도구 이름 충돌

여러 서버가 같은 이름의 도구를 제공할 수 있다. 이 경우 tool_to_session 딕셔너리에서 나중에 등록된 도구가 이전 것을 덮어쓴다. 프로덕션 환경에서는 도구 이름에 서버 접두사를 붙이거나 (예: fs__read_file), 충돌을 감지하여 경고하는 로직을 추가해야 한다.

8.7 에러 핸들링과 재연결

프로덕션 환경에서는 네트워크 끊김, 서버 재시작, 타임아웃 등 다양한 오류 상황에 대비해야 한다. MCP 클라이언트에서 자주 발생하는 에러 유형과 대처 패턴을 살펴본다.

주요 에러 유형

에러원인대처
ConnectionError 서버 프로세스 종료, 네트워크 끊김 재연결 시도
TimeoutError 도구 실행 시간 초과 타임아웃 설정 조정, 재시도
McpError JSON-RPC 에러 (잘못된 파라미터 등) 에러 코드에 따라 분기 처리
도구 실행 에러 도구 내부에서 예외 발생 (isError: true) 에러 메시지를 LLM에 전달하여 대안 모색

재시도 패턴

import asyncio
import logging
from mcp.client.session import ClientSession
from mcp.client.stdio import StdioServerParameters, stdio_client

logger = logging.getLogger(__name__)

async def call_tool_with_retry(
    session: ClientSession,
    tool_name: str,
    args: dict,
    max_retries: int = 3,
    base_delay: float = 1.0,
):
    """도구 호출을 자동 재시도하는 래퍼"""
    for attempt in range(max_retries):
        try:
            result = await asyncio.wait_for(
                session.call_tool(tool_name, args),
                timeout=30.0,  # 30초 타임아웃
            )
            # isError 체크
            if result.isError:
                logger.warning(
                    f"도구 '{tool_name}' 실행 에러: {result.content[0].text}"
                )
            return result

        except asyncio.TimeoutError:
            logger.warning(
                f"도구 '{tool_name}' 타임아웃 "
                f"(시도 {attempt + 1}/{max_retries})"
            )
        except ConnectionError as e:
            logger.error(f"연결 끊김: {e}")
            raise  # 연결 에러는 재시도가 의미 없으므로 상위로 전파
        except Exception as e:
            logger.error(
                f"도구 '{tool_name}' 호출 실패: {e} "
                f"(시도 {attempt + 1}/{max_retries})"
            )

        if attempt < max_retries - 1:
            delay = base_delay * (2 ** attempt)  # 지수 백오프
            logger.info(f"{delay}초 후 재시도...")
            await asyncio.sleep(delay)

    raise RuntimeError(f"도구 '{tool_name}' 호출이 {max_retries}회 모두 실패했다")

연결 유지 및 재연결

async def resilient_connection(server_params, max_reconnects=5):
    """연결 끊김 시 자동 재연결하는 패턴"""
    reconnect_count = 0

    while reconnect_count < max_reconnects:
        try:
            async with stdio_client(server_params) as (read, write):
                async with ClientSession(read, write) as session:
                    await session.initialize()
                    logger.info("서버 연결 성공")
                    reconnect_count = 0  # 성공 시 카운터 초기화

                    # 여기서 에이전트 루프 실행
                    await run_agent(session)

        except ConnectionError:
            reconnect_count += 1
            delay = min(2 ** reconnect_count, 60)  # 최대 60초
            logger.warning(
                f"연결 끊김. {delay}초 후 재연결 "
                f"({reconnect_count}/{max_reconnects})"
            )
            await asyncio.sleep(delay)
        except KeyboardInterrupt:
            logger.info("사용자에 의해 종료")
            break

    if reconnect_count >= max_reconnects:
        logger.error("최대 재연결 횟수 초과. 프로그램 종료.")

8.8 Notification 핸들링

MCP 서버는 도구/리소스/프롬프트 목록이 변경되었을 때, 또는 로그 메시지를 보낼 때 Notification을 클라이언트에 발송한다. 클라이언트는 이 알림을 수신하여 자신의 상태를 갱신할 수 있다.

주요 Notification 유형

메서드발생 시점클라이언트 대응
notifications/tools/list_changed 서버의 도구 목록이 변경됨 list_tools() 재호출하여 목록 갱신
notifications/resources/list_changed 리소스 목록이 변경됨 list_resources() 재호출
notifications/prompts/list_changed 프롬프트 목록이 변경됨 list_prompts() 재호출
notifications/message 서버가 로그 메시지 발송 로그 레벨에 따라 표시/기록

Notification 핸들러 등록

from mcp.client.session import ClientSession
from mcp.types import (
    LoggingMessageNotificationParams,
    ToolListChangedNotification,
)

# 도구 목록 캐시
cached_tools = []

async def setup_notifications(session: ClientSession):
    """Notification 핸들러를 등록한다"""

    # 도구 목록 변경 알림 처리
    async def on_tools_changed(notification):
        nonlocal cached_tools
        print("[알림] 도구 목록이 변경되었다. 다시 조회 중...")
        tools = await session.list_tools()
        cached_tools = tools.tools
        print(f"[알림] 갱신된 도구: {[t.name for t in cached_tools]}")

    # 로그 메시지 수신
    async def on_log_message(params: LoggingMessageNotificationParams):
        level = params.level  # "debug", "info", "warning", "error"
        data = params.data
        print(f"[서버 로그] [{level}] {data}")

    # 핸들러 등록
    session.on_notification(
        "notifications/tools/list_changed",
        on_tools_changed,
    )
    session.on_notification(
        "notifications/message",
        on_log_message,
    )

Notification은 응답을 기대하지 않는 단방향 메시지이다 (id가 없다). 서버는 Notification을 보낸 후 즉시 다음 작업을 계속한다. 클라이언트가 이를 처리하지 않아도 프로토콜 오류가 발생하지 않지만, 도구 목록이 변경되었는데 갱신하지 않으면 LLM이 존재하지 않는 도구를 호출하려 할 수 있다.

8.9 실전 예제: 완전한 AI 에이전트

지금까지 학습한 모든 내용을 결합하여, MCP 클라이언트 + Anthropic API로 동작하는 완전한 AI 에이전트를 구현한다. 이 에이전트는 다음 기능을 갖춘다.

  • 여러 MCP 서버에 동시 연결
  • 모든 서버의 도구를 자동 발견하여 LLM에 통합 제공
  • LLM 에이전트 루프로 자연어 명령 처리
  • 에러 핸들링과 재시도
  • 대화형 인터페이스

에이전트 코드 (agent.py)

"""완전한 MCP AI 에이전트 — 다중 서버 연결 + Anthropic API 에이전트 루프"""
import asyncio
import sys
from contextlib import AsyncExitStack

import anthropic
from mcp.client.session import ClientSession
from mcp.client.stdio import StdioServerParameters, stdio_client


class MCPAgent:
    """MCP 서버들과 Anthropic LLM을 결합한 AI 에이전트"""

    def __init__(self):
        self.sessions: dict[str, ClientSession] = {}
        self.tool_to_session: dict[str, ClientSession] = {}
        self.all_tools = []
        self.anthropic_tools = []
        self.anthropic = anthropic.Anthropic()
        self.exit_stack = AsyncExitStack()
        self.messages = []

    async def connect(self, servers: dict[str, StdioServerParameters]):
        """여러 MCP 서버에 동시 연결한다"""
        for name, params in servers.items():
            print(f"  [{name}] 연결 중...", file=sys.stderr)
            read, write = await self.exit_stack.enter_async_context(
                stdio_client(params)
            )
            session = await self.exit_stack.enter_async_context(
                ClientSession(read, write)
            )
            await session.initialize()
            self.sessions[name] = session

            # 도구 수집
            tools = await session.list_tools()
            for tool in tools.tools:
                self.all_tools.append(tool)
                self.tool_to_session[tool.name] = session
                self.anthropic_tools.append({
                    "name": tool.name,
                    "description": tool.description or "",
                    "input_schema": tool.inputSchema,
                })
            print(
                f"  [{name}] 연결 완료 "
                f"({len(tools.tools)}개 도구)",
                file=sys.stderr,
            )

        print(
            f"\n총 {len(self.all_tools)}개 도구 사용 가능\n",
            file=sys.stderr,
        )

    async def call_tool(self, tool_name: str, args: dict):
        """도구 이름으로 올바른 세션을 찾아 호출한다"""
        session = self.tool_to_session.get(tool_name)
        if session is None:
            raise ValueError(f"알 수 없는 도구: {tool_name}")

        result = await asyncio.wait_for(
            session.call_tool(tool_name, args),
            timeout=30.0,
        )
        return result

    async def chat(self, user_message: str) -> str:
        """사용자 메시지를 처리하고 에이전트 루프를 실행한다"""
        self.messages.append({
            "role": "user",
            "content": user_message,
        })

        while True:
            # LLM 호출
            response = self.anthropic.messages.create(
                model="claude-sonnet-4-20250514",
                max_tokens=4096,
                system=(
                    "너는 유능한 AI 어시스턴트이다. "
                    "사용 가능한 도구를 활용하여 사용자의 요청을 처리한다. "
                    "필요하면 여러 도구를 순차적으로 사용할 수 있다."
                ),
                tools=self.anthropic_tools,
                messages=self.messages,
            )

            self.messages.append({
                "role": "assistant",
                "content": response.content,
            })

            # tool_use가 없으면 루프 종료
            if response.stop_reason != "tool_use":
                final_text = ""
                for block in response.content:
                    if hasattr(block, "text"):
                        final_text += block.text
                return final_text

            # tool_use 블록 처리
            tool_results = []
            for block in response.content:
                if block.type != "tool_use":
                    continue

                print(
                    f"  [도구] {block.name}({block.input})",
                    file=sys.stderr,
                )

                try:
                    result = await self.call_tool(block.name, block.input)
                    result_text = (
                        result.content[0].text if result.content else ""
                    )
                    is_error = getattr(result, "isError", False)
                except Exception as e:
                    result_text = f"도구 호출 실패: {e}"
                    is_error = True

                if is_error:
                    print(
                        f"  [에러] {result_text}",
                        file=sys.stderr,
                    )
                else:
                    print(
                        f"  [결과] {result_text[:100]}...",
                        file=sys.stderr,
                    )

                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result_text,
                    "is_error": is_error,
                })

            self.messages.append({
                "role": "user",
                "content": tool_results,
            })

    def reset(self):
        """대화 이력을 초기화한다"""
        self.messages = []

    async def close(self):
        """모든 연결을 종료한다"""
        await self.exit_stack.aclose()


async def main():
    # 연결할 MCP 서버 목록
    servers = {
        "math": StdioServerParameters(
            command="uv",
            args=["run", "math_server.py"],
        ),
        # 필요에 따라 서버 추가
        # "filesystem": StdioServerParameters(
        #     command="npx",
        #     args=["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
        # ),
    }

    agent = MCPAgent()

    try:
        # 서버 연결
        print("=== MCP AI 에이전트 ===\n", file=sys.stderr)
        await agent.connect(servers)

        # 대화형 루프
        print("질문을 입력한다 (종료: quit)\n")
        while True:
            try:
                user_input = input("사용자> ").strip()
            except EOFError:
                break

            if not user_input:
                continue
            if user_input.lower() in ("quit", "exit", "q"):
                break
            if user_input.lower() == "reset":
                agent.reset()
                print("[대화 초기화]\n")
                continue

            response = await agent.chat(user_input)
            print(f"\n에이전트> {response}\n")

    finally:
        await agent.close()
        print("\n에이전트 종료.", file=sys.stderr)


if __name__ == "__main__":
    try:
        asyncio.run(main())
    except KeyboardInterrupt:
        print("\n\n프로그램이 종료되었다.")

함께 사용할 서버 코드 (math_server.py)

"""수학 도구를 제공하는 MCP 서버"""
import math
from datetime import datetime
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("math-server")

@mcp.tool()
def add(a: float, b: float) -> str:
    """두 수를 더한다"""
    return str(a + b)

@mcp.tool()
def multiply(a: float, b: float) -> str:
    """두 수를 곱한다"""
    return str(a * b)

@mcp.tool()
def sqrt(number: float) -> str:
    """제곱근을 계산한다"""
    if number < 0:
        return "오류: 음수의 제곱근은 계산할 수 없다"
    return str(math.sqrt(number))

@mcp.tool()
def power(base: float, exponent: float) -> str:
    """거듭제곱을 계산한다"""
    return str(base ** exponent)

@mcp.tool()
def current_time() -> str:
    """현재 시각을 반환한다"""
    return datetime.now().strftime("%Y년 %m월 %d일 %H시 %M분 %S초")

if __name__ == "__main__":
    mcp.run()

8.10 다음 단계

이 장에서는 MCP 클라이언트의 구현부터 LLM 에이전트 통합까지 폭넓게 다루었다. 핵심 정리는 다음과 같다.

  • 기본 연결 — stdio_client와 streamable_http_client로 서버에 연결하고, initialize()로 세션을 시작한다
  • 에이전트 루프 — LLM의 tool_use 응답을 감지하여 MCP 도구를 호출하고, 결과를 LLM에 피드백하는 루프가 MCP의 핵심 사용 패턴이다
  • Sampling — 서버가 클라이언트의 LLM을 역으로 호출하여 중간 판단이나 텍스트 생성을 요청하는 고급 기능이다
  • 다중 서버 — 여러 전문 MCP 서버를 동시에 연결하고, 모든 도구를 하나의 LLM에 통합 제공한다
  • 에러 핸들링 — 타임아웃, 재시도, 재연결 패턴으로 안정적인 운영을 보장한다
  • Notification — 서버의 상태 변경 알림을 수신하여 도구 목록을 동적으로 갱신한다

다음 장에서는 Structured Output을 다룬다. MCP 서버에서 Pydantic 모델을 활용하여 타입이 보장된 구조화된 데이터를 반환하는 방법을 학습한다.
