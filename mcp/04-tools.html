Tools 심화

파라미터 타입, 비동기 처리, 에러 핸들링, 그리고 보안까지 도구의 모든 것을 다룬다

Tools 개요

핵심 개념: Tool이란 무엇인가

MCP에서 Tool은 LLM이 직접 호출할 수 있는 서버 측 함수이다. Resources가 읽기 전용 데이터를 제공하는 것과 달리, Tool은 부수 효과(side effect)를 일으킬 수 있다. 외부 API 호출, 파일 쓰기, 데이터베이스 변경 등 실제 세계에 영향을 미치는 작업을 수행한다.

프로토콜 수준에서 Tool은 JSON Schema로 정의된 입력 스키마를 갖는다. LLM은 이 스키마를 보고 어떤 파라미터를 어떤 타입으로 넘겨야 하는지 판단한다. 전체 흐름은 다음과 같다. 

sequenceDiagram
    participant H as Host (LLM 앱)
    participant C as MCP Client
    participant S as MCP Server

    H->>C: 사용 가능한 도구 조회
    C->>S: tools/list
    S-->>C: Tool[] (이름, 설명, inputSchema)
    C-->>H: 도구 목록 전달
    Note over H: LLM이 도구 호출 결정
    H->>C: Tool 호출 요청
    C->>S: tools/call {name, arguments}
    S-->>C: CallToolResult {content[], isError?}
    C-->>H: 결과 전달

tools/list 응답에 포함되는 도구 정보의 구조를 살펴보자.

← tools/list 응답 (단일 도구 예시)
{
  "tools": [
    {
      "name": "calculate",
      "description": "두 수의 사칙연산을 수행한다",
      "inputSchema": {
        "type": "object",
        "properties": {
          "a": { "type": "number", "description": "첫 번째 수" },
          "b": { "type": "number", "description": "두 번째 수" },
          "op": {
            "type": "string",
            "enum": ["add", "sub", "mul", "div"],
            "description": "연산자"
          }
        },
        "required": ["a", "b", "op"]
      }
    }
  ]
}

FastMCP는 Python 함수의 타입 힌트와 docstring을 분석하여 위의 JSON Schema를 자동 생성한다. 개발자가 직접 스키마를 작성할 필요가 없다.

파라미터 타입 심화

FastMCP는 Python 타입 힌트를 JSON Schema로 변환한다. 기본 타입부터 복합 타입, Pydantic 모델까지 다양한 타입을 지원한다.

기본 타입과 Optional

from typing import Optional

@mcp.tool()
def search_files(
    directory: str,                # 필수, string
    pattern: str,                  # 필수, string
    recursive: bool = True,        # 선택, boolean (기본값: true)
    max_results: int = 100,        # 선택, integer (기본값: 100)
    min_size: Optional[float] = None  # 선택, number | null
) -> list[str]:
    """디렉토리에서 파일을 검색한다

    Args:
        directory: 검색할 디렉토리 경로
        pattern: 파일 패턴 (예: "*.py")
        recursive: 하위 디렉토리 포함 여부
        max_results: 최대 결과 수
        min_size: 최소 파일 크기 (bytes). None이면 제한 없음
    """
    import glob
    path = f"{directory}/**/{pattern}" if recursive else f"{directory}/{pattern}"
    files = glob.glob(path, recursive=recursive)
    return files[:max_results]

list와 dict

@mcp.tool()
def process_records(
    records: list[dict],          # 딕셔너리 배열
    tags: list[str] = [],         # 문자열 배열 (기본값: 빈 배열)
    metadata: dict = {}           # 임의의 딕셔너리
) -> dict:
    """레코드를 처리하고 요약을 반환한다

    Args:
        records: 처리할 레코드 목록. 각 레코드는 {"name": str, "value": int} 형태이다.
        tags: 필터링할 태그 목록
        metadata: 추가 메타데이터
    """
    values = [r.get("value", 0) for r in records]
    return {
        "total": len(records),
        "sum": sum(values),
        "average": sum(values) / len(records) if records else 0,
        "tags_applied": tags,
    }

Pydantic BaseModel 입력

Deep Dive: Pydantic 모델로 구조화된 입력 정의

복잡한 입력 구조가 필요한 경우 pydantic.BaseModel을 사용하면 강력한 타입 검증과 명확한 스키마를 동시에 얻을 수 있다. FastMCP는 Pydantic 모델을 자동으로 JSON Schema로 변환하며, 입력 데이터의 유효성 검사도 자동으로 수행한다. 

from pydantic import BaseModel, Field
from enum import Enum

class Priority(str, Enum):
    LOW = "low"
    MEDIUM = "medium"
    HIGH = "high"

class TaskInput(BaseModel):
    """작업 생성에 필요한 입력 데이터"""
    title: str = Field(description="작업 제목")
    description: str = Field(default="", description="작업 상세 설명")
    priority: Priority = Field(default=Priority.MEDIUM, description="우선순위")
    assignees: list[str] = Field(default=[], description="담당자 목록")
    labels: dict[str, str] = Field(default={}, description="레이블 (key-value)")

@mcp.tool()
def create_task(task: TaskInput) -> dict:
    """새 작업을 생성한다

    Args:
        task: 작업 정보
    """
    return {
        "id": "TASK-001",
        "title": task.title,
        "description": task.description,
        "priority": task.priority.value,
        "assignees": task.assignees,
        "labels": task.labels,
        "status": "created",
    }
INFO

Pydantic의 Field(description=...)를 사용하면 각 필드별로 상세한 설명을 JSON Schema에 포함시킬 수 있다. LLM이 필드의 용도를 더 정확히 파악하게 되므로, 복잡한 입력일수록 적극적으로 활용하는 것이 좋다.

반환 타입 심화

도구 함수는 다양한 타입을 반환할 수 있다. FastMCP는 반환값을 MCP 프로토콜의 CallToolResult로 자동 변환한다.

문자열 반환

가장 기본적인 형태이다. 반환된 문자열이 TextContent로 변환된다.

@mcp.tool()
def greet(name: str) -> str:
    """인사 메시지를 반환한다"""
    return f"안녕, {name}!"
# -> CallToolResult(content=[TextContent(text="안녕, 홍길동!")])

dict / list 반환

딕셔너리나 리스트를 반환하면 자동으로 JSON 문자열로 직렬화된 TextContent가 된다.

@mcp.tool()
def get_stats() -> dict:
    """서버 통계를 반환한다"""
    return {"uptime": 3600, "requests": 150, "errors": 2}

이미지 반환 (base64)

Deep Dive: 이미지와 바이너리 데이터 반환

도구에서 이미지를 반환하려면 Image 헬퍼 클래스를 사용한다. 파일 경로, URL, 또는 raw bytes에서 이미지를 생성할 수 있으며, MCP 프로토콜은 이를 base64 인코딩된 ImageContent로 전송한다. 

from mcp.server.fastmcp import FastMCP, Image

mcp = FastMCP("image-tools")

@mcp.tool()
def create_chart(data: list[int], title: str = "Chart") -> Image:
    """데이터를 차트 이미지로 변환한다

    Args:
        data: 차트에 표시할 데이터 값 목록
        title: 차트 제목
    """
    import matplotlib.pyplot as plt
    import io

    fig, ax = plt.subplots()
    ax.bar(range(len(data)), data)
    ax.set_title(title)

    buf = io.BytesIO()
    fig.savefig(buf, format="png")
    buf.seek(0)
    plt.close(fig)

    return Image(data=buf.read(), format="png")


@mcp.tool()
def get_thumbnail(image_path: str) -> Image:
    """이미지의 썸네일을 반환한다

    Args:
        image_path: 원본 이미지 경로
    """
    # 파일 경로에서 직접 Image 생성
    return Image(path=image_path)

에러와 isError 플래그

Deep Dive: isError 플래그의 역할

CallToolResult에는 isError 플래그가 있다. 이 플래그가 true이면 LLM은 해당 결과를 "실패"로 인식하고, 재시도하거나 대안을 찾는 등 적절한 후속 행동을 취한다. 일반적인 Python 예외는 SDK가 자동으로 isError: true로 변환한다.

예외 발생 (자동 isError)
@mcp.tool()
def divide(a: float, b: float) -> float:
    """나눗셈을 수행한다"""
    if b == 0:
        raise ValueError("0으로 나눌 수 없다")
    return a / b
# -> isError: true, content: "0으로 나눌 수 없다"
ToolError로 명시적 에러
from mcp.server.fastmcp.exceptions import ToolError

@mcp.tool()
def delete_file(path: str) -> str:
    """파일을 삭제한다"""
    if not os.path.exists(path):
        raise ToolError(
            f"파일이 존재하지 않는다: {path}"
        )
    os.remove(path)
    return f"삭제 완료: {path}"
# -> isError: true
WARNING

일반 예외(ValueError, RuntimeError 등)와 ToolError 모두 isError: true를 설정한다. 차이점은 ToolError가 의도된 에러 응답임을 코드에서 명확히 드러낸다는 것이다. 내부 시스템 정보가 예외 메시지에 노출되지 않도록 주의하라.

비동기 도구

네트워크 요청, 파일 I/O 등 시간이 오래 걸리는 작업은 async/await를 사용하여 비동기 도구로 정의한다. FastMCP는 동기 함수와 비동기 함수를 모두 지원하지만, I/O가 포함된 도구는 비동기로 작성하는 것이 성능에 유리하다.

기본 비동기 도구

import httpx

@mcp.tool()
async def fetch_url(url: str, timeout: int = 30) -> str:
    """URL의 콘텐츠를 가져온다

    Args:
        url: 가져올 웹 페이지의 URL
        timeout: 타임아웃 (초)
    """
    async with httpx.AsyncClient(timeout=timeout) as client:
        response = await client.get(url)
        response.raise_for_status()
        return response.text[:5000]  # 처음 5000자만 반환

asyncio.gather로 병렬 처리

여러 비동기 작업을 동시에 실행하면 전체 소요 시간을 크게 줄일 수 있다.

import asyncio
import httpx

@mcp.tool()
async def fetch_multiple(urls: list[str]) -> dict[str, str]:
    """여러 URL의 콘텐츠를 동시에 가져온다

    Args:
        urls: 가져올 URL 목록 (최대 10개)
    """
    if len(urls) > 10:
        raise ValueError("한 번에 최대 10개까지만 요청할 수 있다")

    async with httpx.AsyncClient(timeout=15) as client:
        tasks = [client.get(url) for url in urls]
        responses = await asyncio.gather(*tasks, return_exceptions=True)

    results = {}
    for url, resp in zip(urls, responses):
        if isinstance(resp, Exception):
            results[url] = f"에러: {type(resp).__name__}: {resp}"
        else:
            results[url] = resp.text[:1000]
    return results
INFO

asyncio.gather(*tasks, return_exceptions=True)를 사용하면 하나의 요청이 실패해도 나머지 요청은 정상적으로 완료된다. return_exceptions=False(기본값)이면 첫 번째 예외가 발생한 시점에 전체가 취소된다.

Context 활용

Context 객체는 도구 실행 중에 서버-클라이언트 간 상호작용을 가능하게 한다. 파라미터에 ctx: Context를 추가하면 SDK가 자동으로 주입한다. 이 파라미터는 LLM의 입력 스키마에 노출되지 않는다. 

from mcp.server.fastmcp import FastMCP, Context

mcp = FastMCP("Context Demo")

@mcp.tool()
async def process_batch(ctx: Context, items: list[str]) -> str:
    """배치 작업을 수행한다

    Args:
        items: 처리할 항목 목록
    """
    total = len(items)

    for i, item in enumerate(items):
        # 로깅: 클라이언트에 로그 메시지를 전송한다
        await ctx.info(f"처리 중: {item}")

        # 진행률 보고: (현재 진행, 전체)
        await ctx.report_progress(i + 1, total)

        # 실제 작업 시뮬레이션
        await asyncio.sleep(0.3)

    return f"완료: {total}개 항목 처리됨"

Context 주요 메서드

메서드설명사용 예시
ctx.debug(msg)디버그 로그 전송await ctx.debug("상세 정보")
ctx.info(msg)정보 로그 전송await ctx.info("작업 시작")
ctx.warning(msg)경고 로그 전송await ctx.warning("주의 필요")

도구에서 리소스 접근

ctx.read_resource()를 사용하면 같은 서버에 등록된 리소스를 도구 안에서 읽을 수 있다.

@mcp.resource("config://db")
def get_db_config() -> str:
    return json.dumps({"host": "localhost", "port": 5432})

@mcp.tool()
async def query_database(ctx: Context, sql: str) -> str:
    """SQL 쿼리를 실행한다

    Args:
        sql: 실행할 SQL 쿼리
    """
    # 리소스에서 DB 설정을 읽는다
    config_data = await ctx.read_resource("config://db")
    config = json.loads(config_data[0].content)

    await ctx.info(f"DB 연결: {config['host']}:{config['port']}")
    # ... 쿼리 실행
    return "쿼리 결과"
TIP

ctx: Context는 docstring의 Args 섹션에 포함시키지 않아도 된다. SDK가 내부적으로 주입하는 파라미터이므로, LLM에게 설명할 필요가 없다.

에러 처리 심화

도구의 에러 처리는 LLM 경험에 직접적인 영향을 미친다. 에러 메시지가 명확하면 LLM이 스스로 수정하거나 사용자에게 적절한 안내를 제공할 수 있다.

try-except 패턴

import json

@mcp.tool()
def parse_and_validate(raw_json: str) -> dict:
    """JSON을 파싱하고 필수 필드를 검증한다

    Args:
        raw_json: 파싱할 JSON 문자열
    """
    # 1단계: JSON 파싱
    try:
        data = json.loads(raw_json)
    except json.JSONDecodeError as e:
        raise ValueError(
            f"유효하지 않은 JSON이다. "
            f"위치 {e.pos}에서 오류 발생: {e.msg}"
        )

    # 2단계: 필수 필드 검증
    required = ["name", "email"]
    missing = [f for f in required if f not in data]
    if missing:
        raise ValueError(
            f"필수 필드가 누락되었다: {', '.join(missing)}. "
            f"제공된 필드: {', '.join(data.keys())}"
        )

    return data

ToolError 사용

ToolError는 FastMCP가 제공하는 전용 예외 클래스이다. 일반 예외와 동일하게 isError: true를 설정하지만, 코드의 의도를 더 명확히 표현한다. 

from mcp.server.fastmcp.exceptions import ToolError

@mcp.tool()
async def send_email(to: str, subject: str, body: str) -> str:
    """이메일을 전송한다

    Args:
        to: 수신자 이메일 주소
        subject: 제목
        body: 본문
    """
    # 이메일 형식 검증
    if "@" not in to or "." not in to.split("@")[-1]:
        raise ToolError(
            f"유효하지 않은 이메일 주소이다: {to}. "
            f"올바른 형식: user@domain.com"
        )

    try:
        # 이메일 전송 로직
        await _send_smtp(to, subject, body)
        return f"이메일 전송 완료: {to}"
    except ConnectionError:
        raise ToolError("메일 서버에 연결할 수 없다. 나중에 다시 시도하라.")
    except Exception:
        # 내부 오류 상세 정보를 LLM에게 노출하지 않는다
        raise ToolError("이메일 전송에 실패했다. 관리자에게 문의하라.")
WARNING

예외 메시지에 서버 내부 경로, DB 비밀번호, API 키 등이 포함되지 않도록 주의하라. except Exception as e: raise ToolError(str(e)) 패턴은 위험하다. 내부 예외를 로깅한 뒤, 사용자에게는 일반적인 메시지를 반환하는 것이 안전하다.

보안 심화

핵심 개념: MCP 도구의 보안 원칙

MCP 도구는 LLM이 호출하기 때문에, LLM의 판단에 의존하여 보안을 유지할 수 없다. 도구의 서버 측 코드에서 반드시 입력을 검증하고, 접근 권한을 확인하고, 위험한 작업을 차단해야 한다.

Path Traversal 방어

파일 시스템에 접근하는 도구는 ../를 이용한 경로 탐색 공격에 특히 취약하다.

from pathlib import Path

ALLOWED_BASE = Path("/home/user/workspace").resolve()

def _validate_path(user_path: str) -> Path:
    """사용자 입력 경로를 검증하여 안전한 Path를 반환한다"""
    # resolve()로 심볼릭 링크와 ../를 모두 해석한다
    target = (ALLOWED_BASE / user_path).resolve()

    # 허용된 디렉토리 안에 있는지 확인
    if not str(target).startswith(str(ALLOWED_BASE)):
        raise PermissionError(
            f"접근이 거부되었다. 허용된 경로: {ALLOWED_BASE}"
        )
    return target

@mcp.tool()
def read_project_file(file_path: str) -> str:
    """프로젝트 파일을 읽는다

    Args:
        file_path: 파일 경로 (프로젝트 루트 기준 상대 경로)
    """
    safe_path = _validate_path(file_path)

    if not safe_path.exists():
        raise FileNotFoundError(f"파일을 찾을 수 없다: {file_path}")
    if not safe_path.is_file():
        raise ValueError(f"파일이 아닌 경로이다: {file_path}")

    return safe_path.read_text(encoding="utf-8")

입력 검증과 허용 목록

import re

# 허용된 확장자 목록
ALLOWED_EXTENSIONS = {".py", ".js", ".ts", ".json", ".yaml", ".yml", ".md", ".txt"}
# 허용된 도메인 목록
ALLOWED_DOMAINS = {"api.example.com", "data.example.com"}

@mcp.tool()
def read_source_file(file_path: str) -> str:
    """소스 파일을 읽는다 (허용된 확장자만)

    Args:
        file_path: 파일 경로
    """
    safe_path = _validate_path(file_path)
    ext = safe_path.suffix.lower()

    if ext not in ALLOWED_EXTENSIONS:
        raise PermissionError(
            f"허용되지 않은 파일 확장자이다: {ext}. "
            f"허용 목록: {', '.join(sorted(ALLOWED_EXTENSIONS))}"
        )

    return safe_path.read_text(encoding="utf-8")

@mcp.tool()
async def call_api(url: str) -> str:
    """외부 API를 호출한다 (허용된 도메인만)

    Args:
        url: API URL
    """
    from urllib.parse import urlparse
    parsed = urlparse(url)

    if parsed.hostname not in ALLOWED_DOMAINS:
        raise PermissionError(
            f"허용되지 않은 도메인이다: {parsed.hostname}. "
            f"허용 목록: {', '.join(sorted(ALLOWED_DOMAINS))}"
        )

    async with httpx.AsyncClient() as client:
        resp = await client.get(url)
        resp.raise_for_status()
        return resp.text

명령어 인젝션 방어

import subprocess
import shlex

# 허용된 명령어 목록
ALLOWED_COMMANDS = {"ls", "cat", "grep", "wc", "head", "tail"}

@mcp.tool()
def run_safe_command(command: str, args: list[str] = []) -> str:
    """허용된 시스템 명령어를 실행한다

    Args:
        command: 실행할 명령어 (ls, cat, grep 등)
        args: 명령어 인자 목록
    """
    # 허용 목록 확인
    if command not in ALLOWED_COMMANDS:
        raise PermissionError(
            f"허용되지 않은 명령어이다: {command}. "
            f"허용 목록: {', '.join(sorted(ALLOWED_COMMANDS))}"
        )

    # 셸 메타문자 차단: 각 인자를 개별적으로 전달한다
    # shell=True는 절대 사용하지 않는다!
    safe_args = [shlex.quote(a) for a in args]

    result = subprocess.run(
        [command] + args,      # shell=False (기본값)
        capture_output=True,
        text=True,
        timeout=10,            # 타임아웃 설정
    )

    if result.returncode != 0:
        return f"에러 (코드 {result.returncode}):\n{result.stderr}"
    return result.stdout
WARNING

subprocess.run(shell=True)는 절대 사용하지 마라. LLM이 입력한 문자열이 셸에서 그대로 실행되면 ; rm -rf / 같은 명령어 인젝션이 가능하다. 반드시 shell=False와 인자 리스트를 사용하고, 허용 목록으로 명령어를 제한하라.

pytest로 단위 테스트

도구 함수는 일반 Python 함수이므로 직접 호출하여 테스트할 수 있다. 비동기 도구는 pytest-asyncio를 사용한다. 

# test_tools.py
import pytest
from server import mcp

@pytest.mark.asyncio
async def test_tool_call_via_mcp():
    """MCP 프로토콜을 통한 도구 호출 테스트"""
    from mcp.client.session import ClientSession
    from mcp.server.lowlevel import Server

    # in-memory 세션으로 서버와 클라이언트를 연결한다
    async with ClientSession(*mcp._mcp_server.create_streams()) as session:
        await session.initialize()

        # 도구 목록 확인
        tools = await session.list_tools()
        tool_names = [t.name for t in tools.tools]
        assert "calculate" in tool_names

        # 도구 호출
        result = await session.call_tool("calculate", {
            "a": 10, "b": 3, "op": "add"
        })
        assert result.content[0].text == "13"


@pytest.mark.asyncio
async def test_tool_error_handling():
    """도구 에러 처리 테스트"""
    async with ClientSession(*mcp._mcp_server.create_streams()) as session:
        await session.initialize()

        result = await session.call_tool("divide", {
            "a": 10, "b": 0
        })
        assert result.isError is True
        assert "0으로 나눌 수 없다" in result.content[0].text

실전 예제: 종합 서버

지금까지 다룬 모든 개념을 결합한 파일 관리 MCP 서버를 구축한다. 보안, 에러 처리, Context 활용, Pydantic 입력, 이미지 반환을 모두 포함한다. 

"""종합 파일 관리 MCP 서버

보안, 에러 처리, Context 활용, 다양한 반환 타입을 종합 시연한다.
"""
import os
import json
import glob
import asyncio
from pathlib import Path
from datetime import datetime
from enum import Enum
from typing import Optional

from pydantic import BaseModel, Field
from mcp.server.fastmcp import FastMCP, Context, Image
from mcp.server.fastmcp.exceptions import ToolError

mcp = FastMCP("File Manager Pro")

# ── 설정 ────────────────────────────────────────────
WORKSPACE = Path(os.environ.get("WORKSPACE", "~/workspace")).expanduser().resolve()
MAX_FILE_SIZE = 5 * 1024 * 1024  # 5MB
ALLOWED_EXTENSIONS = {".py", ".js", ".ts", ".json", ".yaml", ".yml", ".md", ".txt", ".html", ".css"}


# ── 헬퍼 ────────────────────────────────────────────
def _safe_path(user_path: str) -> Path:
    """경로를 검증하고 안전한 Path를 반환한다"""
    target = (WORKSPACE / user_path).resolve()
    if not str(target).startswith(str(WORKSPACE)):
        raise ToolError(f"접근이 거부되었다. 허용 경로: {WORKSPACE}")
    return target


# ── Pydantic 모델 ───────────────────────────────────
class SearchOptions(BaseModel):
    pattern: str = Field(description="검색 키워드 또는 정규식")
    file_glob: str = Field(default="*.py", description="파일 패턴")
    case_sensitive: bool = Field(default=False, description="대소문자 구분")
    max_results: int = Field(default=50, description="최대 결과 수")

class SortBy(str, Enum):
    NAME = "name"
    SIZE = "size"
    MODIFIED = "modified"


# ── 도구 정의 ───────────────────────────────────────
@mcp.tool(title="파일 목록 조회")
async def list_files(
    ctx: Context,
    directory: str = ".",
    sort_by: SortBy = SortBy.NAME,
    show_hidden: bool = False,
) -> list[dict]:
    """디렉토리의 파일 목록을 반환한다

    Args:
        directory: 디렉토리 경로 (워크스페이스 기준 상대 경로)
        sort_by: 정렬 기준 (name, size, modified)
        show_hidden: 숨김 파일 포함 여부
    """
    safe_dir = _safe_path(directory)
    if not safe_dir.is_dir():
        raise ToolError(f"디렉토리를 찾을 수 없다: {directory}")

    await ctx.info(f"디렉토리 조회: {safe_dir}")

    entries = []
    for item in safe_dir.iterdir():
        if not show_hidden and item.name.startswith("."):
            continue
        stat = item.stat()
        entries.append({
            "name": item.name,
            "type": "directory" if item.is_dir() else "file",
            "size": stat.st_size,
            "modified": datetime.fromtimestamp(stat.st_mtime).isoformat(),
        })

    # 정렬
    sort_key = {
        SortBy.NAME: lambda e: e["name"].lower(),
        SortBy.SIZE: lambda e: e["size"],
        SortBy.MODIFIED: lambda e: e["modified"],
    }
    entries.sort(key=sort_key[sort_by])

    await ctx.info(f"총 {len(entries)}개 항목")
    return entries


@mcp.tool(title="파일 읽기")
async def read_file(
    ctx: Context,
    file_path: str,
    max_lines: Optional[int] = None,
) -> str:
    """파일의 내용을 읽어 반환한다

    Args:
        file_path: 파일 경로 (워크스페이스 기준)
        max_lines: 최대 줄 수 (None이면 전체)
    """
    safe_path = _safe_path(file_path)
    if not safe_path.is_file():
        raise ToolError(f"파일을 찾을 수 없다: {file_path}")

    if safe_path.suffix.lower() not in ALLOWED_EXTENSIONS:
        raise ToolError(
            f"허용되지 않은 파일 형식이다: {safe_path.suffix}. "
            f"허용 목록: {', '.join(sorted(ALLOWED_EXTENSIONS))}"
        )

    size = safe_path.stat().st_size
    if size > MAX_FILE_SIZE:
        raise ToolError(f"파일이 너무 크다: {size:,} bytes (최대 {MAX_FILE_SIZE:,})")

    await ctx.info(f"파일 크기: {size:,} bytes")

    try:
        text = safe_path.read_text(encoding="utf-8")
    except UnicodeDecodeError:
        raise ToolError("텍스트로 읽을 수 없는 파일이다")

    if max_lines is not None:
        lines = text.splitlines(keepends=True)
        if len(lines) > max_lines:
            text = "".join(lines[:max_lines])
            text += f"\n... ({len(lines) - max_lines}줄 생략)"

    return text


@mcp.tool(title="파일 검색")
async def search_files(
    ctx: Context,
    options: SearchOptions,
    directory: str = ".",
) -> list[dict]:
    """파일 내용에서 패턴을 검색한다

    Args:
        options: 검색 옵션 (키워드, 파일 패턴, 대소문자 구분 등)
        directory: 검색 시작 디렉토리
    """
    import re

    safe_dir = _safe_path(directory)
    if not safe_dir.is_dir():
        raise ToolError(f"디렉토리를 찾을 수 없다: {directory}")

    # 파일 목록 수집
    pattern_path = str(safe_dir / "**" / options.file_glob)
    files = glob.glob(pattern_path, recursive=True)
    await ctx.info(f"검색 대상: {len(files)}개 파일")

    # 정규식 컴파일
    flags = 0 if options.case_sensitive else re.IGNORECASE
    try:
        regex = re.compile(options.pattern, flags)
    except re.error as e:
        raise ToolError(f"유효하지 않은 정규식이다: {e}")

    results = []
    for i, fpath in enumerate(files):
        await ctx.report_progress(i + 1, len(files))
        try:
            with open(fpath, "r", encoding="utf-8") as f:
                for line_no, line in enumerate(f, 1):
                    if regex.search(line):
                        results.append({
                            "file": str(Path(fpath).relative_to(WORKSPACE)),
                            "line": line_no,
                            "content": line.strip()[:200],
                        })
                        if len(results) >= options.max_results:
                            await ctx.warning(
                                f"결과가 {options.max_results}개를 초과하여 중단한다"
                            )
                            return results
        except (UnicodeDecodeError, PermissionError):
            continue

    await ctx.info(f"검색 완료: {len(results)}개 결과")
    return results


if __name__ == "__main__":
    mcp.run()

이 서버의 설계 포인트를 정리한다.

  1. 1
    경로 검증 집중화 _safe_path() 헬퍼로 모든 경로 검증을 한 곳에서 처리한다. 중복 제거와 일관성을 동시에 확보한다.
  2. 2
    Pydantic 모델 활용 SearchOptions를 사용하여 복잡한 검색 옵션을 구조화하고, 각 필드에 설명을 부여한다.
  3. 3
    Enum으로 선택지 제한 SortBy Enum으로 정렬 기준을 3가지로 한정한다. LLM이 잘못된 값을 전달하는 것을 원천 차단한다.
  4. 4
    ToolError 활용 사용자에게 명확한 에러 메시지를 전달하되, 시스템 내부 정보는 노출하지 않는다.
  5. 5
    Context 로깅과 진행률 파일 검색처럼 시간이 걸리는 작업에서 진행률을 보고하여 사용자 경험을 개선한다.
