OAuth 인증

OAuth 2.1 인증 흐름, JWT 검증, PKCE 구현, 그리고 외부 IDP 연동까지

왜 인증이 필요한가

로컬에서 stdio 방식으로 실행되는 MCP 서버는 사용자의 프로세스 안에서 동작하므로 별도 인증이 필요 없다. 하지만 원격 MCP 서버(Streamable HTTP)는 네트워크를 통해 노출되므로 다음과 같은 보안 위협에 직면한다:

  • 무단 접근 — 인증 없이 누구나 도구를 호출할 수 있다
  • 데이터 유출 — 민감한 리소스에 비인가 사용자가 접근한다
  • 악의적 도구 호출 — 파일 삭제, DB 수정 등 위험한 도구가 외부에 노출된다
  • 리소스 남용 — Rate limiting 없이 무제한 호출이 가능하다
핵심 개념: 인증(Authentication) vs 인가(Authorization)

인증은 "누구인가?"를 확인하는 과정이다 (e.g., 사용자 로그인, API 키 확인).
인가는 "무엇을 할 수 있는가?"를 결정하는 과정이다 (e.g., admin만 삭제 가능, read 스코프만 허용).
OAuth 2.1은 이 두 가지를 모두 처리하는 프레임워크이다. Authorization Server가 인증을 수행하고, Access Token에 인가 정보(scope)를 담아 발급한다.

OAuth 2.1 핵심 개념

핵심 개념: OAuth 2.1

OAuth 2.1은 OAuth 2.0의 보안 모범 사례(BCP)를 통합한 최신 표준이다. 핵심 변경점은 세 가지이다: (1) PKCE 필수 — 모든 Authorization Code 플로우에 PKCE를 의무화, (2) Implicit 플로우 폐지 — 토큰을 URL fragment에 노출하는 방식을 금지, (3) Refresh Token 회전 — Refresh Token 사용 시 새로운 Refresh Token을 발급하여 탈취 방지.

Scope 기반 접근 제어

Scope는 Access Token이 허용하는 작업의 범위를 정의한다. MCP에서는 도구별로 필요한 scope를 지정할 수 있다: 

# 일반적인 scope 설계

Protected Resource Metadata

MCP 서버는 /.well-known/oauth-protected-resource 엔드포인트에 자신의 보호 메타데이터를 공개한다. 클라이언트는 이 메타데이터를 읽어 Authorization Server의 위치를 파악한다:

→ REQUEST
GET /.well-known/oauth-protected-resource HTTP/1.1
Host: mcp-server.example.com
← RESPONSE
{
  "resource": "https://mcp-server.example.com",
  "authorization_servers": ["https://auth.example.com"],
  "scopes_supported": ["read", "write", "admin"],
  "bearer_methods_supported": ["header"]
}

FastMCP는 AuthSettings에 issuer_url과 resource_server_url을 설정하면 이 메타데이터 엔드포인트를 자동으로 생성한다. 별도 코드를 작성할 필요가 없다.

Resource Server 구축 (서버 측)

MCP 서버를 인증된 Resource Server로 만들려면 두 가지를 구현해야 한다: (1) TokenVerifier — 토큰 검증 로직, (2) AuthSettings — 인증 설정.

TokenVerifier 인터페이스

from mcp.server.auth.provider import AccessToken, TokenVerifier


class SimpleTokenVerifier(TokenVerifier):
    """가장 단순한 토큰 검증기 (학습용)"""

    async def verify_token(self, token: str) -> AccessToken | None:
        """토큰을 검증하고 AccessToken을 반환한다

        유효한 토큰이면 AccessToken을 반환하고,
        유효하지 않으면 None을 반환한다.
        """
        if token == "valid-token":
            return AccessToken(
                token=token,
                client_id="demo-client",

위의 하드코딩된 토큰 검증은 학습용이다. 프로덕션에서는 반드시 다음 섹션의 JWT 검증을 사용해야 한다.

실제 JWT 검증 구현

프로덕션 환경에서는 JWT(JSON Web Token)를 RS256 알고리즘으로 서명 검증해야 한다. Authorization Server의 JWKS(JSON Web Key Set) 엔드포인트에서 공개키를 가져와 검증하는 전체 코드를 살펴보자: 

"""jwt_verifier.py — PyJWT를 사용한 실제 JWT 검증"""

import httpx
import jwt
from jwt import PyJWKClient
from mcp.server.auth.provider import AccessToken, TokenVerifier


class JWTTokenVerifier(TokenVerifier):
    """JWKS 기반 JWT 토큰 검증기

    Authorization Server의 JWKS 엔드포인트에서 공개키를 가져와
    JWT 서명을 검증하고, 클레임을 확인한다.
    """

    def __init__(
        self,
        jwks_url: str,
        issuer: str,
        audience: str,
        algorithms: list[str] | None = None,
    ):
        self.jwks_url = jwks_url
        self.issuer = issuer
        self.audience = audience
        self.algorithms = algorithms or ["RS256"]
        # PyJWT의 JWK 클라이언트 — 공개키 캐싱 내장
        self.jwk_client = PyJWKClient(jwks_url, cache_keys=True)

    async def verify_token(self, token: str) -> AccessToken | None:
        """JWT를 검증하고 AccessToken을 반환한다"""
        try:
            # 1. JWKS에서 토큰의 kid에 해당하는 공개키 조회
            signing_key = self.jwk_client.get_signing_key_from_jwt(token)

            # 2. JWT 디코딩 + 서명 검증 + 클레임 검증
            payload = jwt.decode(
                token,
                signing_key.key,
                algorithms=self.algorithms,
                issuer=self.issuer,
                audience=self.audience,
                options={
                    "verify_exp": True,      # 만료 시간 검증
                    "verify_iat": True,      # 발급 시간 검증
                    "verify_iss": True,      # 발급자 검증
                    "verify_aud": True,      # 대상자 검증
                    "require": ["exp", "iss", "aud", "sub"],
                },
            )

            # 3. scope 추출 (공백 구분 문자열 또는 리스트)
            scopes_raw = payload.get("scope", "")
            if isinstance(scopes_raw, str):
                scopes = scopes_raw.split() if scopes_raw else []
            else:
                scopes = list(scopes_raw)

            # 4. AccessToken 생성
            return AccessToken(
                token=token,
                client_id=payload.get("client_id", payload.get("azp", "")),
                scopes=scopes,
                expires_at=payload.get("exp"),
            )

        except jwt.ExpiredSignatureError:
            # 토큰 만료
            return None
        except jwt.InvalidTokenError:
            # 서명 불일치, 클레임 오류 등
            return None
        except Exception:
            # JWKS 조회 실패 등
            return None

AuthSettings 필드 타입 설명
issuer_url AnyHttpUrl Authorization Server의 URL. 클라이언트가 이 URL로 인증 흐름을 시작한다.
resource_server_url AnyHttpUrl 이 MCP 서버의 URL. Protected Resource Metadata에 포함된다.
required_scopes list[str] 모든 도구 호출에 필요한 최소 scope. 토큰에 이 scope가 포함되어야 접근이 허용된다.

Scope 기반 도구 접근 제어

required_scopes는 서버 전체에 적용되는 최소 요구사항이다. 도구별로 더 세밀한 접근 제어가 필요하다면, 도구 함수 내부에서 scope를 확인해야 한다: 

from mcp.server.fastmcp import Context


@mcp.tool()
async def delete_user(user_id: int, ctx: Context) -> str:
    """사용자를 삭제한다 (admin scope 필요)"""

    # 현재 요청의 Access Token에서 scope 확인
    token = ctx.request_context.access_token
    if token is None or "admin" not in token.scopes:
        raise PermissionError(
            "이 도구는 'admin' scope가 필요하다. "
            f"현재 scope: {token.scopes if token else '없음'}"
        )

    # admin scope가 있는 경우에만 실행
    return f"사용자 {user_id}를 삭제했다"


@mcp.tool()
async def get_user(user_id: int, ctx: Context) -> dict:
    """사용자 정보를 조회한다 (read scope면 충분)"""
    # required_scopes=["read"]로 서버 수준에서 이미 검증됨
    return {"id": user_id, "name": "홍길동"}
TIP

도구 수준의 scope 검사를 데코레이터로 추상화하면 코드를 깔끔하게 유지할 수 있다. @require_scope("admin") 같은 데코레이터를 만들어 사용하자.

PKCE 구현 상세

PKCE(Proof Key for Code Exchange, RFC 7636)는 Authorization Code 가로채기 공격을 방지하는 보안 메커니즘이다. OAuth 2.1에서는 모든 클라이언트(public, confidential 불문)에 PKCE가 필수이다.

Deep Dive: PKCE가 해결하는 공격 벡터

Authorization Code Interception Attack: 악성 앱이 Custom URL Scheme을 가로채거나, 네트워크 중간에서 Authorization Code를 탈취하는 공격이다. PKCE 없이는 탈취한 code만으로 Access Token을 획득할 수 있다. PKCE가 있으면 code를 가로채더라도 code_verifier를 모르므로 토큰을 교환할 수 없다. code_challenge는 code_verifier의 SHA-256 해시이므로, challenge에서 verifier를 역산하는 것은 불가능하다.

PKCE 동작 원리

  1. 1
    code_verifier 생성 43~128자의 랜덤 문자열 (A-Z, a-z, 0-9, "-._~")을 생성한다.
  2. 2
    code_challenge 생성 (S256 변환) code_challenge = BASE64URL(SHA256(code_verifier))
  3. 3
    인증 요청 시 challenge 전송 /authorize 요청에 code_challenge와 code_challenge_method=S256을 포함한다.
  4. 4
    토큰 교환 시 verifier 전송 /token 요청에 code_verifier를 포함한다. Authorization Server는 verifier를 해시하여 이전에 받은 challenge와 일치하는지 확인한다.

Python 구현

"""pkce.py — PKCE code_verifier와 code_challenge 생성"""

import hashlib
import base64
import secrets


def generate_code_verifier(length: int = 128) -> str:
    """PKCE code_verifier를 생성한다

    RFC 7636 Section 4.1:
    - 43~128자의 unreserved characters
    - A-Z, a-z, 0-9, "-", ".", "_", "~"
    """
    # secrets.token_urlsafe()는 A-Za-z0-9_- 문자열을 생성
    # base64url 인코딩 결과이므로 PKCE 요구사항을 만족한다
    return secrets.token_urlsafe(length)[:length]


def generate_code_challenge(verifier: str) -> str:
    """code_verifier에서 S256 code_challenge를 생성한다

    RFC 7636 Section 4.2:
    code_challenge = BASE64URL(SHA256(ASCII(code_verifier)))
    """
    # SHA-256 해시
    digest = hashlib.sha256(verifier.encode("ascii")).digest()
    # Base64URL 인코딩 (패딩 제거)
    return base64.urlsafe_b64encode(digest).rstrip(b"=").decode("ascii")


# 사용 예
verifier = generate_code_verifier()
challenge = generate_code_challenge(verifier)

print(f"code_verifier:  {verifier}")
print(f"code_challenge: {challenge}")
print(f"verifier 길이:  {len(verifier)}")
print(f"challenge 길이: {len(challenge)}")

클라이언트 측 OAuth

인증이 적용된 MCP 서버에 접속하려면 클라이언트에서 OAuth 토큰을 관리해야 한다. MCP Python SDK는 OAuthClientProvider와 TokenStorage 인터페이스를 제공한다.

TokenStorage 구현

"""메모리 기반 토큰 저장소 — 개발/테스트용"""

from mcp.client.auth import TokenStorage
from mcp.shared.auth import OAuthToken, OAuthClientInformationFull


class InMemoryTokenStorage(TokenStorage):
    """프로세스 종료 시 토큰이 사라진다"""

    def __init__(self):
        self._tokens: OAuthToken | None = None
        self._client_info: OAuthClientInformationFull | None = None

    async def get_tokens(self) -> OAuthToken | None:
        return self._tokens

    async def set_tokens(self, tokens: OAuthToken) -> None:
        self._tokens = tokens

    async def get_client_info(self) -> OAuthClientInformationFull | None:
        return self._client_info

    async def set_client_info(self, info: OAuthClientInformationFull) -> None:
        self._client_info = info
"""파일 기반 토큰 저장소 — 프로덕션용"""

import json
from pathlib import Path
from mcp.client.auth import TokenStorage
from mcp.shared.auth import OAuthToken, OAuthClientInformationFull


class FileTokenStorage(TokenStorage):
    """토큰을 로컬 파일에 저장한다"""

    def __init__(self, token_path: str = ".mcp_tokens.json"):
        self._path = Path(token_path)
        self._data: dict = {}
        if self._path.exists():
            self._data = json.loads(self._path.read_text())

    def _save(self):
        self._path.write_text(json.dumps(self._data, indent=2))
        # 파일 권한을 소유자만 읽기/쓰기로 제한
        self._path.chmod(0o600)

    async def get_tokens(self) -> OAuthToken | None:
        raw = self._data.get("tokens")
        return OAuthToken(**raw) if raw else None

    async def set_tokens(self, tokens: OAuthToken) -> None:
        self._data["tokens"] = tokens.model_dump()
        self._save()

    async def get_client_info(self) -> OAuthClientInformationFull | None:
        raw = self._data.get("client_info")
        return OAuthClientInformationFull(**raw) if raw else None

    async def set_client_info(self, info: OAuthClientInformationFull) -> None:
        self._data["client_info"] = info.model_dump()
        self._save()

자동 토큰 갱신 (Refresh Token)

OAuthClientProvider는 Access Token이 만료되면 자동으로 Refresh Token을 사용하여 새 Access Token을 발급받는다. grant_types에 "refresh_token"을 포함해야 한다. 

"""자동 토큰 갱신이 포함된 클라이언트 연결"""

import asyncio
from mcp.client.streamable_http import streamable_http_client
from mcp.client.session import ClientSession


async def connect_with_oauth():
    """OAuth 인증으로 MCP 서버에 연결한다"""

    async with streamable_http_client(
        "https://mcp-server.example.com/mcp",
        auth=oauth_provider,  # OAuthClientProvider를 auth에 전달
    ) as (read_stream, write_stream, _):
        async with ClientSession(read_stream, write_stream) as session:
            await session.initialize()

            # Access Token이 만료되면 자동으로 Refresh Token으로 갱신
            tools = await session.list_tools()
            print(f"도구 {len(tools.tools)}개를 사용할 수 있다")

            # 도구 호출
            result = await session.call_tool("get_user", {"user_id": 1})
            print(result.content)


if __name__ == "__main__":
    asyncio.run(connect_with_oauth())

외부 IDP 연동

실제 프로덕션에서는 직접 Authorization Server를 구축하지 않고, Auth0, Keycloak, Google 등 외부 Identity Provider(IDP)를 사용한다. MCP 서버는 Resource Server로서 IDP가 발급한 JWT를 검증하기만 하면 된다.

Auth0 연동

  1. 1
    Auth0 API 생성 Auth0 Dashboard에서 Applications → APIs → Create API로 새 API를 생성한다. Identifier(audience)를 MCP 서버 URL로 설정한다.
  2. 2
    Auth0 Application 생성 Applications → Create Application에서 Regular Web Application을 생성한다. Callback URL, Logout URL 등을 설정한다.
  3. 3
    Permission(Scope) 설정 API의 Permissions 탭에서 read, write, admin 등 scope를 정의한다.
  4. 4
    MCP 서버에서 JWT 검증 Auth0의 JWKS URL과 issuer를 사용하여 JWTTokenVerifier를 설정한다. 
"""auth0_mcp_server.py — Auth0 연동 MCP 서버"""

from mcp.server.fastmcp import FastMCP
from pydantic import AnyHttpUrl

    "Auth0 Protected Server",
    token_verifier=verifier,

        issuer_url=AnyHttpUrl(f"https://{AUTH0_DOMAIN}"),

)


@mcp.tool()

        "client_id": token.client_id,
        "scopes": token.scopes,

TIP

Auth0의 issuer URL 끝에 /(슬래시)가 포함되어야 한다는 점에 주의하자. https://your-tenant.auth0.com이 아니라 https://your-tenant.auth0.com/이다. 이 차이로 JWT 검증이 실패하는 경우가 매우 흔하다.

보안 베스트 프랙티스

HTTPS 필수

WARNING

프로덕션 환경에서 OAuth는 반드시 HTTPS를 사용해야 한다. HTTP를 사용하면 Bearer Token이 네트워크에서 평문으로 전송되어, 중간자 공격(MITM)으로 토큰이 탈취될 수 있다. 로컬 개발 시에만 http://localhost를 허용한다.

토큰 수명 설정 가이드

토큰 종류 권장 수명 근거
Access Token 15분 ~ 1시간 탈취 시 피해를 최소화. 짧을수록 안전하지만 UX 저하
Refresh Token 7일 ~ 30일 사용자가 재로그인할 필요 없는 기간. 회전(rotation) 적용 권장
Authorization Code 10분 이내 일회성. 사용 즉시 폐기되어야 한다

토큰 저장 보안

  • 파일 시스템에 저장할 때는 chmod 600으로 소유자만 접근 가능하게 설정한다
  • 환경 변수에 토큰을 저장하지 않는다 — 프로세스 목록에서 노출될 수 있다
  • 가능하면 OS의 키체인/Credential Manager를 사용한다 (macOS Keychain, Windows Credential Manager)
  • 로그에 토큰을 절대 출력하지 않는다 — 마스킹 처리한다

Rate Limiting

"""rate_limiter.py — 간단한 인메모리 Rate Limiter"""

import time
from collections import defaultdict


class RateLimiter:
    """슬라이딩 윈도우 기반 Rate Limiter"""

    def __init__(self, max_requests: int = 100, window_seconds: int = 60):
        self.max_requests = max_requests
        self.window = window_seconds
        self._requests: dict[str, list[float]] = defaultdict(list)

    def is_allowed(self, client_id: str) -> bool:
        """요청이 허용되는지 확인한다"""
        now = time.time()
        # 윈도우 밖의 요청 제거
        self._requests[client_id] = [
            t for t in self._requests[client_id]
            if now - t < self.window
        ]
        # 한도 확인
        if len(self._requests[client_id]) >= self.max_requests:
            return False
        self._requests[client_id].append(now)
        return True


# TokenVerifier에서 Rate Limiting 적용
class RateLimitedVerifier(TokenVerifier):
    def __init__(self, inner: TokenVerifier, limiter: RateLimiter):
        self.inner = inner
        self.limiter = limiter

    async def verify_token(self, token: str) -> AccessToken | None:
        result = await self.inner.verify_token(token)
        if result is None:
            return None
        if not self.limiter.is_allowed(result.client_id):
            return None  # Rate limit 초과 시 인증 거부
        return result

"""secure_mcp_server.py — 프로덕션급 인증 MCP 서버"""

import logging
from datetime import datetime
from functools import wraps

import jwt
from jwt import PyJWKClient
from pydantic import AnyHttpUrl, BaseModel, Field

from mcp.server.auth.provider import AccessToken, TokenVerifier
from mcp.server.auth.settings import AuthSettings
from mcp.server.fastmcp import Context, FastMCP

logger = logging.getLogger(__name__)

# ============================================================
# 설정
# ============================================================

AUTH0_DOMAIN = "your-tenant.auth0.com"
AUTH0_AUDIENCE = "https://mcp-server.example.com"
JWKS_URL = f"https://{AUTH0_DOMAIN}/.well-known/jwks.json"
ISSUER = f"https://{AUTH0_DOMAIN}/"


# ============================================================
# JWT 검증기
# ============================================================

class ProductionTokenVerifier(TokenVerifier):
    """Auth0 JWT 검증기"""

    def __init__(self):
        self.jwk_client = PyJWKClient(JWKS_URL, cache_keys=True)

    async def verify_token(self, token: str) -> AccessToken | None:
        try:
            signing_key = self.jwk_client.get_signing_key_from_jwt(token)
            payload = jwt.decode(
                token,
                signing_key.key,
                algorithms=["RS256"],
                issuer=ISSUER,
                audience=AUTH0_AUDIENCE,
                options={
                    "verify_exp": True,
                    "verify_iss": True,
                    "verify_aud": True,
                    "require": ["exp", "iss", "aud", "sub"],
                },
            )

            scopes = payload.get("scope", "").split()
            return AccessToken(
                token=token,
                client_id=payload.get("azp", ""),
                scopes=scopes,
                expires_at=payload.get("exp"),
            )

        except jwt.ExpiredSignatureError:
            logger.warning("만료된 토큰")
            return None
        except jwt.InvalidTokenError as e:
            logger.warning(f"유효하지 않은 토큰: {e}")
            return None


# ============================================================
# Scope 검사 데코레이터
# ============================================================

def require_scope(*required_scopes: str):
    """도구에 scope 요구사항을 추가하는 데코레이터"""
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, ctx: Context, **kwargs):
            token = ctx.request_context.access_token
            if token is None:
                raise PermissionError("인증이 필요하다")

            missing = set(required_scopes) - set(token.scopes)
            if missing:
                raise PermissionError(
                    f"권한이 부족하다. 필요한 scope: {missing}"
                )

            return await func(*args, ctx=ctx, **kwargs)
        return wrapper
    return decorator


# ============================================================
# MCP 서버
# ============================================================

mcp = FastMCP(
    "Secure Production Server",
    token_verifier=ProductionTokenVerifier(),
    auth=AuthSettings(
        issuer_url=AnyHttpUrl(f"https://{AUTH0_DOMAIN}"),
        resource_server_url=AnyHttpUrl(AUTH0_AUDIENCE),
        required_scopes=["read"],
    ),
)


# ============================================================
# 모델 정의
# ============================================================

class UserInfo(BaseModel):
    """사용자 정보"""
    id: int = Field(description="사용자 ID")
    name: str = Field(description="이름")
    email: str = Field(description="이메일")
    role: str = Field(description="역할")


class AuditLog(BaseModel):
    """감사 로그"""
    action: str = Field(description="수행한 작업")
    user: str = Field(description="사용자")
    timestamp: str = Field(description="시각")
    details: str = Field(description="상세 내용")


# ============================================================
# 도구 정의
# ============================================================

@mcp.tool()
async def get_users(ctx: Context) -> list[UserInfo]:
    """사용자 목록을 조회한다 (read scope)"""
    # required_scopes=["read"]로 서버 수준에서 이미 검증됨
    return [
        UserInfo(id=1, name="홍길동", email="hong@example.com", role="admin"),
        UserInfo(id=2, name="김철수", email="kim@example.com", role="user"),
    ]


@mcp.tool()
@require_scope("write")
async def create_user(name: str, email: str, ctx: Context) -> UserInfo:
    """사용자를 생성한다 (write scope 필요)"""
    return UserInfo(id=3, name=name, email=email, role="user")


@mcp.tool()
@require_scope("admin")
async def delete_user(user_id: int, ctx: Context) -> str:
    """사용자를 삭제한다 (admin scope 필요)"""
    logger.info(f"사용자 {user_id} 삭제 요청 by {ctx.request_context.access_token.client_id}")
    return f"사용자 {user_id}를 삭제했다"


@mcp.tool()
@require_scope("admin")
async def get_audit_logs(limit: int = 10, ctx: Context = None) -> list[AuditLog]:
    """감사 로그를 조회한다 (admin scope 필요)"""
    return [
        AuditLog(
            action="user.created",
            user="admin",
            timestamp=datetime.now().isoformat(),
            details=f"새 사용자 생성 #{i}",
        )
        for i in range(1, limit + 1)
    ]


@mcp.tool()
async def whoami(ctx: Context) -> dict:
    """현재 인증된 클라이언트 정보를 반환한다"""
    token = ctx.request_context.access_token
    return {
        "client_id": token.client_id,
        "scopes": token.scopes,
        "authenticated": True,
    }

"""secure_client.py — 인증된 MCP 서버에 접속하는 클라이언트"""

import asyncio
import webbrowser
from mcp.client.auth import OAuthClientProvider
from mcp.client.session import ClientSession
from mcp.client.streamable_http import streamable_http_client
from mcp.shared.auth import OAuthClientMetadata


# 파일 기반 토큰 저장소 (이전 섹션의 FileTokenStorage 사용)
storage = FileTokenStorage(".mcp_tokens.json")


async def handle_redirect(url: str) -> None:
    webbrowser.open(url)


async def handle_callback() -> tuple[str, str | None]:
    code = input("인증 코드를 입력하라: ")
    return code, None


oauth = OAuthClientProvider(
    server_url="https://mcp-server.example.com",
    client_metadata=OAuthClientMetadata(
        client_name="Secure MCP Client",
        redirect_uris=["http://localhost:3000/callback"],
        grant_types=["authorization_code", "refresh_token"],
        response_types=["code"],
        scope="read write admin",
    ),
    storage=storage,
    redirect_handler=handle_redirect,
    callback_handler=handle_callback,
)


async def main():
    async with streamable_http_client(
        "https://mcp-server.example.com/mcp",
        auth=oauth,
    ) as (r, w, _):
        async with ClientSession(r, w) as session:
            await session.initialize()

            # 내 정보 확인
            me = await session.call_tool("whoami", {})
            print(f"인증 정보: {me.content}")

            # 사용자 목록 (read scope)
            users = await session.call_tool("get_users", {})
            print(f"사용자 목록: {users.content}")

            # 사용자 생성 (write scope)
            new_user = await session.call_tool(
                "create_user",
                {"name": "박지성", "email": "park@example.com"},
            )
            print(f"생성 결과: {new_user.content}")

            # 감사 로그 (admin scope)
            logs = await session.call_tool(
                "get_audit_logs",
                {"limit": 5},
            )
            print(f"감사 로그: {logs.content}")


if __name__ == "__main__":
    asyncio.run(main())
