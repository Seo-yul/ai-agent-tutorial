첫 번째 MCP 서버

FastMCP로 첫 서버를 만들고, Inspector와 Claude Desktop에서 테스트한다

FastMCP란?

핵심 개념: FastMCP의 설계 철학

FastMCP는 MCP Python SDK가 제공하는 고수준(High-level) API이다. "Pythonic하게, 데코레이터 하나로, 타입 힌트만으로" MCP 서버를 만들 수 있도록 설계되었다. 개발자는 프로토콜 세부사항(JSON-RPC, 메시지 포맷, Transport 관리)을 전혀 신경 쓸 필요 없이, 일반 Python 함수를 작성하는 것만으로 LLM이 호출할 수 있는 도구를 만든다.

High-level API (FastMCP) vs Low-level API (Server)

FastMCP (고수준) — 권장
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Calculator")

@mcp.tool()
def add(a: int, b: int) -> int:
    """두 수를 더한다"""
    return a + b

mcp.run()

데코레이터 기반. 함수 시그니처에서 스키마를 자동 생성한다. 대부분의 경우 이것으로 충분하다.

Server (저수준)
from mcp.server import Server
import mcp.types as types

server = Server("Calculator")

@server.list_tools()
async def list_tools():
    return [types.Tool(
        name="add",
        description="두 수를 더한다",
        inputSchema={
            "type": "object",
            "properties": {
                "a": {"type": "integer"},
                "b": {"type": "integer"}
            },
            "required": ["a", "b"]
        }
    )]

@server.call_tool()
async def call_tool(name, arguments):
    if name == "add":
        return [types.TextContent(
            type="text",
            text=str(arguments["a"] + arguments["b"])
        )]

직접 핸들러를 등록한다. JSON Schema를 수동으로 정의해야 한다. 고급 커스터마이징이 필요할 때 사용한다.

Deep Dive: FastMCP가 내부적으로 하는 일

@mcp.tool() 데코레이터가 함수에 적용되면 FastMCP는 다음 세 가지를 자동으로 처리한다.

  1. 함수 시그니처 파싱 — Python의 inspect 모듈로 매개변수 이름, 타입 힌트, 기본값을 추출한다.
  2. JSON Schema 생성 — 추출된 타입 정보를 MCP 프로토콜이 요구하는 inputSchema (JSON Schema Draft 7)로 변환한다. int → {"type": "integer"}, str → {"type": "string"}, Optional[str] → nullable 등.
  3. Transport 관리 — mcp.run() 호출 시 기본 stdio Transport를 설정하고, JSON-RPC 메시지의 직렬화/역직렬화, 요청 라우팅을 처리한다.

즉, FastMCP는 "Python 함수 → MCP 프로토콜 엔드포인트" 변환기이다.

최소한의 서버 — 10줄로 시작하기

프로젝트 디렉토리에 server.py 파일을 생성하고 다음 코드를 작성한다.

from mcp.server.fastmcp import FastMCP

# 1. 서버 인스턴스 생성 — 이름은 클라이언트가 서버를 식별할 때 사용된다
mcp = FastMCP("My First Server")

# 2. 도구 등록 — 데코레이터 하나로 MCP 도구가 된다
@mcp.tool()
def add(a: int, b: int) -> int:
    """두 수를 더한다"""
    return a + b

# 3. 서버 실행 — 기본적으로 stdio 전송 방식을 사용한다
if __name__ == "__main__":
    mcp.run()

이 10줄이 완전한 MCP 서버이다. 각 부분을 자세히 분석해보겠다.

줄 단위 분석

줄코드설명
1 from mcp.server.fastmcp import FastMCP 고수준 API 클래스를 임포트한다.
3 mcp = FastMCP("My First Server") 서버 인스턴스를 생성한다. 문자열 인자는 서버 이름으로, initialize 응답의 serverInfo.name에 포함된다.
5-8 @mcp.tool() + 함수 정의 일반 Python 함수를 MCP 도구로 등록한다. 타입 힌트 → JSON Schema, docstring → 도구 설명으로 자동 변환된다.
10-11 mcp.run() 이벤트 루프를 시작하고 클라이언트 연결을 대기한다. 기본 Transport는 stdio이다.
WARNING

docstring은 반드시 작성해야 한다. LLM은 도구의 docstring을 보고 "이 도구가 어떤 상황에서 유용한지" 판단한다. docstring이 없거나 부정확하면 LLM이 도구를 호출하지 않거나 잘못된 인자를 전달할 수 있다.

@mcp.tool()이 프로토콜 레벨에서 하는 일

클라이언트가 서버에 도구 목록을 요청하면 다음과 같은 JSON-RPC 메시지가 교환된다.

→ REQUEST — tools/list
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list",
  "params": {}
}
← RESPONSE
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "tools": [
      {
        "name": "add",
        "description": "두 수를 더한다",
        "inputSchema": {
          "type": "object",
          "properties": {
            "a": { "type": "integer", "title": "A" },
            "b": { "type": "integer", "title": "B" }
          },
          "required": ["a", "b"]
        }
      }
    ]
  }
}

LLM이 도구를 호출하면 다음 메시지가 교환된다.

→ REQUEST — tools/call
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "add",
    "arguments": { "a": 3, "b": 5 }
  }
}
← RESPONSE
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "8"
      }
    ]
  }
}

개발자가 작성한 것은 Python 함수뿐이지만, FastMCP가 이 모든 프로토콜 메시지를 자동으로 생성하고 처리한다.

서버 실행 방법 3가지

  1. 1
    직접 실행 (stdio 모드) 
    uv run server.py

    서버가 stdio 모드로 시작되어 표준 입력(stdin)에서 JSON-RPC 메시지를 대기한다. 터미널에는 아무것도 출력되지 않으며, 이 상태에서 직접 테스트하기는 어렵다. 주로 Claude Desktop이나 다른 MCP 클라이언트가 자식 프로세스로 실행할 때 사용되는 방식이다.

  2. 2
    MCP Inspector로 개발 모드 실행 (권장) 
    # Inspector와 함께 서버 시작
uv run mcp dev server.py

# 출력 예시:
# Starting MCP Inspector...
# Proxy server listening on port 6277
# 🔍 MCP Inspector: http://localhost:5173

    mcp dev는 서버를 프록시 프로세스로 감싸고 MCP Inspector 웹 UI를 함께 시작한다. 브라우저에서 http://localhost:5173을 열면 도구를 직접 테스트할 수 있다. 개발 중에는 항상 이 방법을 사용하는 것을 권장한다.

  3. 3
    Claude Desktop에 등록하여 실제 LLM에서 사용

    claude_desktop_config.json에 서버를 등록하면 Claude가 직접 도구를 호출한다.

    {
  "mcpServers": {
    "my-first-server": {
      "command": "uv",
      "args": [
        "run",
        "--directory", "/absolute/path/to/mcp-demo",
        "server.py"
      ]
    }
  }
}

    등록 후 Claude Desktop을 재시작하면 채팅에서 도구를 사용할 수 있다.

MCP Inspector 상세 활용법

uv run mcp dev server.py를 실행하고 브라우저에서 Inspector를 열면 다음과 같은 인터페이스를 볼 수 있다.

연결 상태 확인

Inspector 상단에 연결 상태가 표시된다. 초록색 "Connected"가 보이면 서버가 정상적으로 연결된 것이다. Transport 유형(stdio)과 서버 이름("My First Server")이 함께 표시된다. 연결이 실패하면 빨간색 "Disconnected"가 나타나며, 서버 실행 로그를 확인해야 한다.

Tools 탭

  1. 1
    도구 목록 조회

    "List Tools" 버튼을 클릭하면 서버에 등록된 모든 도구 목록이 나타난다. 내부적으로 tools/list JSON-RPC 요청이 전송된다.

  2. 2
    도구 선택 및 파라미터 입력

    도구를 선택하면 inputSchema를 기반으로 입력 폼이 자동 생성된다. 각 파라미터의 타입과 설명이 표시되며, 값을 직접 입력할 수 있다.

  3. 3
    실행 및 결과 확인

    "Run Tool" 버튼을 클릭하면 tools/call 요청이 전송되고, 결과가 하단 패널에 JSON 형태로 표시된다.

Resources 탭

서버에 등록된 리소스 목록을 보여준다. "List Resources"를 클릭하면 정적 리소스와 리소스 템플릿이 구분되어 표시된다. 리소스를 선택하고 "Read Resource"를 클릭하면 resources/read 요청이 전송되어 내용을 확인할 수 있다. URI 템플릿(greeting://{name})의 경우 파라미터 입력란이 나타난다.

Prompts 탭

등록된 프롬프트 목록을 보여준다. 프롬프트를 선택하면 필요한 인자(arguments)를 입력할 수 있으며, "Get Prompt"를 클릭하면 완성된 메시지 배열이 표시된다. 프롬프트가 LLM에게 전달될 때 어떤 형태가 되는지 미리 확인할 수 있다.

Notifications / Logs

Inspector 하단의 로그 패널에서는 서버와 클라이언트 사이에 오가는 모든 JSON-RPC 메시지를 실시간으로 볼 수 있다. 요청(Request)과 응답(Response), 알림(Notification)이 시간순으로 표시된다. 프로토콜 레벨에서 무슨 일이 일어나는지 직접 확인할 수 있으므로 디버깅에 매우 유용하다.

TIP

Inspector에서 도구를 테스트할 때, 의도적으로 잘못된 타입의 값(예: 정수 파라미터에 문자열)을 입력해보자. FastMCP가 어떤 에러 메시지를 반환하는지 확인할 수 있다. 이 에러 처리 방식은 4장에서 자세히 다룬다.

3가지 프리미티브를 모두 사용하는 서버

도구(Tool), 리소스(Resource), 프롬프트(Prompt)를 모두 포함하는 종합 예제를 만들어보겠다.

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Demo Server")

# ── 도구 (Tool) ──────────────────────────────────────
# LLM이 직접 호출하여 동작을 수행한다

@mcp.tool()
def add(a: int, b: int) -> int:
    """두 정수를 더한다"""
    return a + b

@mcp.tool()
def multiply(a: int, b: int) -> int:
    """두 정수를 곱한다"""
    return a * b

@mcp.tool()
async def fetch_title(url: str) -> str:
    """URL에서 웹 페이지 제목을 가져온다"""
    import httpx
    async with httpx.AsyncClient() as client:
        response = await client.get(url, follow_redirects=True)
        # 간단한 제목 추출
        html = response.text
        start = html.find("<title>")
        end = html.find("</title>")
        if start != -1 and end != -1:
            return html[start + 7:end].strip()
        return "제목을 찾을 수 없다"

# ── 리소스 (Resource) ────────────────────────────────
# 데이터를 URI로 노출한다. 읽기 전용.

@mcp.resource("config://app")
def get_app_config() -> dict:
    """애플리케이션 설정 정보"""
    return {
        "app_name": "MCP Demo",
        "version": "1.0.0",
        "debug": True,
        "max_retries": 3
    }

@mcp.resource("greeting://{name}")
def get_greeting(name: str) -> str:
    """개인화된 인사말을 반환한다"""
    return f"안녕, {name}! MCP 서버에 온 것을 환영한다."

@mcp.resource("stats://server")
def get_server_stats() -> dict:
    """서버 통계 정보"""
    import datetime
    return {
        "status": "running",
        "uptime": "동적 데이터",
        "timestamp": datetime.datetime.now().isoformat(),
        "registered_tools": 3,
        "registered_resources": 3,
        "registered_prompts": 2
    }

# ── 프롬프트 (Prompt) ───────────────────────────────
# 재사용 가능한 프롬프트 템플릿

@mcp.prompt()
def review_code(code: str, language: str = "python") -> str:
    """코드 리뷰를 요청하는 프롬프트"""
    return f"""다음 {language} 코드를 리뷰해라.
버그, 성능 이슈, 가독성 문제를 찾아서 개선안을 제시해라.

```{language}
{code}
```"""

@mcp.prompt()
def explain_error(error_message: str) -> str:
    """에러 메시지를 분석하는 프롬프트"""
    return f"""다음 에러 메시지를 분석하고 해결 방법을 단계별로 설명해라.

에러:
{error_message}"""

if __name__ == "__main__":
    mcp.run()

각 프리미티브의 프로토콜 메시지

위 서버에서 각 프리미티브가 어떤 JSON-RPC 메시지로 교환되는지 살펴보겠다.

Tool — 도구 호출

→ REQUEST — tools/call
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "multiply",
    "arguments": { "a": 7, "b": 6 }
  }
}
← RESPONSE
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [{ "type": "text", "text": "42" }]
  }
}

Resource — 리소스 읽기

→ REQUEST — resources/read
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "resources/read",
  "params": {
    "uri": "greeting://Alice"
  }
}
← RESPONSE
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "contents": [
      {
        "uri": "greeting://Alice",
        "mimeType": "text/plain",
        "text": "안녕, Alice! MCP 서버에 온 것을 환영한다."
      }
    ]
  }
}

Prompt — 프롬프트 가져오기

→ REQUEST — prompts/get
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "prompts/get",
  "params": {
    "name": "review_code",
    "arguments": {
      "code": "def hello(): print('hi')",
      "language": "python"
    }
  }
}
← RESPONSE
{
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "messages": [
      {
        "role": "user",
        "content": {
          "type": "text",
          "text": "다음 python 코드를 리뷰해라.\n버그, 성능 이슈, 가독성 문제를 찾아서 개선안을 제시해라.\n\n```python\ndef hello(): print('hi')\n```"
        }
      }
    ]
  }
}
INFO

리소스 URI에서 {name} 같은 중괄호 부분은 경로 파라미터이다. greeting://Alice로 요청하면 name에 "Alice"가 전달된다. 중괄호가 없는 URI(config://app)는 정적 리소스로 항상 같은 함수가 호출된다.

트러블슈팅

증상해결 방법
서버가 안 뜸 (망치 아이콘 없음)
  1. JSON 문법 오류가 없는지 확인한다 (쉼표, 따옴표 등).
  2. --directory 경로가 절대 경로인지 확인한다.
  3. 터미널에서 uv run --directory /path/to/project server.py를 직접 실행하여 에러가 없는지 확인한다.
도구가 안 보임
  1. server.py에 @mcp.tool() 데코레이터가 있는지 확인한다.
  2. if __name__ == "__main__": mcp.run()이 있는지 확인한다.
  3. 파일명이 설정의 args와 일치하는지 확인한다.
도구 호출 시 에러 발생
  1. 필요한 패키지가 설치되어 있는지 확인한다 (fetch_title의 경우 uv add httpx).
  2. 서버 코드에 문법 에러가 없는지 확인한다.
  3. mcp dev로 Inspector에서 먼저 테스트해본다.
Deep Dive: LLM은 도구를 어떻게 선택하는가

Claude Desktop이 MCP 서버에 연결하면, 먼저 tools/list를 호출하여 사용 가능한 도구 목록을 가져온다. 이 목록에는 각 도구의 name, description(docstring에서 추출), inputSchema가 포함된다. LLM은 이 정보를 시스템 프롬프트의 일부로 받아, 사용자 요청에 적합한 도구가 있을 때 해당 도구를 호출하기로 "결정"한다. 따라서 도구의 이름과 설명(docstring)이 명확할수록 LLM이 올바르게 도구를 선택할 확률이 높아진다.

서버 초기화 옵션

FastMCP 생성자에는 서버의 동작을 세밀하게 제어할 수 있는 여러 파라미터가 있다.

from mcp.server.fastmcp import FastMCP

mcp = FastMCP(
    name="Production Server",         # 서버 이름 (필수)
    instructions="이 서버는 수학 계산 도구를 제공한다. "
                 "add와 multiply 도구를 사용하여 기본 산술 연산을 수행할 수 있다.",
)
파라미터타입설명
name str 서버의 고유 이름. initialize 응답의 serverInfo.name에 포함되어 클라이언트가 서버를 식별하는 데 사용된다.
instructions str | None LLM에게 전달되는 서버 사용 안내문. 이 문자열은 initialize 응답에 포함되어 LLM이 서버의 용도와 도구 사용법을 이해하는 데 도움을 준다. 잘 작성하면 LLM의 도구 선택 정확도가 높아진다.
lifespan callable | None 서버의 시작/종료 시 실행할 비동기 컨텍스트 매니저. DB 연결 풀 초기화, 외부 서비스 인증 등 서버 수명 주기 관리에 사용된다. 고급 챕터에서 다룬다.
핵심 개념: instructions의 역할

instructions는 서버 전체에 대한 "사용 설명서"를 LLM에게 전달하는 메커니즘이다. 개별 도구의 docstring이 "이 도구는 무엇을 하는가"를 설명한다면, instructions는 "이 서버를 전반적으로 어떻게 활용해야 하는가"를 안내한다. 예를 들어, 도구 호출 순서, 주의사항, 데이터 형식 규칙 등을 기술할 수 있다.

mcp.run() 옵션

mcp.run()에는 Transport 관련 옵션을 지정할 수 있다.

# 기본: stdio 모드 (Claude Desktop, MCP Inspector용)
mcp.run()

# SSE 모드: 원격 클라이언트가 HTTP로 연결 (네트워크 배포 시)
mcp.run(transport="sse")

# Streamable HTTP 모드: 최신 MCP 스펙 (2025+)
mcp.run(transport="streamable-http")
INFO

로컬 개발 및 Claude Desktop 연동에서는 기본 stdio 모드를 사용한다. sse와 streamable-http는 서버를 네트워크에 배포할 때 사용하며, 고급 챕터에서 다룬다.
