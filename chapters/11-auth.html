OAuth 인증

MCP 서버에 OAuth 2.1 인증을 적용한다

왜 인증이 필요한가?

로컬에서 stdio 방식으로 실행되는 MCP 서버는 별도의 인증이 필요 없다. 하지만 원격 MCP 서버(Streamable HTTP 등)는 네트워크를 통해 누구나 접근할 수 있으므로 인증이 필수적이다.

MCP 프로토콜은 OAuth 2.1 기반의 인증 체계를 지원한다. OAuth 2.1은 OAuth 2.0의 보안 모범 사례를 통합한 최신 표준으로, 다음과 같은 특징이 있다:

  • PKCE 필수 — Authorization Code 플로우에서 코드 가로채기 공격을 방지한다
  • Refresh Token 회전 — 탈취된 토큰의 재사용을 방지한다
  • 범위 기반 접근 제어 — 클라이언트의 권한을 세밀하게 제어한다
INFO

MCP의 인증 모델은 크게 두 가지로 나뉜다. Authorization Server(인가 서버)가 토큰을 발급하고, Resource Server(리소스 서버 = MCP 서버)가 토큰을 검증한다. 이 장에서는 Resource Server 구축에 초점을 맞춘다.

Resource Server 구축

MCP 서버를 인증된 Resource Server로 만들려면 TokenVerifier를 구현해야 한다. TokenVerifier는 클라이언트가 보낸 Bearer 토큰의 유효성을 검증하는 역할을 한다. 

from mcp.server.auth.provider import AccessToken, TokenVerifier
from mcp.server.auth.settings import AuthSettings
from mcp.server.fastmcp import FastMCP
from pydantic import AnyHttpUrl

class SimpleTokenVerifier(TokenVerifier):
    """토큰 검증기"""
    async def verify_token(self, token: str) -> AccessToken | None:
        if token == "valid-token":
            return AccessToken(
                token=token,
                client_id="demo-client",
                scopes=["read", "write"],
            )
        return None

mcp = FastMCP(
    "Secure API",
    token_verifier=SimpleTokenVerifier(),
    auth=AuthSettings(
        issuer_url=AnyHttpUrl("https://auth.example.com"),
        resource_server_url=AnyHttpUrl("http://localhost:8000"),
        required_scopes=["read"],
    ),
)

verify_token 메서드는 토큰 문자열을 받아 유효한 경우 AccessToken 객체를 반환하고, 유효하지 않은 경우 None을 반환한다. 실제 프로덕션에서는 JWT 검증, 외부 인가 서버 조회 등을 수행한다.

WARNING

위 예제의 SimpleTokenVerifier는 학습용이다. 실제 서비스에서는 하드코딩된 토큰 대신 JWT 서명 검증이나 OAuth introspection 엔드포인트를 사용해야 한다.

AuthSettings 설정

AuthSettings는 MCP 서버의 인증 관련 설정을 정의한다. 각 필드의 의미는 다음과 같다:

필드 타입 설명
issuer_url AnyHttpUrl 토큰을 발급하는 Authorization Server의 URL. 클라이언트가 이 URL로 인증 요청을 보낸다.
resource_server_url AnyHttpUrl 이 MCP 서버 자신의 URL. Protected Resource Metadata에 포함된다.
required_scopes list[str] 도구 호출에 필요한 최소 권한 범위. 토큰에 이 범위가 포함되어 있어야 접근이 허용된다. 
from mcp.server.auth.settings import AuthSettings
from pydantic import AnyHttpUrl

# 기본 설정
auth_settings = AuthSettings(
    issuer_url=AnyHttpUrl("https://auth.example.com"),
    resource_server_url=AnyHttpUrl("http://localhost:8000"),
    required_scopes=["read"],
)

# 여러 scope를 요구하는 경우
admin_auth_settings = AuthSettings(
    issuer_url=AnyHttpUrl("https://auth.example.com"),
    resource_server_url=AnyHttpUrl("http://localhost:8000"),
    required_scopes=["read", "write", "admin"],
)
TIP

required_scopes는 해당 MCP 서버의 모든 도구에 공통 적용된다. 도구별로 다른 권한이 필요한 경우, verify_token 내부에서 도구별 권한 검사 로직을 추가로 구현할 수 있다.

클라이언트 측 OAuth

인증이 적용된 MCP 서버에 접속하려면 클라이언트에서 OAuth 토큰을 관리해야 한다. MCP Python SDK는 OAuthClientProvider와 TokenStorage 인터페이스를 제공한다.

TokenStorage 구현

TokenStorage는 OAuth 토큰과 클라이언트 정보를 저장하고 조회하는 인터페이스이다. 아래는 메모리 기반의 간단한 구현 예제이다: 

from mcp.client.auth import OAuthClientProvider, TokenStorage
from mcp.shared.auth import OAuthClientMetadata, OAuthToken, OAuthClientInformationFull

class InMemoryTokenStorage(TokenStorage):
    """메모리 기반 토큰 저장소"""

    def __init__(self):
        self.tokens = None
        self.client_info = None

    async def get_tokens(self) -> OAuthToken | None:
        """저장된 토큰을 반환한다"""
        return self.tokens

    async def set_tokens(self, tokens: OAuthToken) -> None:
        """토큰을 저장한다"""
        self.tokens = tokens

    async def get_client_info(self) -> OAuthClientInformationFull | None:
        """클라이언트 정보를 반환한다"""
        return self.client_info

    async def set_client_info(self, client_info: OAuthClientInformationFull) -> None:
        """클라이언트 정보를 저장한다"""
        self.client_info = client_info

OAuthClientProvider 구성

OAuthClientProvider는 OAuth 플로우를 관리하는 핵심 컴포넌트이다. TokenStorage, 클라이언트 메타데이터, 리디렉션 처리 콜백을 조합하여 구성한다: 

import webbrowser
from mcp.client.auth import OAuthClientProvider
from mcp.shared.auth import OAuthClientMetadata

async def handle_redirect(authorization_url: str) -> None:
    """인증 URL로 브라우저를 연다"""
    print(f"브라우저에서 인증을 완료하세요: {authorization_url}")
    webbrowser.open(authorization_url)

async def handle_callback() -> tuple[str, str | None]:
    """인증 콜백을 처리한다. (authorization_code, state)를 반환한다"""
    code = input("인증 코드를 입력하세요: ")
    return code, None

oauth_provider = OAuthClientProvider(
    server_url="http://localhost:8000",
    client_metadata=OAuthClientMetadata(
        client_name="MCP 튜토리얼 클라이언트",
        redirect_uris=["http://localhost:3000/callback"],
        grant_types=["authorization_code", "refresh_token"],
        response_types=["code"],
    ),
    storage=InMemoryTokenStorage(),
    redirect_handler=handle_redirect,
    callback_handler=handle_callback,
)
INFO

실제 프로덕션에서는 InMemoryTokenStorage 대신 파일 기반이나 데이터베이스 기반 저장소를 사용해야 한다. 메모리 저장소는 프로세스가 종료되면 토큰이 사라지기 때문이다.

인증 흐름

MCP에서 OAuth 2.1 인증은 다음과 같은 흐름으로 진행된다. 클라이언트, Authorization Server, Resource Server(MCP 서버) 간의 상호작용을 단계별로 살펴보자. 

┌──────────┐      ┌─────────────────────┐      ┌─────────────────┐
│  Client  │      │ Authorization Server│      │ Resource Server │
│(MCP 클라이언트)│      │   (인가 서버)         │      │  (MCP 서버)      │
└────┬─────┘      └──────────┬──────────┘      └────────┬────────┘
     │                       │                          │
     │  1. MCP 서버에 접속 시도                           │
     │─────────────────────────────────────────────────▶│
     │                       │                          │
     │  2. 401 Unauthorized + Protected Resource Metadata
     │◀─────────────────────────────────────────────────│
     │                       │                          │
     │  3. Authorization 요청 │                          │
     │──────────────────────▶│                          │
     │                       │                          │
     │  4. 로그인/동의 화면    │                          │
     │◀──────────────────────│                          │
     │                       │                          │
     │  5. Authorization Code│                          │
     │◀──────────────────────│                          │
     │                       │                          │
     │  6. Code → Token 교환  │                          │
     │──────────────────────▶│                          │
     │                       │                          │
     │  7. Access Token 발급  │                          │
     │◀──────────────────────│                          │
     │                       │                          │
     │  8. Bearer Token으로 MCP 요청                     │
     │─────────────────────────────────────────────────▶│
     │                       │                          │
     │  9. 토큰 검증 후 응답                              │
     │◀─────────────────────────────────────────────────│
     │                       │                          │
  1. 접속 시도 — 클라이언트가 MCP 서버에 인증 없이 접속을 시도한다
  2. 인증 필요 응답 — 서버가 401 응답과 함께 Protected Resource Metadata를 반환한다. 이 메타데이터에는 Authorization Server 정보가 포함된다
  3. 인증 요청 — 클라이언트가 Authorization Server에 인증을 요청한다
  4. 사용자 인증 — 사용자가 로그인하고 권한 동의를 완료한다
  5. 인가 코드 발급 — Authorization Server가 Authorization Code를 발급한다
  6. 토큰 교환 — 클라이언트가 Authorization Code를 Access Token으로 교환한다
  7. 토큰 발급 — Authorization Server가 Access Token을 발급한다
  8. 인증된 요청 — 클라이언트가 Bearer Token을 포함하여 MCP 서버에 요청한다
  9. 응답 — MCP 서버가 토큰을 검증하고 요청을 처리한다
TIP

MCP SDK의 OAuthClientProvider는 위 2~7단계를 자동으로 처리한다. 개발자는 redirect_handler와 callback_handler만 구현하면 된다.

실전 예제: 인증된 API 서버

토큰 검증이 포함된 MCP 서버와 이에 연결하는 클라이언트의 완성 예제이다.

서버 측 코드

"""인증이 적용된 MCP 서버 예제"""

from mcp.server.auth.provider import AccessToken, TokenVerifier
from mcp.server.auth.settings import AuthSettings
from mcp.server.fastmcp import FastMCP
from pydantic import AnyHttpUrl, BaseModel, Field


# --- 토큰 검증기 ---

# 실제 환경에서는 DB 또는 외부 인가 서버를 조회한다
VALID_TOKENS = {
    "token-admin-001": {
        "client_id": "admin-app",
        "scopes": ["read", "write", "admin"],
    },
    "token-user-001": {
        "client_id": "user-app",
        "scopes": ["read"],
    },
}


class DemoTokenVerifier(TokenVerifier):
    """데모용 토큰 검증기

    실제 프로덕션에서는 JWT 검증 또는 OAuth introspection을 사용한다.
    """

    async def verify_token(self, token: str) -> AccessToken | None:
        """토큰을 검증하고 AccessToken을 반환한다"""
        token_data = VALID_TOKENS.get(token)
        if token_data is None:
            return None

        return AccessToken(
            token=token,
            client_id=token_data["client_id"],
            scopes=token_data["scopes"],
        )


# --- MCP 서버 설정 ---

mcp = FastMCP(
    "Secure API Server",
    token_verifier=DemoTokenVerifier(),
    auth=AuthSettings(
        issuer_url=AnyHttpUrl("https://auth.example.com"),
        resource_server_url=AnyHttpUrl("http://localhost:8000"),
        required_scopes=["read"],
    ),
)


# --- 모델 정의 ---

class SecretData(BaseModel):
    """보호된 데이터"""
    id: int = Field(description="데이터 ID")
    content: str = Field(description="데이터 내용")
    classification: str = Field(description="보안 등급")


class UserInfo(BaseModel):
    """사용자 정보"""
    user_id: str = Field(description="사용자 ID")
    name: str = Field(description="사용자 이름")
    role: str = Field(description="역할")
    permissions: list[str] = Field(description="권한 목록")


# --- 도구 정의 ---

@mcp.tool()
def get_secret_data(data_id: int) -> SecretData:
    """보호된 데이터를 조회한다 (인증 필요)"""
    # 인증된 요청만 이 도구에 도달한다
    return SecretData(
        id=data_id,
        content=f"기밀 데이터 #{data_id}의 내용",
        classification="confidential",
    )


@mcp.tool()
def get_current_user() -> UserInfo:
    """현재 인증된 사용자 정보를 반환한다"""
    return UserInfo(
        user_id="user-001",
        name="홍길동",
        role="admin",
        permissions=["read", "write", "admin"],
    )


@mcp.tool()
def list_secrets() -> list[SecretData]:
    """보호된 데이터 목록을 반환한다"""
    return [
        SecretData(id=1, content="프로젝트 Alpha 접근 키", classification="top-secret"),
        SecretData(id=2, content="데이터베이스 연결 정보", classification="confidential"),
        SecretData(id=3, content="API 엔드포인트 목록", classification="internal"),
    ]


if __name__ == "__main__":
    mcp.run(transport="streamable-http")

클라이언트 측 코드

"""인증된 MCP 서버에 접속하는 클라이언트 예제"""

import asyncio
from mcp.client.streamable_http import streamable_http_client
from mcp.client.session import ClientSession


async def main():
    """인증된 MCP 서버에 접속하여 도구를 호출한다"""

    # Bearer 토큰을 헤더에 포함하여 접속한다
    headers = {
        "Authorization": "Bearer token-admin-001",
    }

    async with streamable_http_client(
        "http://localhost:8000/mcp",
        headers=headers,
    ) as (read_stream, write_stream, _):
        async with ClientSession(read_stream, write_stream) as session:
            await session.initialize()

            # 사용 가능한 도구 목록 조회
            tools = await session.list_tools()
            print("사용 가능한 도구:")
            for tool in tools.tools:
                print(f"  - {tool.name}: {tool.description}")

            print()

            # 현재 사용자 정보 조회
            user_result = await session.call_tool("get_current_user", {})
            print(f"현재 사용자: {user_result.content}")

            # 보호된 데이터 조회
            data_result = await session.call_tool("get_secret_data", {"data_id": 42})
            print(f"기밀 데이터: {data_result.content}")

            # 기밀 데이터 목록 조회
            list_result = await session.call_tool("list_secrets", {})
            print(f"데이터 목록: {list_result.content}")


if __name__ == "__main__":
    asyncio.run(main())

실행 방법

# 1. 서버 실행
python secure_server.py

# 2. 다른 터미널에서 클라이언트 실행
python secure_client.py
WARNING

이 예제는 학습 목적이므로 토큰을 하드코딩하고 있다. 실제 프로덕션 환경에서는 반드시 다음 사항을 지켜야 한다:

  • 토큰을 환경 변수 또는 시크릿 매니저에서 관리한다
  • HTTPS를 사용하여 토큰이 평문으로 전송되지 않도록 한다
  • JWT 서명 검증 또는 OAuth introspection으로 토큰을 검증한다
  • 토큰 만료 및 갱신 로직을 구현한다
TIP

MCP의 인증 체계는 OAuth 2.1 표준을 따르므로, Auth0, Keycloak, Google OAuth 등 기존 OAuth 인프라와 쉽게 통합할 수 있다. TokenVerifier에서 해당 인가 서버의 JWT를 검증하도록 구현하면 된다.
