첫 번째 MCP 서버

FastMCP로 첫 번째 MCP 서버를 만들고 실행한다

FastMCP 소개

FastMCP는 MCP Python SDK가 제공하는 고수준(high-level) API이다. 복잡한 프로토콜 세부사항을 숨기고, Python 데코레이터만으로 MCP 서버를 구축할 수 있게 해준다.

FastMCP의 주요 특징:

  • 데코레이터 기반으로 도구, 리소스, 프롬프트를 간단히 등록한다.
  • 함수의 타입 힌트를 자동으로 파싱하여 입력 스키마를 생성한다.
  • 함수의 docstring을 도구 설명으로 자동 사용한다.
  • 동기/비동기 함수를 모두 지원한다.

최소한의 서버

가장 간단한 MCP 서버를 만들어보자. 프로젝트 디렉토리에 server.py 파일을 생성하고 다음 코드를 작성한다. 

from mcp.server.fastmcp import FastMCP

# MCP 서버 인스턴스 생성
mcp = FastMCP("My First Server")

@mcp.tool()
def add(a: int, b: int) -> int:
    """두 수를 더한다"""
    return a + b

if __name__ == "__main__":
    mcp.run()

이 10줄의 코드가 완전한 MCP 서버이다. LLM이 add 도구를 호출하면 두 정수를 더한 결과를 반환한다.

코드 분석

각 줄이 무엇을 하는지 자세히 살펴보자.

1. FastMCP 임포트 및 인스턴스 생성

from mcp.server.fastmcp import FastMCP

# 서버 이름은 클라이언트가 서버를 식별하는 데 사용된다
mcp = FastMCP("My First Server")

FastMCP 클래스의 인스턴스를 생성한다. 인자로 전달하는 문자열은 서버의 이름이다. 이 이름은 클라이언트가 여러 서버를 구분할 때 사용된다.

2. 도구 등록

@mcp.tool()
def add(a: int, b: int) -> int:
    """두 수를 더한다"""
    return a + b

@mcp.tool() 데코레이터는 일반 Python 함수를 MCP 도구로 등록한다. SDK가 자동으로 처리하는 것들:

  • 함수 이름 → 도구 이름 (add)
  • 타입 힌트 (a: int, b: int) → 입력 JSON 스키마
  • docstring ("두 수를 더한다") → 도구 설명 (LLM이 언제 이 도구를 쓸지 판단하는 근거)
  • 반환 타입 → 자동 직렬화
WARNING

docstring은 반드시 작성해야 한다. LLM은 docstring을 보고 도구의 용도를 파악한다. docstring이 없으면 LLM이 도구를 올바르게 사용하지 못할 수 있다.

3. 서버 실행

if __name__ == "__main__":
    mcp.run()

mcp.run()은 서버를 시작하고 클라이언트의 연결을 대기한다. 기본적으로 stdio(표준 입출력) 전송 방식을 사용한다. 이는 서버와 클라이언트가 같은 머신에서 프로세스 간 통신을 하는 방식이다.

서버 실행

서버를 실행하는 방법은 두 가지가 있다.

방법 1: 직접 실행

# 서버 직접 실행 (stdio 모드)
uv run server.py

이 방식은 서버가 stdio로 입력을 대기하므로, 터미널에서는 아무 출력도 보이지 않는다. 실제 테스트를 위해서는 MCP Inspector를 사용하는 것이 좋다.

방법 2: MCP Inspector로 개발 모드 실행 (권장)

# MCP Inspector와 함께 개발 모드로 실행
uv run mcp dev server.py

mcp dev 명령은 MCP Inspector를 함께 시작한다. 브라우저에서 Inspector UI를 열어 도구를 직접 테스트할 수 있다.

TIP

개발 중에는 항상 mcp dev를 사용하는 것을 권장한다. Inspector에서 도구를 호출하고 결과를 즉시 확인할 수 있어 디버깅이 훨씬 편리하다.

MCP Inspector로 테스트

uv run mcp dev server.py를 실행하면 터미널에 Inspector URL이 표시된다. 브라우저에서 해당 URL(보통 http://localhost:5173)을 열면 Inspector UI가 나타난다.

Inspector에서 테스트하는 순서:

  1. 좌측 패널에서 "Tools" 탭을 클릭한다.
  2. 등록된 도구 목록에서 "add"를 선택한다.
  3. 파라미터 입력란에 a: 3, b: 5를 입력한다.
  4. "Run Tool" 버튼을 클릭한다.
  5. 결과 패널에 8이 표시되는 것을 확인한다.
INFO

MCP Inspector는 MCP SDK에 포함된 개발 도구이다. 서버의 모든 도구, 리소스, 프롬프트를 탐색하고 테스트할 수 있는 웹 UI를 제공한다.

리소스와 프롬프트 추가

서버에 도구만 있는 것이 아니라, 리소스(Resource)와 프롬프트(Prompt)도 추가해보자. 앞서 만든 server.py를 확장한다. 

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("My First Server")

# ── 도구 (Tool) ─────────────────────────────
@mcp.tool()
def add(a: int, b: int) -> int:
    """두 수를 더한다"""
    return a + b

@mcp.tool()
def multiply(a: int, b: int) -> int:
    """두 수를 곱한다"""
    return a * b

# ── 리소스 (Resource) ────────────────────────
@mcp.resource("greeting://{name}")
def get_greeting(name: str) -> str:
    """개인화된 인사말을 반환한다"""
    return f"안녕하세요, {name}님! MCP 서버에 오신 것을 환영한다."

@mcp.resource("config://app")
def get_config() -> dict:
    """애플리케이션 설정을 반환한다"""
    return {
        "app_name": "MCP Demo",
        "version": "1.0.0",
        "debug": True
    }

# ── 프롬프트 (Prompt) ────────────────────────
@mcp.prompt()
def review_code(code: str) -> str:
    """코드 리뷰 프롬프트"""
    return f"다음 코드를 리뷰해주세요:\n\n{code}"

@mcp.prompt()
def explain_error(error_message: str) -> str:
    """에러 설명 프롬프트"""
    return f"다음 에러 메시지를 분석하고 해결 방법을 제안해주세요:\n\n{error_message}"

if __name__ == "__main__":
    mcp.run()

각 프리미티브의 데코레이터를 정리하면 다음과 같다:

프리미티브 데코레이터 특징
Tool @mcp.tool() LLM이 호출. 계산, API 호출 등 동작 수행.
Resource @mcp.resource("uri://{param}") URI 패턴으로 데이터 노출. 읽기 전용.
Prompt @mcp.prompt() 재사용 가능한 프롬프트 템플릿.
TIP

리소스 URI에서 {name}과 같은 중괄호 부분은 경로 파라미터이다. greeting://Alice로 요청하면 name에 "Alice"가 전달된다.
