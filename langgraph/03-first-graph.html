첫 번째 그래프

StateGraph로 첫 번째 그래프를 만들고 실행한다

가장 간단한 그래프

이론을 배웠으니 이제 직접 코드를 작성하여 그래프를 만들어 본다. 먼저 두 개의 노드를 연결하는 가장 간단한 그래프부터 시작한다. 이 예제에서는 LLM 호출 없이 순수 Python 함수만으로 그래프의 동작 원리를 이해한다. 

graph LR
    START_NODE((START)) --> A["Node A"] --> B["Node B"] --> END_NODE((END))

"""01_first_graph.py - 가장 간단한 LangGraph 예제"""

from langgraph.graph import START, StateGraph
from typing_extensions import TypedDict


# 1. State 정의: 그래프에서 공유할 데이터 구조
class State(TypedDict):
    text: str


# 2. 노드 함수 정의: State를 받아 업데이트를 반환
def node_a(state: State) -> dict:
    """Node A: 텍스트에 처리 표시를 추가한다"""
    return {"text": state["text"] + " → A 처리됨"}


def node_b(state: State) -> dict:
    """Node B: 텍스트에 처리 표시를 추가한다"""
    return {"text": state["text"] + " → B 처리됨"}


# 3. 그래프 구성
graph = StateGraph(State)

# 4. 노드 등록
graph.add_node("node_a", node_a)
graph.add_node("node_b", node_b)

# 5. 엣지 연결: START → node_a → node_b (→ END는 자동)
graph.add_edge(START, "node_a")
graph.add_edge("node_a", "node_b")

# 6. 컴파일: 실행 가능한 객체 생성
app = graph.compile()

# 7. 실행: 초기 State를 넣고 결과를 받는다
result = app.invoke({"text": "시작"})
print(result)
# 출력: {'text': '시작 → A 처리됨 → B 처리됨'}

실행 결과:

$ python 01_first_graph.py
{'text': '시작 → A 처리됨 → B 처리됨'}

"시작"이라는 텍스트가 Node A를 거치면서 "시작 → A 처리됨"이 되고, 이어서 Node B를 거치면서 "시작 → A 처리됨 → B 처리됨"이 되었다. 각 노드가 순서대로 실행되며 State를 업데이트하는 과정이 잘 드러난다.

코드 한 줄씩 분석

위 코드의 각 부분이 어떤 역할을 하는지 상세히 살펴본다.

Step 1: State 정의

from typing_extensions import TypedDict

class State(TypedDict):
    text: str

TypedDict를 사용하여 그래프에서 공유할 데이터의 구조를 정의한다. 이 예제에서는 text라는 문자열 필드 하나만 가진다. State는 그래프의 모든 노드가 읽고 쓸 수 있는 공유 저장소와 같다. 타입 힌트를 통해 어떤 데이터가 오고 가는지 명확하게 알 수 있다.

Step 2: 노드 함수 정의

def node_a(state: State) -> dict:
    """Node A: 텍스트에 처리 표시를 추가한다"""
    return {"text": state["text"] + " → A 처리됨"}

노드 함수는 두 가지 규칙을 따른다:

  • 입력: 현재 State를 인자로 받는다.
  • 출력: 업데이트할 필드만 담은 딕셔너리를 반환한다.

중요한 점은 전체 State를 반환할 필요가 없다는 것이다. 변경하고 싶은 필드만 반환하면 LangGraph가 자동으로 기존 State에 병합(merge)한다. 이 패턴 덕분에 노드 함수를 간결하게 작성할 수 있다.

Step 3: 그래프 구성

# StateGraph 생성 — State 클래스를 전달
graph = StateGraph(State)

# 노드 등록 — (이름, 함수) 쌍으로 추가
graph.add_node("node_a", node_a)
graph.add_node("node_b", node_b)

# 엣지 연결 — 실행 순서를 정의
graph.add_edge(START, "node_a")   # 시작점 → node_a
graph.add_edge("node_a", "node_b") # node_a → node_b

StateGraph(State)로 그래프 인스턴스를 생성할 때 State 클래스를 전달한다. 이를 통해 LangGraph는 어떤 구조의 데이터가 그래프를 흐르는지 알게 된다.

add_node()는 노드를 등록하고, add_edge()는 노드 간의 연결을 정의한다. START는 LangGraph가 제공하는 특수 상수로, 그래프의 진입점을 나타낸다.

Step 4: 컴파일과 실행

# 컴파일: 그래프 구조를 검증하고 실행 가능한 객체를 생성
app = graph.compile()

# 실행: 초기 State를 전달하고 최종 State를 받는다
result = app.invoke({"text": "시작"})

compile()은 정의한 그래프의 구조를 검증하고 실행 가능한 CompiledGraph 객체를 반환한다. 이 단계에서 연결되지 않은 노드, 순환 참조 등의 문제가 발견되면 에러를 발생시킨다.

invoke()는 초기 State를 딕셔너리로 전달받아 그래프를 처음부터 끝까지 실행하고 최종 State를 반환한다.

TIP

그래프 정의(노드/엣지 추가)와 실행(invoke)이 분리되어 있다는 것이 중요하다. 한 번 컴파일한 그래프는 여러 번 실행할 수 있으며, 매번 다른 초기 State를 전달할 수 있다.

END 노드 명시적 사용

위 예제에서 마지막 노드(node_b)에 연결된 다음 노드가 없으면 LangGraph가 자동으로 그래프를 종료한다. 하지만 명시적으로 END를 사용하는 것이 권장된다. 코드의 의도가 명확해지고, 특히 조건부 분기가 있는 경우 필수적이다. 

from langgraph.graph import START, END, StateGraph
from typing_extensions import TypedDict


class State(TypedDict):
    text: str


def node_a(state: State) -> dict:
    return {"text": state["text"] + " → A 처리됨"}


def node_b(state: State) -> dict:
    return {"text": state["text"] + " → B 처리됨"}


graph = StateGraph(State)
graph.add_node("node_a", node_a)
graph.add_node("node_b", node_b)

# 명시적으로 시작과 끝을 연결
graph.add_edge(START, "node_a")
graph.add_edge("node_a", "node_b")
graph.add_edge("node_b", END)  # 명시적 종료

app = graph.compile()
result = app.invoke({"text": "시작"})
print(result)
# {'text': '시작 → A 처리됨 → B 처리됨'}
INFO

END를 명시하지 않아도 마지막 노드에서 자동으로 종료되지만, 조건부 엣지에서 특정 조건일 때 즉시 종료하려면 END를 반드시 사용해야 한다. 좋은 습관으로 항상 명시적으로 END를 연결하는 것을 권장한다.

3개 이상 노드 연결

노드를 더 추가하여 긴 파이프라인을 만들 수 있다. 아래 예제에서는 4개의 노드를 순차적으로 연결한다. 

graph LR
    START_NODE((START)) --> V["입력검증"] --> P["전처리"] --> AN["분석"] --> F["출력포맷"] --> END_NODE((END))

"""4개 노드 파이프라인 예제"""

from langgraph.graph import START, END, StateGraph
from typing_extensions import TypedDict


class State(TypedDict):
    text: str
    word_count: int
    is_valid: bool


def validate_input(state: State) -> dict:
    """입력 검증: 텍스트가 비어있지 않은지 확인"""
    text = state["text"]
    is_valid = len(text.strip()) > 0
    print(f"[검증] 입력 텍스트: '{text}' → 유효: {is_valid}")
    return {"is_valid": is_valid}


def preprocess(state: State) -> dict:
    """전처리: 텍스트를 정리한다"""
    text = state["text"].strip().lower()
    print(f"[전처리] 정리된 텍스트: '{text}'")
    return {"text": text}


def analyze(state: State) -> dict:
    """분석: 단어 수를 센다"""
    words = state["text"].split()
    count = len(words)
    print(f"[분석] 단어 수: {count}")
    return {"word_count": count}


def format_output(state: State) -> dict:
    """출력 포맷: 최종 결과를 정리한다"""
    result = f"분석 결과: '{state['text']}' (단어 {state['word_count']}개)"
    print(f"[출력] {result}")
    return {"text": result}


# 그래프 구성
graph = StateGraph(State)

graph.add_node("validate", validate_input)
graph.add_node("preprocess", preprocess)
graph.add_node("analyze", analyze)
graph.add_node("format", format_output)

graph.add_edge(START, "validate")
graph.add_edge("validate", "preprocess")
graph.add_edge("preprocess", "analyze")
graph.add_edge("analyze", "format")
graph.add_edge("format", END)

app = graph.compile()

# 실행
result = app.invoke({
    "text": "  Hello LangGraph World  ",
    "word_count": 0,
    "is_valid": False
})

print("\n최종 결과:", result)

실행 결과:

[검증] 입력 텍스트: '  Hello LangGraph World  ' → 유효: True
[전처리] 정리된 텍스트: 'hello langgraph world'
[분석] 단어 수: 3
[출력] 분석 결과: 'hello langgraph world' (단어 3개)

최종 결과: {'text': "분석 결과: 'hello langgraph world' (단어 3개)", 'word_count': 3, 'is_valid': True}

각 노드가 순서대로 실행되면서 State가 점진적으로 업데이트되는 것을 확인할 수 있다. validate 노드는 is_valid만, preprocess 노드는 text만, analyze 노드는 word_count만 업데이트한다. 각 노드가 자기 관심사에만 집중하는 것이 핵심이다.

실전 예제: 텍스트 처리 파이프라인

지금까지 배운 내용을 종합하여 좀 더 실용적인 예제를 만들어 본다. 이 파이프라인은 입력 텍스트를 받아 정제 → 분석 → 요약의 3단계를 거쳐 처리한다. 

graph LR
    START_NODE((START)) --> C["정제\n(clean)"] --> AN["분석\n(analyze)"] --> S["요약\n(summarize)"] --> END_NODE((END))

"""실전 예제: 텍스트 처리 파이프라인"""

from langgraph.graph import START, END, StateGraph
from typing_extensions import TypedDict
import re


class TextState(TypedDict):
    """텍스트 처리 파이프라인의 상태"""
    raw_text: str       # 원본 텍스트
    clean_text: str     # 정제된 텍스트
    stats: dict         # 분석 통계 결과
    summary: str        # 최종 요약


def clean_text(state: TextState) -> dict:
    """정제 노드: 텍스트를 깨끗하게 정리한다"""
    raw = state["raw_text"]

    # 1. 앞뒤 공백 제거
    cleaned = raw.strip()

    # 2. 연속된 공백을 하나로 통합
    cleaned = re.sub(r'\s+', ' ', cleaned)

    # 3. 특수 문자 제거 (한글, 영문, 숫자, 기본 문장부호만 유지)
    cleaned = re.sub(r'[^\w\s가-힣.,!?]', '', cleaned)

    print(f"[정제] 원본 길이: {len(raw)} → 정제 후: {len(cleaned)}")
    return {"clean_text": cleaned}


def analyze_text(state: TextState) -> dict:
    """분석 노드: 텍스트의 통계 정보를 추출한다"""
    text = state["clean_text"]

    # 단어 분리
    words = text.split()

    # 문장 분리 (마침표, 느낌표, 물음표 기준)
    sentences = re.split(r'[.!?]+', text)
    sentences = [s.strip() for s in sentences if s.strip()]

    # 통계 계산
    stats = {
        "char_count": len(text),           # 문자 수
        "word_count": len(words),           # 단어 수
        "sentence_count": len(sentences),   # 문장 수
        "avg_word_length": round(
            sum(len(w) for w in words) / max(len(words), 1), 1
        ),                                  # 평균 단어 길이
        "unique_words": len(set(w.lower() for w in words)),  # 고유 단어 수
    }

    print(f"[분석] 문자: {stats['char_count']}, "
          f"단어: {stats['word_count']}, "
          f"문장: {stats['sentence_count']}")
    return {"stats": stats}


def summarize_text(state: TextState) -> dict:
    """요약 노드: 분석 결과를 사람이 읽기 쉬운 형태로 요약한다"""
    stats = state["stats"]
    text = state["clean_text"]

    # 첫 50자를 미리보기로 사용
    preview = text[:50] + "..." if len(text) > 50 else text

    summary = (
        f"텍스트 분석 요약\n"
        f"{'=' * 40}\n"
        f"미리보기: {preview}\n"
        f"문자 수: {stats['char_count']:,}자\n"
        f"단어 수: {stats['word_count']:,}개\n"
        f"문장 수: {stats['sentence_count']}개\n"
        f"평균 단어 길이: {stats['avg_word_length']}자\n"
        f"고유 단어 수: {stats['unique_words']}개\n"
        f"{'=' * 40}"
    )

    print(f"[요약] 요약문 생성 완료")
    return {"summary": summary}


# 그래프 구성
graph = StateGraph(TextState)

graph.add_node("clean", clean_text)
graph.add_node("analyze", analyze_text)
graph.add_node("summarize", summarize_text)

graph.add_edge(START, "clean")
graph.add_edge("clean", "analyze")
graph.add_edge("analyze", "summarize")
graph.add_edge("summarize", END)

# 컴파일
app = graph.compile()

# 그래프 구조 확인
app.get_graph().print_ascii()

# 실행
sample_text = """
    LangGraph는   LangChain 팀이 만든 프레임워크이다.
    에이전트의 실행 흐름을   그래프로 모델링한다!
    각 노드가 작업을 수행하고, 엣지가 흐름을 연결한다.
    이를 통해 복잡한 AI 워크플로우를 쉽게 구축할 수 있다.
"""

result = app.invoke({
    "raw_text": sample_text,
    "clean_text": "",
    "stats": {},
    "summary": ""
})

print("\n" + result["summary"])

[분석] 문자: 129, 단어: 18, 문장: 4

이 예제에서 주목할 점

  • 관심사 분리: 각 노드는 하나의 작업에만 집중한다. 정제 노드는 텍스트를 정리하고, 분석 노드는 통계를 내고, 요약 노드는 결과를 포맷팅한다.
  • 점진적 상태 구축: State가 단계마다 풍부해진다. 처음에는 raw_text만 있지만, 파이프라인을 거치면서 clean_text, stats, summary가 채워진다.
  • 재사용성: 각 노드 함수는 독립적이어서 다른 그래프에서도 재사용할 수 있다. 예를 들어 clean_text 노드를 다른 파이프라인에서도 사용할 수 있다.
  • 테스트 용이성: 각 노드 함수를 개별적으로 테스트할 수 있다. State 딕셔너리를 직접 만들어 함수에 전달하면 된다.
INFO

이 예제에서는 LLM을 사용하지 않았지만, 실제 프로덕션에서는 summarize_text 노드에서 LLM을 호출하여 자연어 요약을 생성하거나, analyze_text 노드에서 감정 분석을 수행하는 등 AI 기능을 노드에 자연스럽게 통합할 수 있다. 이후 챕터에서 LLM 연동 방법을 다룬다.

스트리밍 실행: stream()

invoke()는 모든 노드의 실행이 완료될 때까지 기다린 후 최종 결과를 반환한다. 반면 stream()을 사용하면 각 노드가 실행을 마칠 때마다 중간 결과를 받을 수 있다. 

"""stream()을 사용한 실시간 진행 확인"""

# invoke() 대신 stream() 사용
for step in app.stream({
    "raw_text": "LangGraph는 훌륭한 프레임워크이다.",
    "clean_text": "",
    "stats": {},
    "summary": ""
}):
    # step은 {노드이름: 업데이트된_State} 형태의 딕셔너리
    node_name = list(step.keys())[0]
    print(f"\n--- {node_name} 노드 완료 ---")
    print(f"업데이트: {step[node_name]}")
    print()

stream()은 제너레이터를 반환하므로 for 루프로 각 단계의 결과를 순차적으로 처리할 수 있다. 장시간 실행되는 그래프에서 사용자에게 진행 상황을 보여줄 때 유용하다.

정리

이번 챕터에서 배운 내용을 정리한다:

  • StateGraph에 노드(함수)와 엣지(연결)를 추가하여 그래프를 구성한다.
  • 노드 함수는 State를 받아 업데이트 딕셔너리를 반환하는 규칙을 따른다.
  • START와 END로 그래프의 시작과 끝을 명시한다.
  • compile()로 컴파일하고 invoke()로 실행한다.
  • print_ascii()와 draw_mermaid_png()로 그래프를 시각화할 수 있다.
  • stream()을 사용하면 각 노드의 실행 결과를 실시간으로 받을 수 있다.

다음 챕터에서는 State를 더 깊이 이해하고, Reducer, MessagesState 등 고급 State 관리 기법을 배운다.
