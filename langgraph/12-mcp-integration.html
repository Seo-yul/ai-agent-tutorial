MCP 연동

LangGraph 에이전트에서 MCP 서버의 도구를 사용한다

MCP + LangGraph = 최강 조합

이 튜토리얼의 MCP 섹션에서 우리는 MCP(Model Context Protocol)를 사용하여 표준화된 도구 서버를 만드는 방법을 배웠다. LangGraph 섹션에서는 정교한 에이전트 흐름을 제어하는 방법을 배웠다. 이 두 가지를 결합하면 가장 강력한 AI 에이전트 시스템을 구축할 수 있다.

INFO

MCP 도구는 LangGraph 입장에서 일반 LangChain 도구와 동일하게 취급된다. 에이전트는 도구가 MCP 서버에서 온 것인지, 로컬에서 정의된 것인지 구분하지 않는다. 이것이 MCP 표준화의 핵심 가치이다 — 도구의 출처에 관계없이 동일한 인터페이스로 사용할 수 있다.

langchain-mcp-adapters

MCP 서버의 도구를 LangGraph에서 사용하려면 langchain-mcp-adapters 라이브러리가 필요하다. 이 라이브러리는 MCP 클라이언트 역할을 하며, MCP 서버가 제공하는 도구를 LangChain의 BaseTool 형식으로 자동 변환해 준다. 

# 필수 패키지 설치
pip install langchain-mcp-adapters langgraph langchain-anthropic

langchain-mcp-adapters의 핵심 클래스는 MultiServerMCPClient이다. 이 클래스는 여러 MCP 서버에 동시에 연결하고, 각 서버의 도구를 LangChain 도구 리스트로 통합하여 제공한다.

지원하는 트랜스포트

트랜스포트 설정 키 사용 사례
stdio command, args 로컬에서 실행되는 MCP 서버 (프로세스 기반)
streamable_http url 원격 HTTP 기반 MCP 서버
sse url Server-Sent Events 기반 (레거시)

stdio MCP 서버 연결

가장 일반적인 사용 패턴은 stdio 트랜스포트로 로컬 MCP 서버를 연결하는 것이다. stdio 방식에서는 MCP 서버가 별도의 프로세스로 실행되며, 표준 입출력(stdin/stdout)을 통해 클라이언트와 통신한다.

간단한 MCP 서버 준비

먼저 연결할 MCP 서버를 준비한다. 간단한 수학 도구를 제공하는 MCP 서버이다. 

# math_server.py — 수학 도구를 제공하는 MCP 서버
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("수학 도구 서버")


@mcp.tool()
def add(a: float, b: float) -> float:
    """두 수를 더한다"""
    return a + b


@mcp.tool()
def multiply(a: float, b: float) -> float:
    """두 수를 곱한다"""
    return a * b


@mcp.tool()
def factorial(n: int) -> int:
    """n의 팩토리얼을 계산한다 (n!)"""
    if n < 0:
        raise ValueError("음수의 팩토리얼은 정의되지 않는다")
    if n <= 1:
        return 1
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result


@mcp.tool()
def power(base: float, exponent: float) -> float:
    """base의 exponent 거듭제곱을 계산한다"""
    return base ** exponent


if __name__ == "__main__":
    mcp.run(transport="stdio")

LangGraph에서 stdio MCP 서버 사용

import asyncio
from langchain_mcp_adapters.client import MultiServerMCPClient
from langgraph.prebuilt import create_react_agent
from langchain_anthropic import ChatAnthropic

model = ChatAnthropic(model="claude-sonnet-4-20250514")


async def main():
    # MultiServerMCPClient를 비동기 컨텍스트 매니저로 사용
    async with MultiServerMCPClient(
        {
            "math": {
                "command": "python",
                "args": ["math_server.py"],
                "transport": "stdio",
            }
        }
    ) as client:
        # MCP 서버의 도구를 LangChain 도구로 변환
        tools = client.get_tools()

        print(f"사용 가능한 도구: {[t.name for t in tools]}")
        # 출력: ['add', 'multiply', 'factorial', 'power']

        # ReAct 에이전트 생성
        agent = create_react_agent(model, tools)

        # 에이전트 실행
        result = await agent.ainvoke({
            "messages": [
                {"role": "user", "content": "15의 팩토리얼을 계산하고, 그 결과에 2를 곱해줘"}
            ]
        })

        # 결과 출력
        for msg in result["messages"]:
            print(f"[{msg.type}] {msg.content}")


asyncio.run(main())
TIP

uv 패키지 매니저를 사용하는 경우, command를 "uv"로, args를 ["run", "math_server.py"]로 설정한다. 이렇게 하면 가상 환경 활성화 없이 바로 MCP 서버를 실행할 수 있다.

실행 흐름 상세

위 코드가 실행되면 내부적으로 다음과 같은 과정이 일어난다:

  1. MultiServerMCPClient가 math_server.py를 별도 프로세스로 실행한다
  2. MCP 프로토콜을 통해 서버에서 사용 가능한 도구 목록을 조회한다
  3. 각 MCP 도구를 LangChain의 BaseTool 객체로 변환한다 (이름, 설명, 파라미터 스키마 포함)
  4. create_react_agent가 이 도구들을 사용하는 ReAct 에이전트 그래프를 생성한다
  5. 에이전트가 LLM에게 도구 목록을 알려주고, LLM이 필요한 도구를 호출한다
  6. 도구 호출 요청이 MCP 프로토콜을 통해 서버로 전달되고, 결과가 에이전트로 반환된다
  7. 비동기 컨텍스트 매니저가 종료되면 MCP 서버 프로세스가 정리된다

Streamable HTTP MCP 서버 연결

원격에서 실행되는 MCP 서버에 연결할 때는 Streamable HTTP 트랜스포트를 사용한다. 이 방식은 MCP 서버가 HTTP 엔드포인트로 배포되어 있을 때 적합하며, 네트워크를 통해 도구를 호출한다.

HTTP MCP 서버 준비

# weather_server.py — 날씨 정보를 제공하는 HTTP MCP 서버
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("날씨 도구 서버")


@mcp.tool()
def get_weather(city: str) -> dict:
    """지정한 도시의 현재 날씨를 조회한다"""
    # 실제로는 날씨 API를 호출한다
    weather_data = {
        "서울": {"temp": 22, "condition": "맑음", "humidity": 45},
        "부산": {"temp": 25, "condition": "구름 조금", "humidity": 60},
        "제주": {"temp": 27, "condition": "흐림", "humidity": 70},
    }
    data = weather_data.get(city, {"temp": 20, "condition": "정보 없음", "humidity": 50})
    return {
        "city": city,
        "temperature": data["temp"],
        "condition": data["condition"],
        "humidity": data["humidity"],
    }


@mcp.tool()
def get_forecast(city: str, days: int = 3) -> list[dict]:
    """지정한 도시의 날씨 예보를 조회한다"""
    # 실제로는 날씨 예보 API를 호출한다
    return [
        {"day": i + 1, "city": city, "temp": 20 + i, "condition": "맑음"}
        for i in range(days)
    ]


if __name__ == "__main__":
    # Streamable HTTP 트랜스포트로 서버 실행
    mcp.run(transport="streamable-http", host="0.0.0.0", port=8000)

LangGraph에서 HTTP MCP 서버 사용

import asyncio
from langchain_mcp_adapters.client import MultiServerMCPClient
from langgraph.prebuilt import create_react_agent
from langchain_anthropic import ChatAnthropic

model = ChatAnthropic(model="claude-sonnet-4-20250514")


async def main():
    async with MultiServerMCPClient(
        {
            "weather": {
                # 원격 MCP 서버의 URL (Streamable HTTP 엔드포인트)
                "url": "http://localhost:8000/mcp",
                "transport": "streamable_http",
            }
        }
    ) as client:
        tools = client.get_tools()
        print(f"날씨 도구: {[t.name for t in tools]}")
        # 출력: ['get_weather', 'get_forecast']

        agent = create_react_agent(model, tools)

        result = await agent.ainvoke({
            "messages": [
                {"role": "user", "content": "서울과 부산의 현재 날씨를 비교해줘"}
            ]
        })

        # 최종 응답만 출력
        print(result["messages"][-1].content)

INFO

Streamable HTTP와 SSE(Server-Sent Events)의 차이점: SSE는 레거시 방식으로 단방향 스트리밍만 지원하는 반면, Streamable HTTP는 양방향 통신을 지원하며 MCP 프로토콜의 권장 방식이다. 새 프로젝트에서는 항상 streamable_http를 사용하는 것이 좋다.

여러 MCP 서버 동시 연결

MultiServerMCPClient의 진정한 힘은 이름에서 알 수 있듯이 여러 MCP 서버를 동시에 연결할 수 있다는 점이다. 각 서버의 도구가 하나의 리스트로 통합되므로, 에이전트는 마치 모든 도구가 로컬에 있는 것처럼 자연스럽게 사용할 수 있다. 

import asyncio
from langchain_mcp_adapters.client import MultiServerMCPClient
from langgraph.prebuilt import create_react_agent
from langchain_anthropic import ChatAnthropic

model = ChatAnthropic(model="claude-sonnet-4-20250514")


async def main():
    # 여러 MCP 서버를 동시에 연결
    async with MultiServerMCPClient(
        {
            # stdio 트랜스포트: 로컬 수학 서버
            "math": {
                "command": "python",
                "args": ["math_server.py"],
                "transport": "stdio",
            },
            # streamable_http 트랜스포트: 원격 날씨 서버
            "weather": {
                "url": "http://localhost:8000/mcp",
                "transport": "streamable_http",
            },
            # stdio 트랜스포트: 로컬 파일 서버
            "files": {
                "command": "python",
                "args": ["file_server.py"],
                "transport": "stdio",
            },
        }
    ) as client:
        # 모든 서버의 도구가 하나의 리스트로 통합된다
        all_tools = client.get_tools()
        print(f"전체 도구 수: {len(all_tools)}")
        print(f"도구 목록: {[t.name for t in all_tools]}")
        # 출력 예: ['add', 'multiply', 'factorial', 'power',
        #          'get_weather', 'get_forecast',
        #          'read_file', 'write_file', 'list_files']

        # 모든 도구를 사용하는 에이전트 생성
        agent = create_react_agent(model, all_tools)

        # 에이전트는 필요에 따라 적절한 도구를 자동으로 선택한다
        result = await agent.ainvoke({
            "messages": [
                {"role": "user", "content": (
                    "서울의 현재 기온을 확인하고, "
                    "그 기온에 1.8을 곱한 후 32를 더해서 화씨로 변환해줘"
                )}
            ]
        })

        print(result["messages"][-1].content)


asyncio.run(main())

여러 MCP 서버를 연결할 때 도구 이름이 충돌하지 않도록 주의해야 한다. 예를 들어 두 서버가 모두 search라는 도구를 제공하면 문제가 발생할 수 있다. MCP 서버를 설계할 때 도구 이름에 접두사를 붙이거나, MultiServerMCPClient의 설정으로 이름 충돌을 관리해야 한다.

커스텀 그래프에 MCP 도구 통합

create_react_agent는 빠르게 에이전트를 만들기에 편리하지만, 복잡한 워크플로우에서는 그래프를 직접 설계해야 할 때가 있다. 이 경우 MCP 도구를 직접 그래프의 ToolNode에 통합할 수 있다. 

import asyncio
from langchain_mcp_adapters.client import MultiServerMCPClient
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.prebuilt import ToolNode
from langchain_anthropic import ChatAnthropic
from langchain_core.messages import SystemMessage

model = ChatAnthropic(model="claude-sonnet-4-20250514")


async def main():
    async with MultiServerMCPClient(
        {
            "math": {
                "command": "python",
                "args": ["math_server.py"],
                "transport": "stdio",
            },
            "weather": {
                "url": "http://localhost:8000/mcp",
                "transport": "streamable_http",
            },
        }
    ) as client:
        # MCP 도구를 가져온다
        tools = client.get_tools()

        # 모델에 도구를 바인딩한다
        model_with_tools = model.bind_tools(tools)

        # ── 노드 함수 정의 ────────────────────────────────

        def agent_node(state: MessagesState) -> dict:
            """에이전트 노드: LLM이 도구 호출 여부를 결정한다"""
            response = model_with_tools.invoke([
                SystemMessage(content="당신은 유능한 AI 어시스턴트이다. 필요하면 도구를 사용하라."),
                *state["messages"]
            ])
            return {"messages": [response]}

        def should_use_tools(state: MessagesState) -> str:
            """마지막 메시지에 도구 호출이 있으면 tools 노드로, 없으면 종료"""
            last_message = state["messages"][-1]
            if hasattr(last_message, "tool_calls") and last_message.tool_calls:
                return "tools"
            return END

        # ── 그래프 조립 ───────────────────────────────────

        # ToolNode는 MCP 도구를 포함한 모든 도구를 자동으로 실행한다
        tool_node = ToolNode(tools)

        graph = StateGraph(MessagesState)
        graph.add_node("agent", agent_node)
        graph.add_node("tools", tool_node)

        graph.add_edge(START, "agent")
        graph.add_conditional_edges(
            "agent",
            should_use_tools,
            {
                "tools": "tools",
                END: END,
            }
        )
        # 도구 실행 후 다시 에이전트로 돌아간다 (결과를 보고 추가 작업 결정)
        graph.add_edge("tools", "agent")

        app = graph.compile()

        # ── 실행 ──────────────────────────────────────────

        result = await app.ainvoke({
            "messages": [
                {"role": "user", "content": "서울 기온의 세제곱을 계산해줘"}
            ]
        })

        print(result["messages"][-1].content)


asyncio.run(main())
TIP

커스텀 그래프의 장점은 도구 호출 전후에 추가 로직을 넣을 수 있다는 것이다. 예를 들어 도구 호출 전에 권한 검사를 하거나, 도구 호출 결과를 캐싱하거나, 특정 도구 호출에 대해 사람의 승인을 요청하는 등의 로직을 그래프에 추가할 수 있다.

도구 호출에 Human-in-the-loop 추가

MCP 도구 중 민감한 작업(파일 삭제, 데이터 수정 등)을 수행하는 도구가 있다면, 실행 전에 사람의 승인을 받는 것이 안전하다. LangGraph의 interrupt를 활용하면 도구 실행을 일시 정지하고 사용자 승인을 기다릴 수 있다. 

from langgraph.checkpoint.memory import InMemorySaver
from langgraph.types import interrupt, Command

# 체크포인터 설정 (interrupt 사용에 필수)
checkpointer = InMemorySaver()


def human_approval_tool_node(state: MessagesState) -> dict:
    """민감한 도구 호출 전에 사람의 승인을 요청한다"""
    last_message = state["messages"][-1]

    if hasattr(last_message, "tool_calls"):
        for tool_call in last_message.tool_calls:
            # 민감한 도구인지 확인
            sensitive_tools = ["write_file", "delete_file", "execute_command"]
            if tool_call["name"] in sensitive_tools:
                # 사람에게 승인 요청 — 여기서 그래프 실행이 일시 정지된다
                approval = interrupt({
                    "question": f"다음 도구를 실행해도 되겠는가?",
                    "tool_name": tool_call["name"],
                    "tool_args": tool_call["args"],
                })

                if approval.get("approved") != True:
                    # 거부된 경우 도구를 실행하지 않는다
                    from langchain_core.messages import ToolMessage
                    return {
                        "messages": [
                            ToolMessage(
                                content="사용자가 도구 실행을 거부했다.",
                                tool_call_id=tool_call["id"],
                            )
                        ]
                    }

    # 승인된 경우 또는 민감하지 않은 도구인 경우 정상 실행
    return tool_node.invoke(state)

LangGraph 연동을 위한 MCP 서버 설계 팁

TIP

LangGraph에서 사용할 MCP 서버를 설계할 때 다음을 고려해야 한다:

  • 도구 이름을 명확하고 고유하게 짓는다 (예: math_add 대신 add)
  • docstring을 상세하게 작성한다 — LLM이 이 설명을 보고 도구 사용 여부를 결정한다
  • 파라미터 타입 힌트를 정확히 지정한다 — 이것이 도구의 입력 스키마가 된다
  • 반환 값을 구조화된 형태(dict, list)로 제공하면 LLM이 결과를 더 잘 해석한다
  • 에러 처리를 명확히 한다 — 예외를 발생시키면 에이전트가 적절히 대응할 수 있다

실전 예제: MCP 기반 리서치 에이전트

이번 섹션에서는 처음부터 끝까지 완전한 예제를 만든다. 두 개의 MCP 서버(검색 도구, 계산 도구)를 만들고, LangGraph 에이전트에서 이 두 서버를 동시에 연결하여 사용자의 질문에 답변하는 리서치 에이전트를 구축한다.

mcp = FastMCP("검색 도구 서버")

@mcp.tool()
def search(query: str) -> dict:
    """주어진 쿼리로 지식 베이스를 검색한다

    Args:
        query: 검색할 키워드 또는 질문
    """
    results = []
    query_lower = query.lower()

    for topic, data in KNOWLEDGE_BASE.items():
        # 주제명이나 내용에 쿼리가 포함되어 있으면 결과에 추가
        if (query_lower in topic.lower() or
            query_lower in data["summary"].lower() or
            query_lower in data["details"].lower()):
            results.append({
                "topic": topic,
                "summary": data["summary"],
                "details": data["details"],
            })

    if not results:
        return {"found": False, "message": f"'{query}'에 대한 검색 결과가 없다."}

    return {"found": True, "count": len(results), "results": results}


@mcp.tool()
def get_stats(topic: str) -> dict:
    """특정 주제의 통계 정보를 조회한다

    Args:
        topic: 통계를 조회할 주제명
    """
    for key, data in KNOWLEDGE_BASE.items():
        if topic.lower() in key.lower():
            return {
                "topic": key,
                "stats": data.get("stats", {}),
            }
    return {"error": f"'{topic}'에 대한 통계 정보를 찾을 수 없다."}


@mcp.tool()
def list_topics() -> list[str]:
    """사용 가능한 모든 주제 목록을 반환한다"""
    return list(KNOWLEDGE_BASE.keys())


if __name__ == "__main__":
    mcp.run(transport="stdio")

Step 2: 계산 MCP 서버 만들기

# calc_server.py — 계산 도구를 제공하는 MCP 서버
import math
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("계산 도구 서버")


@mcp.tool()
def calculate(expression: str) -> dict:
    """수학 표현식을 안전하게 계산한다

    Args:
        expression: 계산할 수학 표현식 (예: "2 + 3 * 4")
    """
    # 안전한 수학 함수만 허용
    allowed_names = {
        "abs": abs, "round": round, "min": min, "max": max,
        "sum": sum, "pow": pow, "len": len,
        "sqrt": math.sqrt, "log": math.log, "log10": math.log10,
        "sin": math.sin, "cos": math.cos, "tan": math.tan,
        "pi": math.pi, "e": math.e,
        "ceil": math.ceil, "floor": math.floor,
    }

    try:
        result = eval(expression, {"__builtins__": {}}, allowed_names)
        return {
            "expression": expression,
            "result": result,
            "type": type(result).__name__,
        }
    except Exception as e:
        return {"error": f"계산 오류: {str(e)}", "expression": expression}


@mcp.tool()
def statistics(numbers: list[float]) -> dict:
    """숫자 목록의 통계를 계산한다

    Args:
        numbers: 통계를 계산할 숫자 목록
    """
    if not numbers:
        return {"error": "빈 목록이 전달되었다."}

    n = len(numbers)
    mean = sum(numbers) / n
    sorted_nums = sorted(numbers)
    median = sorted_nums[n // 2] if n % 2 == 1 else (sorted_nums[n // 2 - 1] + sorted_nums[n // 2]) / 2
    variance = sum((x - mean) ** 2 for x in numbers) / n
    std_dev = math.sqrt(variance)

    return {
        "count": n,
        "sum": sum(numbers),
        "mean": mean,
        "median": median,
        "min": min(numbers),
        "max": max(numbers),
        "std_dev": round(std_dev, 4),
        "variance": round(variance, 4),
    }


@mcp.tool()
def unit_convert(value: float, from_unit: str, to_unit: str) -> dict:
    """단위를 변환한다

    Args:
        value: 변환할 값
        from_unit: 원래 단위 (예: "km", "celsius", "kg")
        to_unit: 변환할 단위 (예: "miles", "fahrenheit", "pounds")
    """
    conversions = {
        ("km", "miles"): lambda v: v * 0.621371,
        ("miles", "km"): lambda v: v * 1.60934,
        ("celsius", "fahrenheit"): lambda v: v * 9/5 + 32,
        ("fahrenheit", "celsius"): lambda v: (v - 32) * 5/9,
        ("kg", "pounds"): lambda v: v * 2.20462,
        ("pounds", "kg"): lambda v: v * 0.453592,
        ("meters", "feet"): lambda v: v * 3.28084,
        ("feet", "meters"): lambda v: v * 0.3048,
    }

    key = (from_unit.lower(), to_unit.lower())
    if key not in conversions:
        return {"error": f"'{from_unit}' → '{to_unit}' 변환은 지원하지 않는다."}

    result = conversions[key](value)
    return {
        "original": {"value": value, "unit": from_unit},
        "converted": {"value": round(result, 4), "unit": to_unit},
    }


if __name__ == "__main__":
    mcp.run(transport="stdio")

Step 3: LangGraph 리서치 에이전트 구축

# research_agent.py — MCP 기반 리서치 에이전트
"""
두 개의 MCP 서버(검색, 계산)를 동시에 연결하여
사용자의 질문에 답변하는 LangGraph 에이전트.
"""
import asyncio
from langchain_mcp_adapters.client import MultiServerMCPClient
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.prebuilt import ToolNode
from langchain_anthropic import ChatAnthropic
from langchain_core.messages import SystemMessage


model = ChatAnthropic(model="claude-sonnet-4-20250514")

# 에이전트의 시스템 프롬프트
SYSTEM_PROMPT = """당신은 리서치 전문 AI 어시스턴트이다.

사용자의 질문에 답변하기 위해 다음 도구를 활용한다:
- search: 지식 베이스에서 정보를 검색한다
- get_stats: 특정 주제의 통계 정보를 조회한다
- list_topics: 사용 가능한 주제 목록을 확인한다
- calculate: 수학 계산을 수행한다
- statistics: 숫자 데이터의 통계를 계산한다
- unit_convert: 단위를 변환한다

답변 시 다음 원칙을 따른다:
1. 먼저 관련 정보를 검색한다
2. 필요하면 계산 도구를 사용한다
3. 검색 결과와 계산 결과를 종합하여 명확한 답변을 제공한다
4. 근거가 되는 정보의 출처를 명시한다"""


async def main():
    # 두 MCP 서버에 동시 연결
    async with MultiServerMCPClient(
        {
            "search": {
                "command": "python",
                "args": ["search_server.py"],
                "transport": "stdio",
            },
            "calc": {
                "command": "python",
                "args": ["calc_server.py"],
                "transport": "stdio",
            },
        }
    ) as client:
        # MCP 도구를 가져와서 LangChain 도구로 변환
        tools = client.get_tools()
        model_with_tools = model.bind_tools(tools)
        tool_node = ToolNode(tools)

        print(f"연결된 도구: {[t.name for t in tools]}")

        # ── 노드 함수 정의 ────────────────────────────────

        def agent_node(state: MessagesState) -> dict:
            """에이전트 노드: 질문을 분석하고 도구 호출을 결정한다"""
            response = model_with_tools.invoke([
                SystemMessage(content=SYSTEM_PROMPT),
                *state["messages"]
            ])
            return {"messages": [response]}

        def should_continue(state: MessagesState) -> str:
            """도구 호출이 있으면 tools로, 없으면 종료"""
            last_message = state["messages"][-1]
            if hasattr(last_message, "tool_calls") and last_message.tool_calls:
                return "tools"
            return END

        # ── 그래프 조립 ───────────────────────────────────

        graph = StateGraph(MessagesState)
        graph.add_node("agent", agent_node)
        graph.add_node("tools", tool_node)

        graph.add_edge(START, "agent")
        graph.add_conditional_edges(
            "agent",
            should_continue,
            {"tools": "tools", END: END}
        )
        graph.add_edge("tools", "agent")

        app = graph.compile()

        # ── 대화 루프 ────────────────────────────────────

        print("\n리서치 에이전트가 준비되었다. 질문을 입력하라. (종료: quit)\n")

        while True:
            user_input = input("사용자: ")
            if user_input.lower() in ("quit", "exit", "종료"):
                print("에이전트를 종료한다.")
                break

            result = await app.ainvoke({
                "messages": [
                    {"role": "user", "content": user_input}
                ]
            })

            # 최종 응답 출력
            final_message = result["messages"][-1]
            print(f"\n에이전트: {final_message.content}\n")


if __name__ == "__main__":
    asyncio.run(main())

정리

이번 챕터에서 배운 내용을 정리한다:

  • MCP + LangGraph를 결합하면 표준화된 도구 생태계와 정교한 에이전트 흐름 제어를 동시에 활용할 수 있다.
  • langchain-mcp-adapters의 MultiServerMCPClient를 사용하면 MCP 도구를 LangChain 도구로 자동 변환할 수 있다.
  • stdio 트랜스포트는 로컬 MCP 서버 연결에, streamable_http 트랜스포트는 원격 서버 연결에 사용한다.
  • 여러 MCP 서버를 동시에 연결하여 하나의 에이전트에서 모든 도구를 사용할 수 있다.
  • create_react_agent로 빠르게 에이전트를 만들거나, 커스텀 그래프로 세밀한 제어가 가능하다.
  • MCP 서버를 잘 설계하면(명확한 도구명, 상세한 docstring, 정확한 타입 힌트) LLM이 도구를 더 효과적으로 사용한다.
  • 민감한 도구에는 Human-in-the-loop 패턴을 적용하여 안전성을 확보할 수 있다.

이 챕터는 MCP 튜토리얼과 LangGraph 튜토리얼을 잇는 핵심 연결 고리이다. MCP로 만든 도구를 LangGraph 에이전트에서 사용하는 전체 흐름을 이해했으므로, 이제 자신만의 MCP 서버를 만들고 LangGraph 에이전트와 연동하는 실전 프로젝트에 도전해 보기 바란다.
