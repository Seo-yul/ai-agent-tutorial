RAG 튜토리얼
기본 과정

환경 설정

RAG 개발을 위한 Python 환경, 패키지, 벡터 DB를 구성한다

필수 도구

RAG 시스템을 구축하기 위해 다음 도구들이 필요하다.

도구 최소 버전 역할
Python 3.11+ 핵심 프로그래밍 언어
uv 0.4+ 패키지 관리 및 프로젝트 관리
Docker 24+ 벡터 DB 등 로컬 인프라 실행

Python 설치 확인

python3 --version
# 출력 예시: Python 3.12.4

Python 3.11 미만이 설치되어 있다면 python.org에서 최신 버전을 설치하거나, 아래 uv를 통해 Python을 관리할 수 있다.

uv 설치

uv는 Rust로 작성된 고속 Python 패키지 매니저이자 프로젝트 관리 도구이다. pip보다 10~100배 빠르며, 가상 환경을 자동으로 관리해준다.

macOS / Linux:

curl -LsSf https://astral.sh/uv/install.sh | sh

Windows (PowerShell):

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

설치 후 버전을 확인한다.

uv --version
# 출력 예시: uv 0.7.x

Docker 설치 확인

벡터 데이터베이스를 로컬에서 실행하기 위해 Docker가 필요하다.

docker --version
# 출력 예시: Docker version 27.x.x

# Docker 데몬이 실행 중인지 확인
docker ps
# 에러 없이 컨테이너 목록(비어있어도 됨)이 출력되면 정상

Docker가 없다면 Docker 공식 사이트에서 Docker Desktop을 설치한다.

TIP

macOS 사용자는 brew install --cask docker로도 Docker Desktop을 설치할 수 있다. Linux 사용자는 Docker Engine만 설치해도 충분하다.

프로젝트 생성

uv를 사용하여 RAG 프로젝트를 초기화한다.

# 새 프로젝트 생성
uv init rag-tutorial

# 프로젝트 디렉토리로 이동
cd rag-tutorial

uv init을 실행하면 pyproject.toml, .python-version, hello.py 등의 파일이 자동으로 생성된다. uv는 uv add 명령어를 실행할 때 가상 환경(.venv)을 자동으로 생성하고 관리한다.

INFO

uv를 사용하면 python -m venv, pip install, pip freeze 등을 직접 관리할 필요가 없다. uv add로 패키지를 추가하면 의존성 해결, 가상 환경 관리, lock 파일 생성이 모두 자동으로 처리된다.

핵심 패키지 설치

RAG 파이프라인 구축에 필요한 패키지들을 설치한다. 역할별로 나누어 살펴보자.

LangChain 핵심 패키지

# LangChain 프레임워크 및 LLM 통합
uv add langchain langchain-community langchain-openai langchain-anthropic

벡터 스토어 연동

# 벡터 DB 클라이언트
uv add langchain-qdrant langchain-postgres
uv add qdrant-client psycopg[binary] pgvector

문서 처리

# PDF 파싱 및 문서 전처리
uv add pypdf unstructured[pdf]

평가 프레임워크

# RAG 성능 평가
uv add ragas

패키지 역할 정리

패키지 역할
langchain RAG 파이프라인 구축을 위한 핵심 프레임워크. 체인, 프롬프트, 출력 파서 등을 제공한다.
langchain-community 커뮤니티가 기여한 통합 모듈. 다양한 문서 로더, 벡터 스토어 등을 포함한다.
langchain-openai OpenAI API 통합. ChatGPT와 임베딩 모델(text-embedding-3-small 등)을 사용할 수 있다.
langchain-anthropic Anthropic Claude API 통합. Claude 모델을 LLM으로 사용할 수 있다.
langchain-qdrant Qdrant 벡터 DB와의 LangChain 통합.
langchain-postgres PostgreSQL(pgvector)과의 LangChain 통합.
qdrant-client Qdrant 벡터 DB Python 클라이언트.
psycopg[binary] PostgreSQL Python 드라이버 (psycopg3).
pgvector pgvector 확장과 상호작용하기 위한 Python 라이브러리.
pypdf PDF 파일에서 텍스트를 추출하는 순수 Python 라이브러리.
unstructured[pdf] 복잡한 레이아웃의 PDF도 처리할 수 있는 고급 문서 파서.
ragas RAG 성능 평가 프레임워크. Faithfulness, Answer Relevancy 등의 메트릭을 제공한다.
WARNING

unstructured[pdf]는 시스템 레벨 의존성(poppler, tesseract 등)이 필요할 수 있다. macOS에서는 brew install poppler tesseract, Ubuntu에서는 sudo apt install poppler-utils tesseract-ocr로 설치한다. 간단한 PDF만 다룬다면 pypdf만으로도 충분하다.

API 키 설정

LLM과 임베딩 모델을 사용하려면 API 키가 필요하다. OpenAI와 Anthropic 중 하나만 있어도 진행할 수 있지만, 두 가지 모두 준비하면 모델을 비교해볼 수 있다.

API 키 발급

  • OpenAI: platform.openai.com/api-keys에서 발급한다.
  • Anthropic: console.anthropic.com/settings/keys에서 발급한다.

환경 변수로 설정

터미널에서 직접 환경 변수를 설정할 수 있다.

# OpenAI API 키
export OPENAI_API_KEY="sk-..."

# Anthropic API 키
export ANTHROPIC_API_KEY="sk-ant-..."

.env 파일 사용 (권장)

매번 환경 변수를 설정하는 것은 번거롭다. 프로젝트 루트에 .env 파일을 만들어 API 키를 관리하는 것이 편리하다.

BASH 
# .env 파일 생성
cat << 'EOF' > .env
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
EOF

python-dotenv를 설치하면 Python에서 .env 파일을 자동으로 읽을 수 있다.

uv add python-dotenv

Python 코드에서 다음과 같이 사용한다.

PYTHON 
from dotenv import load_dotenv
import os

# .env 파일에서 환경 변수 로드
load_dotenv()

# 환경 변수 사용
openai_key = os.getenv("OPENAI_API_KEY")
anthropic_key = os.getenv("ANTHROPIC_API_KEY")
WARNING

.env 파일에는 민감한 API 키가 포함되므로, 반드시 .gitignore에 추가해야 한다. 실수로 GitHub에 API 키를 푸시하면 과금 피해를 입을 수 있다.

BASH 
# .gitignore에 .env 추가
echo ".env" >> .gitignore

벡터 스토어 설치 (Docker)

RAG 시스템에서 벡터 스토어는 임베딩된 문서를 저장하고 유사도 검색을 수행하는 핵심 인프라이다. 이 튜토리얼에서는 pgvector와 Qdrant 두 가지를 다룬다. 둘 중 하나만 설치해도 되고, 둘 다 설치하여 비교해볼 수도 있다.

옵션 A: pgvector (PostgreSQL + 벡터 확장)

pgvector는 PostgreSQL에 벡터 검색 기능을 추가하는 확장(extension)이다. 이미 PostgreSQL을 사용하고 있는 조직이라면, 별도의 벡터 DB를 도입하지 않고 기존 인프라에 벡터 검색을 추가할 수 있다는 장점이 있다.

BASH 
# pgvector가 포함된 PostgreSQL 16 컨테이너 실행
docker run -d --name pgvector \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=rag_tutorial \
  -p 5432:5432 \
  pgvector/pgvector:pg16

컨테이너가 정상적으로 실행되었는지 확인한다.

BASH 
# 컨테이너 상태 확인
docker ps --filter name=pgvector

# PostgreSQL 접속 테스트
docker exec -it pgvector psql -U postgres -d rag_tutorial -c "SELECT 1;"

pgvector 확장을 활성화한다.

BASH 
# pgvector 확장 생성
docker exec -it pgvector psql -U postgres -d rag_tutorial -c "CREATE EXTENSION IF NOT EXISTS vector;"

# 확장이 정상 설치되었는지 확인
docker exec -it pgvector psql -U postgres -d rag_tutorial -c "SELECT extname, extversion FROM pg_extension WHERE extname = 'vector';"
INFO

pgvector는 vector 타입과 함께 cosine distance, L2 distance, inner product 등의 거리 연산자를 제공한다. HNSW, IVFFlat 인덱스를 지원하여 대규모 데이터에서도 빠른 검색이 가능하다.

옵션 B: Qdrant

Qdrant는 벡터 검색에 특화된 전용 데이터베이스이다. Rust로 작성되어 성능이 뛰어나고, REST API와 gRPC를 모두 지원한다. 필터링, 페이로드 저장, 멀티테넌시 등 벡터 검색에 특화된 기능이 풍부하다.

BASH 
# Qdrant 컨테이너 실행
docker run -d --name qdrant \
  -p 6333:6333 \
  -p 6334:6334 \
  -v qdrant_data:/qdrant/storage \
  qdrant/qdrant

Qdrant가 정상적으로 실행되었는지 확인한다.

BASH 
# 컨테이너 상태 확인
docker ps --filter name=qdrant

# Qdrant 헬스체크 (REST API)
curl -s http://localhost:6333/healthz
# 출력: OK

Qdrant는 웹 대시보드도 제공한다. 브라우저에서 http://localhost:6333/dashboard에 접속하면 컬렉션 관리, 데이터 탐색 등을 GUI로 할 수 있다.

TIP

Qdrant는 -v qdrant_data:/qdrant/storage 옵션으로 Docker 볼륨을 마운트했다. 이렇게 하면 컨테이너를 삭제해도 데이터가 보존된다. pgvector도 같은 방식으로 -v pgdata:/var/lib/postgresql/data를 추가하면 데이터를 영속화할 수 있다.

pgvector vs Qdrant 간략 비교

비교 항목 pgvector Qdrant
유형 PostgreSQL 확장 전용 벡터 DB
장점 기존 PostgreSQL 인프라 활용, SQL 쿼리와 벡터 검색 결합 가능 벡터 검색 최적화, 풍부한 필터링, 웹 대시보드
적합한 경우 이미 PostgreSQL을 사용 중이거나 관계형 데이터와 벡터를 함께 다뤄야 할 때 벡터 검색 성능이 최우선이거나 대규모 벡터 데이터를 다룰 때
포트 5432 6333 (REST), 6334 (gRPC)
INFO

벡터 스토어에 대한 더 자세한 비교와 활용법은 06장 벡터 스토어에서 다룬다. 지금은 로컬 개발 환경에서 하나를 띄워두기만 하면 충분하다.

설치 검증

모든 설치가 정상적으로 완료되었는지 Python 스크립트로 확인한다.

PYTHON 
"""verify_setup.py - RAG 튜토리얼 환경 검증 스크립트"""

import sys

def check_python_version():
    """Python 버전 확인"""
    version = sys.version_info
    assert version.major == 3 and version.minor >= 11, \
        f"Python 3.11+ 필요 (현재: {version.major}.{version.minor})"
    print(f"[OK] Python {version.major}.{version.minor}.{version.micro}")

def check_packages():
    """핵심 패키지 import 확인"""
    packages = {
        "langchain": "langchain",
        "langchain_openai": "langchain-openai",
        "langchain_anthropic": "langchain-anthropic",
        "langchain_qdrant": "langchain-qdrant",
        "langchain_postgres": "langchain-postgres",
        "pypdf": "pypdf",
        "ragas": "ragas",
    }

    for module_name, package_name in packages.items():
        try:
            __import__(module_name)
            print(f"[OK] {package_name}")
        except ImportError:
            print(f"[FAIL] {package_name} - 'uv add {package_name}'으로 설치 필요")

def check_api_keys():
    """API 키 설정 확인"""
    import os
    try:
        from dotenv import load_dotenv
        load_dotenv()
    except ImportError:
        pass

    openai_key = os.getenv("OPENAI_API_KEY", "")
    anthropic_key = os.getenv("ANTHROPIC_API_KEY", "")

    if openai_key and openai_key.startswith("sk-"):
        print("[OK] OPENAI_API_KEY 설정됨")
    else:
        print("[WARN] OPENAI_API_KEY 미설정")

    if anthropic_key and anthropic_key.startswith("sk-ant-"):
        print("[OK] ANTHROPIC_API_KEY 설정됨")
    else:
        print("[WARN] ANTHROPIC_API_KEY 미설정")

def check_pgvector():
    """pgvector 연결 확인"""
    try:
        import psycopg
        conn = psycopg.connect(
            "postgresql://postgres:postgres@localhost:5432/rag_tutorial"
        )
        cur = conn.execute("SELECT 1")
        cur.fetchone()
        conn.close()
        print("[OK] pgvector 연결 성공")
    except Exception as e:
        print(f"[SKIP] pgvector 연결 실패 - {e}")

def check_qdrant():
    """Qdrant 연결 확인"""
    try:
        from qdrant_client import QdrantClient
        client = QdrantClient(host="localhost", port=6333)
        client.get_collections()
        print("[OK] Qdrant 연결 성공")
    except Exception as e:
        print(f"[SKIP] Qdrant 연결 실패 - {e}")

def check_llm_connection():
    """LLM API 연결 확인"""
    import os
    try:
        from dotenv import load_dotenv
        load_dotenv()
    except ImportError:
        pass

    # OpenAI 테스트
    if os.getenv("OPENAI_API_KEY"):
        try:
            from langchain_openai import ChatOpenAI
            llm = ChatOpenAI(model="gpt-4o-mini", max_tokens=10)
            response = llm.invoke("Hello")
            print(f"[OK] OpenAI API 연결 성공")
        except Exception as e:
            print(f"[FAIL] OpenAI API 연결 실패 - {e}")

    # Anthropic 테스트
    if os.getenv("ANTHROPIC_API_KEY"):
        try:
            from langchain_anthropic import ChatAnthropic
            llm = ChatAnthropic(model="claude-sonnet-4-20250514", max_tokens=10)
            response = llm.invoke("Hello")
            print(f"[OK] Anthropic API 연결 성공")
        except Exception as e:
            print(f"[FAIL] Anthropic API 연결 실패 - {e}")

if __name__ == "__main__":
    print("=" * 50)
    print("RAG 튜토리얼 환경 검증")
    print("=" * 50)

    print("\n--- Python 버전 ---")
    check_python_version()

    print("\n--- 패키지 설치 ---")
    check_packages()

    print("\n--- API 키 ---")
    check_api_keys()

    print("\n--- 벡터 DB ---")
    check_pgvector()
    check_qdrant()

    print("\n--- LLM 연결 ---")
    check_llm_connection()

    print("\n" + "=" * 50)
    print("검증 완료!")
    print("=" * 50)

프로젝트 구조

앞으로의 챕터에서 점진적으로 파일을 추가해나갈 것이다. 최종적으로 완성될 프로젝트의 디렉토리 구조는 다음과 같다. 

rag-tutorial/
├── data/               # PDF 등 원본 문서
│   └── sample.pdf
├── src/
│   ├── __init__.py
│   ├── loader.py       # 문서 로딩 (03장)
│   ├── chunker.py      # 청킹 전략 (04장)
│   ├── embedder.py     # 임베딩 (05장)
│   ├── store.py        # 벡터 저장 (06장)
│   ├── retriever.py    # 검색 (07장)
│   └── chain.py        # RAG 체인 (07~08장)
├── tests/
│   └── test_pipeline.py
├── .env                # API 키 (gitignore 대상)
├── .gitignore
├── pyproject.toml      # 프로젝트 설정 및 의존성
├── uv.lock             # 의존성 lock 파일
└── verify_setup.py     # 환경 검증 스크립트

각 파일의 역할을 정리하면 다음과 같다.

파일/디렉토리 역할 관련 챕터
data/ RAG에 사용할 원본 문서(PDF 등)를 저장하는 디렉토리 03장
src/loader.py PDF, HTML 등 다양한 형식의 문서를 LangChain Document 객체로 변환 03장
src/chunker.py 문서를 의미 있는 청크로 분할하는 다양한 전략 구현 04장
src/embedder.py 텍스트 청크를 벡터로 변환하는 임베딩 모델 래퍼 05장
src/store.py 벡터를 저장하고 유사도 검색을 수행하는 벡터 스토어 관리 06장
