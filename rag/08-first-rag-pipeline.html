RAG 튜토리얼
기본 과정

첫 번째 RAG 파이프라인

지금까지 배운 모든 구성요소를 조립하여 end-to-end RAG 시스템을 완성한다

전체 아키텍처 복습

기본 과정의 마지막 챕터이다. 지금까지 문서 로딩, 청킹, 임베딩, 벡터 저장, 검색, 생성을 하나씩 배워왔다. 이 챕터에서는 이 모든 요소를 하나의 프로젝트로 조립하여 실제로 실행 가능한 완전한 RAG 시스템을 구축한다.

단계별 사용 도구 요약

단계 도구 파일
설정 pydantic-settings, python-dotenv config.py
문서 로딩 PyPDFLoader ingest.py
청킹 RecursiveCharacterTextSplitter ingest.py
임베딩 OpenAI text-embedding-3-small ingest.py, retriever.py
벡터 저장 pgvector 또는 Qdrant ingest.py, retriever.py
검색 LangChain Retriever retriever.py
생성 LCEL Chain + ChatOpenAI/ChatAnthropic chain.py
실행 argparse CLI main.py

프로젝트 구조

하나의 거대한 스크립트에 모든 코드를 넣지 않는다. 역할별로 모듈을 분리하여 유지보수가 쉬운 구조로 만든다.

Step 1: 설정 (config.py)

모든 설정을 한 곳에서 관리한다. pydantic-settings를 사용하면 환경 변수를 타입 안전하게 읽을 수 있고, 기본값 설정과 유효성 검증도 자동으로 처리된다.

PYTHON 
# src/config.py
"""RAG 파이프라인 설정."""

from enum import Enum

from pydantic_settings import BaseSettings, SettingsConfigDict


class VectorStoreType(str, Enum):
    """지원하는 벡터 스토어 종류."""
    PGVECTOR = "pgvector"
    QDRANT = "qdrant"


class LLMProvider(str, Enum):
    """지원하는 LLM 프로바이더."""
    OPENAI = "openai"
    ANTHROPIC = "anthropic"


class Settings(BaseSettings):
    """환경 변수 기반 설정 클래스."""

    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
    )

    # API 키
    openai_api_key: str = ""
    anthropic_api_key: str = ""

    # 임베딩 설정
    embedding_model: str = "text-embedding-3-small"

    # 벡터 스토어 설정
    vector_store_type: VectorStoreType = VectorStoreType.PGVECTOR
    postgres_connection: str = "postgresql+psycopg://user:password@localhost:5432/ragdb"
    qdrant_url: str = "http://localhost:6333"
    qdrant_collection: str = "my_documents"
    collection_name: str = "my_documents"

    # 청킹 설정
    chunk_size: int = 1000
    chunk_overlap: int = 200

    # 검색 설정
    search_k: int = 4
    search_type: str = "similarity"

    # LLM 설정
    llm_provider: LLMProvider = LLMProvider.OPENAI
    openai_model: str = "gpt-4o-mini"
    anthropic_model: str = "claude-sonnet-4-20250514"
    llm_temperature: float = 0.0
    llm_max_tokens: int = 1024


# 싱글턴 인스턴스
settings = Settings()
TIP

pydantic-settings의 BaseSettings는 환경 변수 이름을 자동으로 매핑한다. 예를 들어 openai_api_key 필드는 OPENAI_API_KEY 환경 변수에서 값을 읽는다. 대소문자 변환이 자동으로 이루어지므로 별도 매핑이 필요 없다.

Step 2: 인덱싱 파이프라인 (ingest.py)

인덱싱 파이프라인은 PDF 문서를 받아서 로딩 → 청킹 → 임베딩 → 벡터 저장을 수행한다. 이 과정은 문서당 한 번만 실행하면 된다.

PYTHON 
# src/ingest.py
"""문서 인덱싱 파이프라인: 로딩 → 청킹 → 임베딩 → 벡터 저장."""

import hashlib
from pathlib import Path

from langchain_community.document_loaders import PyPDFLoader
from langchain_core.documents import Document
from langchain_openai import OpenAIEmbeddings
from langchain_text_splitters import RecursiveCharacterTextSplitter

from src.config import Settings, VectorStoreType, settings


def load_pdf(file_path: str) -> list[Document]:
    """PDF 파일을 로딩하여 Document 리스트로 반환한다."""
    path = Path(file_path)
    if not path.exists():
        raise FileNotFoundError(f"파일을 찾을 수 없다: {file_path}")
    if not path.suffix.lower() == ".pdf":
        raise ValueError(f"PDF 파일만 지원한다: {file_path}")

    loader = PyPDFLoader(str(path))
    documents = loader.load()

    print(f"  로딩 완료: {len(documents)}개 페이지")
    return documents


def chunk_documents(
    documents: list[Document],
    chunk_size: int | None = None,
    chunk_overlap: int | None = None,
) -> list[Document]:
    """문서를 청크 단위로 분할한다."""
    _chunk_size = chunk_size or settings.chunk_size
    _chunk_overlap = chunk_overlap or settings.chunk_overlap

    splitter = RecursiveCharacterTextSplitter(
        chunk_size=_chunk_size,
        chunk_overlap=_chunk_overlap,
        separators=["\n\n", "\n", ". ", " ", ""],
        length_function=len,
    )

    chunks = splitter.split_documents(documents)

    # 각 청크에 고유 ID 부여 (중복 인덱싱 방지용)
    for chunk in chunks:
        content_hash = hashlib.md5(chunk.page_content.encode()).hexdigest()[:12]
        source = chunk.metadata.get("source", "unknown")
        page = chunk.metadata.get("page", 0)
        chunk.metadata["chunk_id"] = f"{source}:p{page}:{content_hash}"

    print(f"  청킹 완료: {len(documents)}개 페이지 → {len(chunks)}개 청크")
    print(f"  청크 크기: {_chunk_size}, 오버랩: {_chunk_overlap}")
    return chunks


def get_embeddings() -> OpenAIEmbeddings:
    """임베딩 모델 인스턴스를 반환한다."""
    return OpenAIEmbeddings(model=settings.embedding_model)


def store_to_pgvector(chunks: list[Document], embeddings: OpenAIEmbeddings) -> None:
    """청크를 pgvector에 저장한다."""
    from langchain_postgres import PGVector

    PGVector.from_documents(
        documents=chunks,
        embedding=embeddings,
        collection_name=settings.collection_name,
        connection=settings.postgres_connection,
        pre_delete_collection=False,  # 기존 데이터 유지
    )
    print(f"  pgvector 저장 완료: {len(chunks)}개 청크")


def store_to_qdrant(chunks: list[Document], embeddings: OpenAIEmbeddings) -> None:
    """청크를 Qdrant에 저장한다."""
    from langchain_qdrant import QdrantVectorStore
    from qdrant_client import QdrantClient

    client = QdrantClient(url=settings.qdrant_url)

    QdrantVectorStore.from_documents(
        documents=chunks,
        embedding=embeddings,
        url=settings.qdrant_url,
        collection_name=settings.qdrant_collection,
    )
    print(f"  Qdrant 저장 완료: {len(chunks)}개 청크")


def ingest(file_path: str) -> int:
    """전체 인덱싱 파이프라인을 실행한다.

    Args:
        file_path: PDF 파일 경로.

    Returns:
        저장된 청크 수.
    """
    print(f"\n{'='*50}")
    print(f"인덱싱 시작: {file_path}")
    print(f"{'='*50}")

    # 1. 문서 로딩
    print("\n[1/4] 문서 로딩...")
    documents = load_pdf(file_path)

    # 2. 청킹
    print("\n[2/4] 문서 청킹...")
    chunks = chunk_documents(documents)

    # 3. 임베딩 모델 준비
    print("\n[3/4] 임베딩 모델 준비...")
    embeddings = get_embeddings()
    print(f"  모델: {settings.embedding_model}")

    # 4. 벡터 스토어 저장
    print(f"\n[4/4] 벡터 스토어 저장 ({settings.vector_store_type.value})...")
    if settings.vector_store_type == VectorStoreType.PGVECTOR:
        store_to_pgvector(chunks, embeddings)
    elif settings.vector_store_type == VectorStoreType.QDRANT:
        store_to_qdrant(chunks, embeddings)

    print(f"\n{'='*50}")
    print(f"인덱싱 완료! 총 {len(chunks)}개 청크 저장됨")
    print(f"{'='*50}\n")

    return len(chunks)

중복 인덱싱 방지

위 코드에서 chunk_id를 생성하는 로직에 주목하라. 소스 파일명, 페이지 번호, 내용의 MD5 해시를 조합하여 각 청크에 고유 ID를 부여한다. 이 ID를 활용하면 같은 문서를 다시 인덱싱할 때 중복을 감지하고 건너뛸 수 있다.

INFO

프로덕션 환경에서는 중복 인덱싱 방지가 매우 중요하다. 같은 문서가 여러 번 저장되면 검색 시 동일한 내용이 중복 반환되어 LLM의 컨텍스트 윈도우를 낭비하게 된다. chunk_id를 벡터 스토어의 document ID로 활용하면 upsert(있으면 업데이트, 없으면 삽입) 패턴을 적용할 수 있다.

Step 3: 검색기 설정 (retriever.py)

검색기(Retriever)는 벡터 스토어에 연결하여 사용자 질문과 관련된 문서를 찾는 역할을 한다. 인덱싱 때 사용한 것과 동일한 임베딩 모델과 동일한 벡터 스토어에 연결해야 한다.

PYTHON 
# src/retriever.py
"""벡터 스토어 연결 및 Retriever 설정."""

from langchain_core.vectorstores import VectorStoreRetriever
from langchain_openai import OpenAIEmbeddings

from src.config import VectorStoreType, settings


def get_embeddings() -> OpenAIEmbeddings:
    """인덱싱 때와 동일한 임베딩 모델을 반환한다."""
    return OpenAIEmbeddings(model=settings.embedding_model)


def get_pgvector_store():
    """pgvector 벡터 스토어 인스턴스를 반환한다."""
    from langchain_postgres import PGVector

    return PGVector(
        collection_name=settings.collection_name,
        connection=settings.postgres_connection,
        embeddings=get_embeddings(),
    )


def get_qdrant_store():
    """Qdrant 벡터 스토어 인스턴스를 반환한다."""
    from langchain_qdrant import QdrantVectorStore
    from qdrant_client import QdrantClient

    client = QdrantClient(url=settings.qdrant_url)

    return QdrantVectorStore(
        client=client,
        collection_name=settings.qdrant_collection,
        embedding=get_embeddings(),
    )


def get_retriever() -> VectorStoreRetriever:
    """설정에 따라 적절한 Retriever를 반환한다."""
    if settings.vector_store_type == VectorStoreType.PGVECTOR:
        store = get_pgvector_store()
    elif settings.vector_store_type == VectorStoreType.QDRANT:
        store = get_qdrant_store()
    else:
        raise ValueError(f"지원하지 않는 벡터 스토어: {settings.vector_store_type}")

    retriever = store.as_retriever(
        search_type=settings.search_type,
        search_kwargs={"k": settings.search_k},
    )

    return retriever

이 모듈의 핵심은 get_retriever() 함수이다. 설정 파일의 VECTOR_STORE_TYPE 값에 따라 pgvector 또는 Qdrant에 연결하고, LangChain Retriever 인터페이스를 반환한다. 호출하는 쪽에서는 어떤 벡터 스토어를 사용하는지 신경 쓸 필요 없이 retriever.invoke(query)만 호출하면 된다.

Step 4: RAG 체인 (chain.py)

RAG 체인은 검색기(Retriever), 프롬프트, LLM을 하나로 연결하는 핵심 모듈이다. LCEL을 사용하여 각 컴포넌트를 파이프라인으로 구성한다.

PYTHON 
# src/chain.py
"""RAG 체인: 검색 + 프롬프트 + LLM."""

from langchain_core.documents import Document
from langchain_core.language_models import BaseChatModel
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import Runnable, RunnableParallel, RunnablePassthrough

from src.config import LLMProvider, settings
from src.retriever import get_retriever


# ── 프롬프트 템플릿 ──────────────────────────────────────

RAG_PROMPT = ChatPromptTemplate.from_messages([
    ("system", """너는 주어진 문서를 기반으로 질문에 답변하는 AI 어시스턴트이다.
다음 규칙을 반드시 따르라:

1. 반드시 제공된 문서 내용만을 근거로 답변하라.
2. 문서에 답변할 수 있는 정보가 없으면 "제공된 문서에서 해당 정보를 찾을 수 없습니다"라고 명확히 말하라.
3. 답변할 때 근거가 되는 문서의 출처(파일명, 페이지)를 함께 명시하라.
4. 추측이나 문서 외부의 지식을 사용하지 마라.
5. 답변은 명확하고 구조화된 형태로 작성하라."""),

    ("human", """다음 문서를 참고하여 질문에 답변하라.

[문서]
{context}

[질문]
{question}"""),
])


# ── 유틸리티 함수 ──────────────────────────────────────

def format_docs(docs: list[Document]) -> str:
    """검색된 문서 리스트를 하나의 문자열로 합친다."""
    formatted = []
    for i, doc in enumerate(docs, 1):
        source = doc.metadata.get("source", "알 수 없음")
        page = doc.metadata.get("page", "")
        header = f"[문서 {i}] (출처: {source}"
        if page != "":
            header += f", 페이지: {page}"
        header += ")"
        formatted.append(f"{header}\n{doc.page_content}")
    return "\n\n".join(formatted)


def extract_sources(docs: list[Document]) -> list[dict]:
    """문서의 출처 정보만 추출한다."""
    sources = []
    seen = set()
    for doc in docs:
        source = doc.metadata.get("source", "N/A")
        page = doc.metadata.get("page", "N/A")
        key = f"{source}:{page}"
        if key not in seen:
            seen.add(key)
            sources.append({
                "source": source,
                "page": page,
                "preview": doc.page_content[:100] + "...",
            })
    return sources


# ── LLM 팩토리 ──────────────────────────────────────

def get_llm() -> BaseChatModel:
    """설정에 따라 적절한 LLM 인스턴스를 반환한다."""
    if settings.llm_provider == LLMProvider.OPENAI:
        from langchain_openai import ChatOpenAI
        return ChatOpenAI(
            model=settings.openai_model,
            temperature=settings.llm_temperature,
            max_tokens=settings.llm_max_tokens,
        )
    elif settings.llm_provider == LLMProvider.ANTHROPIC:
        from langchain_anthropic import ChatAnthropic
        return ChatAnthropic(
            model=settings.anthropic_model,
            temperature=settings.llm_temperature,
            max_tokens=settings.llm_max_tokens,
        )
    else:
        raise ValueError(f"지원하지 않는 LLM 프로바이더: {settings.llm_provider}")


# ── 체인 생성 ──────────────────────────────────────

def create_rag_chain() -> Runnable:
    """답변만 반환하는 기본 RAG 체인을 생성한다."""
    retriever = get_retriever()
    llm = get_llm()

    chain = (
        {"context": retriever | format_docs, "question": RunnablePassthrough()}
        | RAG_PROMPT
        | llm
        | StrOutputParser()
    )

    return chain


def create_rag_chain_with_sources() -> Runnable:
    """답변과 출처를 함께 반환하는 RAG 체인을 생성한다."""
    retriever = get_retriever()
    llm = get_llm()

    chain = RunnableParallel(
        answer=(
            {"context": retriever | format_docs, "question": RunnablePassthrough()}
            | RAG_PROMPT
            | llm
            | StrOutputParser()
        ),
        sources=retriever | extract_sources,
    )

    return chain

LLM 교체가 쉬운 구조

get_llm() 함수는 .env의 LLM_PROVIDER 설정만 바꾸면 OpenAI에서 Claude로 즉시 전환할 수 있도록 설계되었다. 코드를 수정할 필요 없이 환경 변수 하나만 변경하면 된다.

BASH

# Claude로 전환 (코드 수정 불필요)

Step 5: 실행 (main.py)

main.py는 CLI(Command Line Interface)로 두 가지 모드를 제공한다: ingest(문서 인덱싱)와 query(질의응답)이다.

PYTHON 
# main.py
"""RAG 파이프라인 CLI 진입점."""

import argparse
import sys
import time


def cmd_ingest(args: argparse.Namespace) -> None:
    """문서 인덱싱 명령을 실행한다."""
    from src.ingest import ingest

    start = time.time()
    num_chunks = ingest(args.path)
    elapsed = time.time() - start

    print(f"소요 시간: {elapsed:.1f}초")
    print(f"저장된 청크 수: {num_chunks}개")


def cmd_query(args: argparse.Namespace) -> None:
    """질의응답 명령을 실행한다."""
    from src.chain import create_rag_chain_with_sources

    chain = create_rag_chain_with_sources()

    question = args.question
    print(f"\n질문: {question}")
    print(f"{'─'*50}")

    start = time.time()
    result = chain.invoke(question)
    elapsed = time.time() - start

    print(f"\n답변:\n{result['answer']}")
    print(f"\n{'─'*50}")
    print("참고 출처:")
    for src in result["sources"]:
        print(f"  - {src['source']} (p.{src['page']})")
    print(f"\n소요 시간: {elapsed:.1f}초")


def cmd_interactive(args: argparse.Namespace) -> None:
    """대화형 질의응답 모드를 실행한다."""
    from src.chain import create_rag_chain_with_sources

    chain = create_rag_chain_with_sources()

    print("\n대화형 RAG 모드 (종료: quit 또는 Ctrl+C)")
    print(f"{'='*50}")

    while True:
        try:
            question = input("\n질문: ").strip()
            if not question:
                continue
            if question.lower() in ("quit", "exit", "q"):
                print("종료.")
                break

            start = time.time()
            result = chain.invoke(question)
            elapsed = time.time() - start

            print(f"\n답변:\n{result['answer']}")
            print(f"\n참고 출처:")
            for src in result["sources"]:
                print(f"  - {src['source']} (p.{src['page']})")
            print(f"(소요 시간: {elapsed:.1f}초)")

        except KeyboardInterrupt:
            print("\n종료.")
            break


def main() -> None:
    """CLI 진입점."""
    parser = argparse.ArgumentParser(
        description="RAG 파이프라인 CLI",
        formatter_class=argparse.RawDescriptionHelpFormatter,
        epilog="""
사용 예시:
  # PDF 문서 인덱싱
  python main.py ingest --path data/sample.pdf

  # 단일 질의
  python main.py query "이 문서의 핵심 내용은 무엇인가?"

  # 대화형 모드
  python main.py interactive
        """,
    )

    subparsers = parser.add_subparsers(dest="command", help="실행할 명령")

    # ingest 명령
    ingest_parser = subparsers.add_parser("ingest", help="PDF 문서를 인덱싱한다")
    ingest_parser.add_argument(
        "--path", required=True, help="인덱싱할 PDF 파일 경로"
    )

    # query 명령
    query_parser = subparsers.add_parser("query", help="질문에 답변한다")
    query_parser.add_argument("question", help="질문 내용")

    # interactive 명령
    subparsers.add_parser("interactive", help="대화형 질의응답 모드")

    args = parser.parse_args()

    if args.command is None:
        parser.print_help()
        sys.exit(1)

    if args.command == "ingest":
        cmd_ingest(args)
    elif args.command == "query":
        cmd_query(args)
    elif args.command == "interactive":
        cmd_interactive(args)


if __name__ == "__main__":
    main()

발생할 수 있는 문제와 해결

처음 RAG를 구축하면 다양한 문제에 부딪힌다. 아래는 가장 흔한 문제와 해결 방법이다.

증상 원인 해결 방법
"관련 문서를 찾지 못함" 청크 크기가 너무 크거나, 임베딩 모델의 한계 청크 크기를 500~800으로 줄이기, text-embedding-3-large로 교체
"환각이 발생함" 프롬프트에 환각 방지 지시가 약하거나, 관련 없는 문서가 검색됨 프롬프트에 "문서에 없으면 모른다고 하라" 강조, k값 줄이기
"답변이 너무 일반적" 검색된 문서가 너무 적거나, 청크 크기가 너무 작음 k값을 6~8로 증가, 청크 크기를 1200~1500으로 늘리기
"속도가 느림" 임베딩/LLM API 호출 지연, 벡터 검색 속도 임베딩 결과 캐시, 벡터 인덱스 튜닝 (HNSW), 스트리밍 적용
"중복된 내용이 검색됨" 유사한 청크가 여러 개 존재 MMR 검색(search_type="mmr")으로 전환
"API 키 오류" .env 파일의 키가 잘못되었거나 누락됨 .env 파일 확인, API 키 재발급
WARNING

문제가 발생하면 한 번에 여러 설정을 바꾸지 마라. 한 번에 하나씩 변경하고 결과를 확인해야 어떤 변경이 효과가 있었는지 정확히 파악할 수 있다. 예를 들어, 청크 크기와 k값을 동시에 바꾸면 어느 쪽의 영향인지 알 수 없다.

기본 RAG(Naive RAG)의 한계

지금까지 구축한 것은 Naive RAG이다. 동작은 하지만, 프로덕션 환경에서는 여러 한계에 부딪힌다:

  • 검색 품질 한계: 단순 유사도 검색만으로는 사용자의 의도를 정확히 파악하지 못한다. "회사의 휴가 정책이 뭐야?"라는 질문에 "휴가"라는 단어가 포함된 무관한 문서가 검색될 수 있다.
  • 키워드 매칭 실패: 의미적으로는 관련이 있지만 표현이 다른 경우(예: "연차" vs "유급 휴가")를 놓칠 수 있다. 벡터 검색이 이를 어느 정도 해결하지만, 완벽하지는 않다.
  • 컨텍스트 부족/과잉: 고정된 k값으로는 질문의 복잡도에 맞는 적절한 양의 문서를 제공하기 어렵다.
  • 다중 질문 처리 불가: "A의 장점과 B의 단점을 비교하라"와 같은 복합 질문에 대응하기 어렵다.

다음 단계

기본 과정을 완료했다. 이제 동작하는 RAG 시스템이 있고, 그 한계도 이해했다.

심화 과정에서는 이 한계를 극복하는 고급 기법들을 하나씩 적용해 나갈 것이다:

  • 쿼리 변환(Query Transformation): 사용자 질문을 검색에 최적화된 형태로 변환한다. HyDE, 쿼리 분해, Step-back Prompting 등.
  • 하이브리드 검색(Hybrid Search): 벡터 검색과 키워드 검색(BM25)을 결합하여 검색 정확도를 높인다.
  • Re-ranking: Cross-encoder를 사용하여 검색 결과를 재순위화한다.
  • RAG 평가(Evaluation): RAGAS 프레임워크로 RAG 성능을 정량적으로 측정한다.
  • 프로덕션 배포: Kubernetes 환경에서 RAG 시스템을 운영한다.

각 기법을 적용할 때마다 이 챕터에서 만든 파이프라인 위에 점진적으로 추가하므로, 기본 코드를 잘 이해해두면 심화 과정이 훨씬 수월하다.
